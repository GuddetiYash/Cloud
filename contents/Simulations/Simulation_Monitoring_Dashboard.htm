Overview

The Simulation Dashboard feature offers real-time progress updates on the Status, Key metrics, and logs for ongoing Simulations. It provides users with the necessary information to assess whether the Simulation is progressing as anticipated.  

If deviations are detected, users can make informed decisions, including the option to cancel the simulation. 

NOTE 1: To view the data within the Simulation Dashboard, you need to have the Diagnostics parameter enabled under the model settings in PLEXOS Desktop. 

 

NOTE 2: As the Simulation Monitoring Dashboard is currently in Beta release, there are a few limitations to be aware of.  

  • Firstly, the dashboard only populates data for the studies that have locale settings of en-US, en-AU and en- GB.
  • If your model is configured with only one region, you may not be able to view certain charts such as Load, Unserved Energy, Price, and Generation, as they will not load. Else all other components of the dashboard will remain visible and accessible. 
  • Currently, this feature is available only for Power and Gas Modules.  

 

Once you have completed the above settings, you can get the dashboard by following the below steps: 

  1. Navigate to the Simulations page within a Study. 
  2. Find a Simulation that is currently in Progress and Double-click anywhere in the row. 
  3. A new tab will open on the Simulations page showing the Simulation Dashboard as shown in the images below. 
  4. The charts shown will continuously update as the Simulation progresses. You have the option to cancel the Run at any time during this process. 

Power Model Widgets

To gain a clear understanding of each chart's meaning, please refer to the following definitions:

  • Unserved Energy : Unserved Energy is System Load that could not be met due to a shortage in Generation and/or transmission capacity.
  • Memory: Memory here refers to the memory consumption of the current phase per step.
  • Load: Load is the total load that occurred in the system.
  • Generation: Generation Chart represents System level generation per step per phase. It shows data for the current running phase only.
  • Estimated time to completion: It is the Estimated time for the current running Simulation Phase. 
  • Status Count : It is the progress of the Simulation phases for your current Model run in percentage 
  • Price : It is the average System price.
  • Time: It is the total time taken to solve a current step per phase.

Gas Simulations Widgets

The  Gas Simulations widgets are Gas Demand, Gas Supply/Gas Production, Gas Shortage, and Gas Price which shows the data related to Gas Simulations. 

  • Gas Demand: This refers to the collective gas demand originating from all gas zones within the current model being executed. Gas Demand is measured in terajoules (TJ), over a specified period. The X-Axis represents the timeline for Step Start Date and Y-Axis represents the Gas Demand in in terajoules (TJ). 
  • Gas Supply: Gas Supply represents the total gas output from all gas fields across the various gas zones within the current model being executed. The X-Axis represents the timeline for Step Start Date and Y-Axis represents the Gas Supply in terajoules (TJ). This widget is visible only when the Model engine Version is 9.2 R07 and above. 
  • Gas Production: Gas Production denotes the cumulative output of gas from all gas fields across the different gas zones within the current model being executed. This widget is visible only when the Model engine Version is 9.2 R06 and below. 
  • Gas Shortage: Gas Shortage signifies the combined deficit between gas demand and available supply across all gas zones within the current model being executed. The X-Axis represents the timeline for Step Start Date and Y-Axis represents the Gas Shortage in terajoules (TJ). 
  • Gas Price: Gas Price indicates the aggregate of average gas prices observed across all gas nodes within every gas zone in the model being executed. The X-Axis represents the timeline for Step Start Date and Y-Axis represents the Gas Price in dollars per gigajoule ($/GJ). 


NOTE: For PLEXOS Versions 9.2 R07 and above, the Gas Supply widget is shown instead of the Gas Production widget.

Manage Long Running Simulations with Real-Time CPU, Memory, and Final Gap Widgets

The Simulation Monitoring Dashboard includes additional widgets for CPU Usage and Memory Usage to provide clearer insights into Virtual Machine (VM) Performance and the Final Gap widget for active Simulations. These widgets display the data exclusively when a Simulation is in the “In progress” state, aiding in managing long-running simulations such as for the LT plan where we have single-step simulations. After the June release, users can view these widgets for “Successful” Simulations as well. 

 

These widgets are beneficial for executing long LT models, where these long-running models sometimes hang or crash without indicating the cause of the issue. These widgets allow users to track the status of LT simulations and take corrective actions. 

 

Example of the Final Gap, CPU Usage and Memory Usage widgets on the Simulation Monitoring Dashboard: 

  • Final Gap: The Final Gap widget displays the disparity, represented in percentage, between the intended outcome and the actual result of simulations. The X-axis denotes the step start date, while the Y-axis indicates the Final Gap percentage.  
  • CPU Usage: The CPU Usage widget illustrates the utilization of the central processing unit (CPU) over time. The X-axis denotes the time in HH:MM format, while the Y-axis represents the CPU Usage percentage. By monitoring this widget, users can assess the load on the CPU during simulations, aiding in resource optimization and performance management. 
  • Memory Usage: The Memory Usage widget presents the percentage of memory (RAM) utilized over time during simulations. The X-axis represents time in HH:MM format, while the Y-axis indicates the Memory Usage percentage. This visualization enables users to track memory consumption trends, identify potential memory-related issues, and optimize resource allocation for efficient Simulation execution. 

View multiple Simulations at a time using the Simulation Dashboard

In Simulation Monitoring Dashboard, you can now open upto 5 tabs simultaneously with each tab displaying a different run’s Simulation content. The content of each dashboard displayed in a tab will remain dynamic, allowing you to view the latest update for each widget and each piece of information. If you aren’t happy with the output, you can cancel the Simulation. This feature empowers you to efficiently oversee various aspects of the Simulation without constantly switching between different views.

 

Add Models to the Simulation Group

In Simulation Dashboard, you can use the ADD TO SIMULATION GROUP button to easily add the Models to a Simulation Group and run all the simulations of that group simultaneously, eliminating manual selection for future runs. 

Follow the below steps to access the Add to Simulation Group feature:

  1. On the Simulation Dashboard, click on the ADD TO SIMULATION GROUP button available on the top right side of the Dashboard. 
  2. Select an existing Simulation Group or create a new Simulation Group as per your requirements and click on the ADD button.
