Manage Study Level Simulation Preferences

 

The Overview page allows users to set Study level Simulation Preferences where Analysts can set the default Engine Version, default Virtual Machine, and default Locale settings for the selected Study. These preferences are only applicable for the Single Run type. 

 

The Study level Simulation Preferences offer the following benefits: 

  • Reduces the need to repeatedly adjust common settings for the selected Study. 
  • Ensures uniformity in the Engine Version, Virtual Machine, and Locale settings across all Models within the Study. 
  • This enhancement simplifies workflow, saving time and effort for users.

 

Once the Analysts have set the Study level Simulation preferences on the Overview page, users can view those preferences labelled as Study Default in the Run Configuration for easier identification. 

 

For example, 

 

Refer to the scenarios below to understand how the Study level Simulation Preferences work for available Models within the Study. 

Scenario I: If the Study level Simulation Preferences are defined: 

  1. If the Model has no previous Run, the system will display the Recommendation card with the Study level Simulation Preferences to Run the Model. 


  2. If the Model has previous Runs, the system will override the Study level Simulation Preferences to display the Previous Configuration card (configuration of the previous Run) and the PLEXOS Cloud system default Recommendation card to Run the Model. 

 

Scenario II: If the Study level Simulation Preferences are not defined: 

  1. If the Model has no previous Runs and no Study level Simulation Preferences set, the system will display the PLEXOS Cloud system default Recommendation card to Run the Model. 

 

Follow the steps below to set the Simulation Preferences at the Study level: 

 

  1. Log in to your PLEXOS Cloud environment. 
  2. On the Overview page, scroll right to the top and locate the Simulation Preferences section. 


  3. Click on the CHANGE button. 

  4. Specify the desired Engine Version, Virtual Machine and Locale settings.
    • Locale Settings: Locale settings determine the language, region, date, time, and other cultural conventions within the system, ensuring alignment with user preferences and geographical norms.
       
  5. Click on the APPLY button. 


  6. You can view the applied Study level Simulation Preferences on the Study Overview page.
