Overview

The Simulations page allows users to monitor in-progress and completed Simulations. All users have access to the User Simulation Management page which allows them to manage all their simulations in one location. The User Simulation Management page shows all key details for the runs during the selected time period as well as key metrics at the top and a trend chart to show simulations over time.

 



To access the Simulations page;

 

  1. Sign In to PLEXOS Cloud using your credentials.
  2. Navigate to the Simulations page from the left-side panel.

 

NOTE: With the User Simulation Management page, users can see the details of the Simulations which are triggered only by them.

[A] Simulation Metrics

At the top of the User Simulation Management page, you will see some metrics that provide a high-level summary of the overall simulations run by the user. See the metrics details as shown below:

 


NOTE: You can use the navigation arrows to view hidden metrics cards in the list and can easily navigate to the required metric.

 
  1. Executions: Shows the number of runs triggered by the user. For a Distributed Run, the execution count would be considered as 1.
  2. Average  Run Time: Shows the average time taken per Simulation.
  3. Current Core Usage: Shows the total number of Cores currently in use to execute the ongoing simulations.
  4. Recommendations: Shows the number of recommendations provided and accepted for the selected timeframe.
    NOTE: Currently, recommendations are only provided to Successful Single Runs.
  5. Cores Used/Saved: The first number shows the number of cores that were saved because of users accepting recommendations. The second number is the total cores used to Run all Simulations.

[B] Simulation History Table

See the list of columns below to learn more about the actions you can perform within the User Simulation Management page:

 

1.  Status: The Status column shows the status of the Simulation(s) triggered by you. See the Simulation Status Description above to learn more about each status and its description.

 

2.  Study: Shows the Study Name triggered by you. Click on the Study Name and it will re-direct you to the Solutions page for the selected Study > Model.  

 

3.  Model: Shows the Model Name executed for each Study.
 
4.  Started: Shows the date and time when the Simulation was triggered.
 
5.  Elapsed Time (mins): Shows the total time, in minutes, from when the Simulation was triggered by the user to when it completed.
 
For example,

For a completed Simulation, the Elapsed Time equals the completed time minus the Simulation triggered time.

For running Simulations, the Elapsed Time equals the current time minus the Simulation triggered time. 
 
6.  Completed: Shows the date and time when the Simulation was completed.

7.  Engine Time: Shows the actual time it took to run the Simulation on the Engine. While the Elapsed Time includes the time waiting in the queue, spinning up the machine, and deploying the engine, Engine is only the actual run time of the simulation.

8.  Engine Version: Shows the type of Engine Version and revision (ex: 9.1 R01) selected for the Simulation.

9.  CPU (core): Shows the number of cores used for triggering the Simulation.

10.  Memory (GB): Shows the Memory size of the Virtual Machine (VM) in GB (Gigabytes) for the Simulation triggered.

12.  Simulation Type: Shows the Simulation Type (Single or Distributed) selected by a user for triggering the Simulation.

12.  Study Id: Shows a unique Study ID for reference.

13.  Execution Id: Shows a unique ID for the Virtual Machine selected while configuring a Run.

14.  Simulation Id: Shows the unique Simulation ID for each Run.

15.  Tags: Tags are metadata generated for each Simulation. You can attach these pre-defined Tags while en-queuing Simulations from the API.

16.  Rec. Provided: Shows the total count of the recommendations provided.

17.  Rec. Accepted: Shows the total count of the recommendations accepted.

18.  Cores Saved: Shows the Cores saved, if any, from the selected recommendation.

19.  Actions: Allows you to perform the actions below for the triggered Simulations.
 
a.  Artifacts: Opens a panel on the right-side of the screen and allows you to view the logs, download the logs, or download the solution (PLEXOS Compatible).


 
b.  Simulation Messages: Shows the details of the Cloud and Engine logs captured while the Simulation was running. These logs can be used to analyze the Run.
 
c.  Stop simulation: Click on this icon to Stop or Cancel the ongoing Simulation.
 
d. Diagnostic: Click on the Diagnostic icon to view Simulation Information that includes below information: (How to Enable Diagnostic Information?)

[B.1] Simulation Status Descriptions

The Status column shows the status of the Simulation(s) triggered by you. See the table below to learn more about each status and its description:

Status Description
Assigned to Pool
Assigned to Agent
Cancelled
Completed with Error
Successful
Failed
New
Pre-Processing
Post-Processing
Queued
In Progress

 

  •  Assigned to Pool: Among many pools available (scale sets/worker pools), a particular pool is assigned to this Simulation.
  •  Assigned to Agent: A VM within that pool has been assigned, and the simulation will be picked by the Simulation agent shortly.
  • Canceled: The Simulation that is Canceled by you or the system.
  • Completed with Error: The Simulation that completed with some error or is completed successfully, but post-processing is failed. 
  • Successful: The Simulation Run is successful, and post-processing is also successful.
  • Failed: The Simulation is failed with an error from PLEXOS Engine, and it will not generate a Solution file.
  • New: The new Simulation that you triggered.
  •  Pre-Processing: In this stage, the simulation is picked up by the VM, and the system downloads the input data required for the simulations. It will also perform licensing checks during pre-processing.
  •  Post-Processing: In this stage, the system performs ETL (Extract, Transform, Load) on the Solution file and converts it from a zip file to Parquet files.
  •  Queued: The Simulation that is queued in the system.
  • In Progress – The Simulation is running and in Progress state. 

[C] Simulation Analytics Button

The Simulation Analytics button navigates you to the Simulation Analytics page. The Simulation Analytics page serves as a centralized hub where you can monitor and track all Simulations running across your Tenant. It consolidates all essential data into a single interface, enabling you to easily monitor and track Simulation performance, resource usage, and outcomes. It eliminates the need to navigate through multiple pages to gather critical Simulation metrics. 

 

The Simulation Analytics page is accessible to all users within their Tenants. This page provides insights into key performance indicators such as core usage, credit consumption, memory efficiency, Simulation statuses, error details, and more. This empowers you to make data-driven decisions and optimize Simulation performance. 

 

[D] Toggle for Grouping the Distributed Runs

The Group by Distributed Runs option is available in the Simulation History table on the Simulation page. 


This feature improves readability and organization for Distributed Run types by allowing you to group related simulation entries in a more meaningful and user-friendly format. 


This grouping feature only applies to Distributed Run Types, such as Split and Monte Carlo runs. 


 
➤ How Grouping Works?

When the Group by Distributed Runs toggle is turned on: 
 

  1. Primary Grouping by Execution ID: Simulations are grouped by their execution ID, helping you easily identify and analyze all runs related to the same execution. 

  2.  Secondary Grouping by Simulation Type: Within each execution ID, runs are further categorized into Split or Monte Carlo, offering a cleaner, structured view. 

  3. Pinned Columns for Context: The Models and Status columns are pinned to the left, making them always visible during horizontal scrolling, ensuring clarity on what was run and its current state. 

 
This functionality streamlines the experience of reviewing distributed simulations by improving navigation, readability, and contextual awareness.
