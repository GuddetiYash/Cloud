Overview

NOTE: The Executive View is included in the Analytics bundle of PLEXOS Cloud.

 

A new feature has been introduced in PLEXOS Cloud called Executive View, which offers a user-friendly interface specifically tailored for managers and executives. This interface allows decision-makers to conveniently review reports that are shared by Analysts within their tenant.

 

The Executive View provides Analysts with a way to collaborate with the Executives within their Tenant and get their feedback. This way they can focus more on modeling and take help from Executives on important decisions. By utilizing this, Analysts can now simplify the process of manipulating the output and directly share it with decision-makers. With just a few clicks, analysts can efficiently share their work.

 

There are two distinct flows for the Executive View – Publish and View.

  • In the Publish Flow, users with an Analyst license type (Analytics Bundle) can publish their work specifically for decision-makers (Executives).

  • The View Flow presents Reports shared by Analysts, which are accessible by Executives, Reviewers, and Analysts that they can utilize to evaluate views and make informed decisions.

 

Publish Flow


Under this flow, Analysts can create Executive Views that can be shared with the users within their tenant. To create the Executive View, follow the steps mentioned below:  

 

  1. In PLEXOS Cloud, navigate to the Overview tab and click on the CREATE EXECUTIVE VIEW button.

     


  2. You will be re-directed to the Publish View wizard.

     

  3. Search for or select the Model whose Reports Analysts want to share as Views. They will only be able to see Models that have Solutions added to BI. Choose the Model that Analysts wish to share as Views, either by searching or selecting. They will have visibility only to the Models that have Solutions added to BI.

  4. Select the desired BI Report(s) you would like to share with the users.

  5. Before you share the Reports with the users, a preview is provided to understand how the Reports would be presented in the Executive View when shared as Views. 

    At this step, you can select or deselect multiple models with their own Reports.

  6. You can rearrange the order of the Reports tab, simply click on the Report tab label and drag it to rearrange the order as desired to view the Reports. Once you've completed the reordering, the tab will automatically be selected, allowing you to easily interact with your Reports.



    NOTE: The Tab Reordering functionality is available in both the Publish and View flow of the Executive View. Please note that any changes made to the Tab Reorder within these Reports will only apply to the user(s) who makes the changes. Additionally, if a user refreshes the page, logs out, or closes the browser tab, the tab reordering and closure settings will reset to their default state.

  7. In the preview, you can adjust the date range and select the type of Generator, Category, Fuel Type, and Region to view in the report. Refer to the GIF below for a graphical representation. Once checked, click on the NEXT button to proceed.

  8. Clicking on the NEXT button in the above step will prompt you to provide a custom name and description for the Report which will be shown to the users. Click on the NEXT button once done.  

    NOTE: Report Name and Description are optional, the analyst can either choose a desired name (should not exceed 60 characters limit) and write a description (1000 characters limit) or go with the system-generated name. 

     

  9. Clicking on the NEXT button will redirect you to the Review and Share View tab. Under View Details, Enter a unique name ( 60 characters limit) for View. Here View name is a mandatory input. You are also prompted to add a View Description (1000 characters limit), but this is an optional step. 

      

  10. Under Summary-Selected Reports, you can see all the selected Reports, Edit the Report Description, and Remove (Delete) unnecessary Reports.

  11. To Remove a Report, click on the kebab menu (three dots) beside the Reports and click on the Remove option.


     

  12. To modify the Report Information, click on the kebab menu next to the Report Name and Select the Edit option to get redirected to the Add Report Information page. 

    NOTE: You can either edit the Report Information on the Add Report Information tab when creating a Report.

  13. Now, once the selected Reports are confirmed click on the Select Executive dropdown to share the Report with the decision makers (Executives, Analysts, or Reviewers within the same Tenant. )

     


  14. Once the users are selected Click SHARE WITH SELECTED USERS to share the View.

  15. This action will publish the View and send the email notification to the selected users.

View Flow

 

Upon sharing the View, an email notification will be sent to the Selected users, enabling them to access the Views and interact with the Reports for analysis and decision-making by clicking on the GO TO VIEW button


Clicking on the GO TO VIEW button  will redirect them to the Executive View page. The user will be prompted to enter their credentials to login into PLEXOS Cloud. Once validated they would be able to see the Views.

 

 

The list of shared Views can be accessed on the left panel from where users can select the desired View to see its associated Reports.

The View page  will show details like View name, Description of the View, created by and the date on which it was created.

 

NOTE: Views can now be accessed by all the Analyst irrespective whether they have access to the View or not. From the View Management Grid, click the View Name to open the View in a new tab. Analysts can perform all the actions in the View such as Edit, Save As and Delete.

 

NOTE:

A maximum of 15 BI Reports will be displayed within a Single view.

 

 

The user can expand or collapse the section and view the Report in Full screen.

 

 

NOTE:

Currently the users do not have an option to delete any of the views.

 

View Management

Manage Executive Views

Executive View management is now available for users to see a detailed list of Views published within a Study . The Study Overview page contains a new Views tab which displays a complete list of all the Executive Views published for the Study . View management allows users to better manage already shared Views and provide more control to perform further actions on Views. From Actions menu, the already published Views can be deleted from the list. 

Reviewers can also access the View Management Grid, providing them visibility into existing Views. Since Reviewers have read-only privileges in PLEXOS Cloud, they do not have access to the Action menu options within the View Management Grid.

 

The Views tab includes the following details for each Executive View:

 

  • View Name: The unique name of the View.
  • View Description: The description of the View, which the Analyst entered at the time of publishing the View.
  • Created By: The user who created the View.
  • Created On: The date and time when the View were created.
  • Number of Reports: Shows a list of Reports shared within the View. Upon hovering on the number, it shows the name of the model which is the combination of the Model name and default BI Report name.
  • Actions

 

 

Actions on Views:  You can perform certain actions such as Edit, Delete and Save As (Copy) with Views.

 a) Delete:  If the View is no longer needed Analysts with access to the Study can delete the published View. To delete a View, click on the delete button icon under     Actions. 

  • When you click "DELETE," you'll receive a confirmation prompt with an alert shown below.

 

  •  Click on the DELETE button to confirm. Upon confirmation the View will be deleted.

 NOTE:  Be aware that this action is irreversible, and once deleted, users will no longer have access to the View. 

 

b) Edit:

 

The "Edit" action opens a wizard for the selected view. With Edit, users can update the following fields: Report Name, Report Description, View Name, View Description and update the list of shared users. It is important to note that, under specific circumstances, such as when a user unpublishes a Solution used in a View or deletes an original BI Report, some of the Reports may become unavailable within that View. When this scenario arises, a popup/dialogue will appear informing the user about the unavailability of the reports, then they can proceed with the Edit action by clicking CONTINUE WITH EDIT.

 

b) Save As (Copy):

 

When clicked, the action "Save As (Copy)" launches the wizard with the latest published Solutions for the Study in the existing View template.  A new View will be created with the name 'copy of previous name.' (This name can be changed.)

This new View includes all previously selected reports and details from the source View, along with the list of pre-populated users. Analysts  have the flexibility to edit any of the pre-populated fields according to their requirements.  

 

 

NOTE:


The View Management grid has been updated to sort entries by “Created On” field, ensuring that the most recently shared View is displayed at the top of the list.
Please note that the 'Edit' and 'Save As' options are exclusively available for new Views only. These options will be disabled for the Views created before this release. 

Executive View UI Enhancements

 

  • Bookmark a View URL: With this feature, you can now bookmark a View URL in your browser. When you revisit the bookmarked URL, it will automatically display the most recently published View as the default View .

  • Created On: Now the “Created on” Date of a published View will show the Timestamp along with the Date when the View was created.

  • Enhanced Solution Visibility for Analysts: Analysts can now see the Solution added to BI for the selected Model and Report during the publishing process.
    This ensures that users are aware of the solution being used to load the report and minimizes the risk of sharing the wrong report. Currently, users can only see the solution name used; they cannot select the desired Solution.

  • Show or Hide the Analyst’s name that Created the Executive View:  

    Analysts now have the option to hide the 'View Created By' field from users accessing the View, allowing users to concentrate more on the content rather than additional details. While these details will still be recorded and displayed in the View Management Grid, by default they will remain hidden from users accessing the View from the simplified Executive View interface.
