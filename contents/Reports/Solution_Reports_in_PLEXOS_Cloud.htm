Solution Reports in PLEXOS Cloud

Users can now create and customize their own reports in PLEXOS Cloud to filter meaningful insights from their solution data. The Solution Reports on PLEXOS Cloud run on configurations that are provided as part of a valid JSON file.

NOTE: This feature is applicable only for Analytics Tier customers.

Navigating and Viewing Reports

[A] Navigating to Reports 

  1. To navigate to reports, go to a successful solution and click on the Reports icon.
  2. From there, the Reports panel should be visible to view EE Reports and My Reports under the dropdowns.


EE Reports contains a list of pre-defined Solution output reports. These Reports are built to address common analysis scenarios and are useful as is or as a starting point to build your own Reports. My Reports contains a list of custom reports from the configuration files that this user and others in their organization have uploaded.

 

[B] Viewing a Report 

Select any EE Report or My Report by clicking the checkbox beside it. The Report will then be rendered and displayed on the righthand side as tabs.
 

 

 NOTE: A maximum of 5 Reports can be selected at a time. 
 

[C] Drill Into Functionality

Users can use the Drill Into functionality to gain further insight into Solution Report data. If you are analyzing Generation for Monthly or Daily data and want a better understanding of when the Generation is occurring on a more granular basis, Drill Into can fulfill this purpose.

  1. After selecting a Report (from EE Reports or My Reports) from the Reports tab, right-click on a cell and Drill Into will appear as an option. 
  2. If you click Interval, you go can go down one level and view the daily as well as monthly data. If you click on Year, you can go up one level and view data for 2020. 
  3. After clicking Year, you see the data for 2023 summarized in the table. You also have an additional option: Reset, which will take you to the initial data view for the Report.


    For example, the results below if you click Day to view daily data for January, It will show all the results for January.

Searching Reports

Reports can be easily searched by clicking the search icon beside Reports. This search bar will filter through all EE Reports and My Reports available based on the search criteria. 

 

 

Viewing Charts and Exporting Data

  1. To export the report data, you can click on the cloud icon in the top right-hand corner of the screen. All columns and values in the report will be exported, even if the columns are collapsed. You can choose to export the data in either .csv or .xslx format. 
  2. You can chart your data as shown below. Select the desired data range by clicking cells your clicking and dragging to select a region, then right click and select Chart Range. Multiple chart types will appear. From here, you can select the chart format and display your data. 
  3. If you navigate to the right-hand corner of the chart window, you can select one of three options. 
  4. If you click on the three parallel lines , you will be presented with the following tabs. You can modify the chart type under Settings, set the data range under Data, and change chart formatting under Format. You can also use the arrows on the bottom right of the window to change the color scheme. 
  5. The chain icon will link and unlink the chart to the grid. If linked, the highlighted report data will sync with the charted data. The download icon allows you to export the chart in PNG format. This is useful for sharing the chart or embedding it in a presentation or document. 

Adding and Modifying Reports 

Users have three different options for creating a Report. The first option is to export an existing Report and modify it. The second option is to start from scratch. The third is to Save as Report from the Output Data Grid. The first and third options are recommended since they present the JSON schema for you and are straight forward. 

 

[A] Creating a New Report 

  1. To create a new Report, click on the + sign next to the Reports header as shown below. 
  2. A dialog box will appear where you can drag and drop or browse to upload your configuration JSON file. 

 

 NOTE: The number of columns per Report is limited to 50. You also cannot upload a report with the same name as an existing report.
 

[B] Modifying an Existing Report 

To modify an existing EE Report, right click on it and select either Export or Save As.
 

 


Export exports the Report to a JSON file where it can be modified. You can then upload the JSON file, which creates a new Report under My Reports. Save As immediately creates a new Report under My Reports in PLEXOS Cloud.
 

Reports under EE Reports cannot be edited. They can only be duplicated with Save As, or exported and reimported as user reports to be edited. Only reports in the My Reports section can be edited. To edit a Report, right click the report and select the Edit option to edit the file. 

 

 

 

This allows you to edit the JSON file in PLEXOS Cloud itself. After updating the Report, you can either Update the Report or Save As to generate a new Report. 

 

 
 

[C] Save as Report from Data Grid

You can save data from the Solutions Data Grid as a Solutions Report through the Save as Report function. Navigate to the Data Grid and, query the desired data from your Solution, then click Apply. In the top right-hand corner of your window, you can select the button for Save as Report.
 

 NOTE: This functionality is currently available for only Daily, Weekly, Monthly, Quarterly, and Yearly reported data.
 

 

After clicking Save as Report, the window shown below should appear. Here, you can enter the Report Name and Group Name. The Group Name is the group under which the selected properties will be added. After entering the values, click Save. 
 

 NOTE: The Report name must be unique. The Solution Reports support up to 50 columns of data, which means that the combination of Objects and Properties cannot be more than 50 in the Data Grid. 
 

 


After clicking Save, the following popup will appear. Click on View Report or the Reports icon to be directed to your Report.

Understanding the Configuration JSON File

Users can supply a custom-defined JSON configuration file via the configurator with the following sample schema. A sample configuration file is attached along with this article. Read more to understand the attributes structure of the file. 

A free third-party tool can be used to edit the JSON file to maintain neater and more consistent formatting. 

[A] Components of the JSON file 

The following fields explain the JSON schema field by field:
 

1. reportName: The name you choose for the report.
 

2. filters: This is where you set the reporting period type and simulation phase.
 

a. periodTypeId: Day: 1, Week: 2, 3: Month, 4: Year, Quarter: 7.

b. phaseId: 1: LT, 2: PASA, 3: MT, 4: ST. 
 

3. propertyGroups: All the reported groupNames that are rendered in PLEXOS Cloud as expandable and collapsible column groups. 
 

a. groupName: The name of the column group. This category heading is what appears above the group of child property names in PLEXOS Cloud, and can optionally be expanded and collapsed. 

b. phaseId: (optional) Allows you to define phase for the Group that is different from the Report level phase.

c. children: Child properties under the column group (groupName). Each child takes information from your Solution data, and is described with the information below:
 

i. name (string): The property name, which should not exceed 50 characters. This is the column name that appears below the groupName. 

ii. hidden (boolean): (optional) A field to define if the property is shown when column group is not expanded. If set to True, the property will be hidden unless the groupName is expanded. If set to False, the property will be displayed when the groupName is not expanded. If not defined, the value will default to False. 

iii. UoM (string): (optional) A field to define Unit of Measure (UoM) of the property. If not defined, the UoM is the PLEXOS default for the output property. 

iv. filters: (object, each filter property is an array of strings): (optional) A field to define filters. Users can supply category filters or object/unit level filters as arrays. Each array can contain up to 10 filter items.
 

1. Object filter: The user can list any object name in the system, such as generator, fuel, region, etc. 

                        "filters": { 

                            "objects": [ 

                                "East", 

                                "North" 

                            ], 

                         }

 

2. Category filter: The user needs to follow a specific pattern in defining the categories (ClassName_CategoryName) 

                        "filters": { 

                            "categories": [ 

                                "Generator_Coal", 

                                "Generator_Oil" 

                            ] 

                        }, 
 

v. phaseId: (optional) A field to define phase for the property that is different from the Report or Group level phase (1: LT, 2: PASA, 3: MT, 4: ST). 

vi. formula (string): If this value requires computation using multiple properties, such formula can be provided here. To supply a formula the user needs to follow a specific pattern (ChildClassName_CollectionName_PropertyName) to access the property from solution file, combined with arithmetic operators to return computed values reported in the model solution. Currently, addition, subtraction, multiplication, division, and exponentiation are supported. 

vii. id (string): (optional) A field to define unique id used to calculate derived children. Derived children are values that are computed in the UI based on the values of child properties. The child id can be defined freely by the user and has nothing to do with the solution data. It is only referred to in the browser in order to compute derived child properties. Below are a few rules that must be followed when supplying values for the column id:
 

1. id must be unique 

2. id should start with c. For example, "c1", "c2". 
 

d. derivedChildren: Value is computed in the browser, using the child properties’ id as reference and value is computed in UI. 
 

i. There must be less than 50 columns total of child and derived child properties.

User Defined Property Calculation

Custom column calculation is a basic mathematical operation such as multiplication, division, addition, and subtraction that you can perform on the generated Solution Reports under the Solutions page. This article helps you understand how to perform these mathematical calculations on Solution Reports and view them in BI Analytics to see the custom-generated values. 

 

You can perform the following custom mathematical calculations in the schema version of the generated Solution Reports file: 

  • Multiplication by float 
  • Division by float 
  • Addition by float 
  • Subtract by float 

 

For example, in the screenshot below, the Build Cost is defined with the formula:
"Generator_Generators_Annualized Build Cost + Battery_Batteries_Annualized Build Cost".

 

If you wish to convert this cost into Gigawatt (GW) units or perform any other custom calculation, you can edit the existing formula to include simple mathematical operations such as multiplication, division, addition, or subtraction. In this scenario, to convert Build Cost to GW units, you can add the division method to obtain the desired value:  
"Generator_Generators_Annualized Build Cost + Battery_Batteries_Annualized Build Cost / 1000".

Aggregation and Transpose


Aggregation

The Aggregation method handles to fulfill the requirement of getting the values for Sum, Average, Min, and Max.
In this method, we have provided the functionality to edit the “Json” file of the report. In that file, you can add the required Aggregation function (SUM, AVG, MIN, and MAX).

The reporting for aggregation needs to be flexible enough to handle different aggregation methods. For example, Generation output you would want to get the total of all aggregated generators while Market Price you would want to get the average of all aggregated regions. 
The Aggregation method allows you to fulfill these requirements for getting the values for Sum, Average, Min, and Max in your Reports.


Follow the steps below to use the Aggregation method in your reports:

1. Open any Successful Solution
2. Go to Reports
3. Check any report to see the data. For example – the IRP report
 


 
  4. Make a copy of it, if you do not want to alter the data of your original Report.
  5. Right-click on the selected Report and click Save As
 
 

 

6. Enter a unique name for this Report and click on Save As once again.
 

 

7. The copy of this Report will be saved under Custom Reports (My Reports)

Auto Split Feature

Auto-Split feature allows to report each individual category or object in the system by just enabling a flag. 

You no longer needed to hardcode object names in reports. This enables Solution reports to be Study agnostic.

In Solutions Reports, alongside “reportByCategory” and “reportByObjects”, you can now retrieve data by Sample Ids by using the flag "reportBySamples". There is a new flag under "auto populate children" called "reportBySamples". At once only one of them can be true for one property, others will be false.

 

For instance, consider the following configuration:

The Auto sample generation flag in Solution Reports as shown below

{

  "Formula": "Generator_Generations_Generation",

   "reportBySamples": true

}

  • "Formula" corresponds to the formula or data source you want to analyze.
  • "reportBySamples": “true”, is the flag that enables the auto-generation of sample data.

In the above example, “reportBySample” is set to true then for Monte Carlo studies with 20 samples, in this case, there will be 20 columns generated and with each column will fetch report data against that sample as shown in the below image:

Date Range Filter 

Date Range Filter will improve your data analysis and reporting with Solution Reports. This helps in customizing your Reports by specifying start and end dates in the YYYY-MM-DDT00:00:00 format directly in the Report's JSON file. It is a simple and flexible way to customize your reports. The advantage of this feature is it allows you to filter out any data that is not relevant to the time period of interest.

 

For example, if you wish to retrieve the data for only January 2023, you can set the date range as:


“date range”: [“2023-01-01T00:00:00”, “2023-01-31T23:59:59”]
