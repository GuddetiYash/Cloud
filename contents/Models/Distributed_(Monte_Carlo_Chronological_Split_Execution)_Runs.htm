Distributed Runs

A Distributed Run enables you to control and maximize the computing resources for a model run. You can select the Virtual Machines Type and the Number of VM to allocate to your run by specifying them in the drop-downs below.

 

Currently, PLEXOS Cloud supports below mentioned Distribution Methods for Distribution Run type. 

  1. Chronological Split Execution
  2. Monte Carlo (Auto-sample)
  3. Monte Carlo
  4. Parallel Models
  5. Monte Carlo Outage
  6. Planning

Follow the steps below to access the above-mentioned Distribution Methods: 

 

  1. Navigate to the Models page.

  2. Select the Model or multiple Models you wish to run.   After selecting the Model(s), the Run Models panel will appear in the top-right corner of the screen. 

  3. Drag the panel to the left to view the full Run Configuration Panel.


      NOTE: When you select the Models to run, the Run Configuration Panel provides key information, such as available Credits, Concurrent Cores, the selected Model name, and Configuration cards like Previous Run Details, Previous Configuration, Recommendations, and Study Recommendations. Selecting one of these cards and clicking Run can trigger the Model Run instantly.

  4. On the Run Configuration Panel, select the desired configuration card that you wish to customize.

  5. Click Customize to enter a new window where you can adjust the selected configuration.

  6. In the Run Type section, you can view and select a Distribute Run Method of your choice.

1. Chronological Split Execution

Chronological Split Execution allows you to divide the chronological horizon to run the ST phase divisions in individual VMs, and as a result reduce run times and improve performance.

 

This type of Distributed Run can be enabled in PLEXOS Cloud as long as you have an ST Schedule object enabled on your Model. During the simulation, the distribution method runs individual splits on separate VMs, and the final solution files are stitched together to view from the Parent Simulation.

 

Understanding Chronological Split Execution simulation structure and solution stitching


Parent Simulation

 

    • Runs MT/LT phases
    • Plexos splits ST phases into new simulations which are enqueued as individual simulations.
      • Each ST will possibly run on its own VM, but will have access to and may use the LT/MT from the parent so it doesn't re-run long running phases
    • Raw ST Child Simulation solution data will be available to download as part of the Parent Simulation. Note: This solution data will not contain any reportable information.
    • PLEXOS stitches the ST solution data after all are done 
      • Final Stitched solution data will be available from the Parent Simulation
        • Hybrid/Parquet etl process only happens on stitched solution data
    • In the case of 1 or more children failing to complete successfully, stitching will still be attempted and may still produce partial output.


Child Simulation

    • Typically runs the individual ST splits
    • Will not directly have solution data of its own
      • Log data, PLEXOS logs, raw solution data will be associated with the parent simulation
    • Has access too and may use any data available from the parent MT/LT simulation
      • Additional optimizations are in the works to help reduce time and processing of ST Child Simulations.

2. Monte Carlo (Auto - Sample)

 

Monte Carlo (Auto-sample) allows you to run endogenous samples on multiple machines, then stitch your solutions together. With Monte Carlo (Auto-Sample), the total number of samples run will equal the Object Sample Count from the Stochastic Object multiplied by the number of requested instances. This means that if the model defines a Stochastic Object with 30 samples, every simulation instance will run 30 samples using different random number seeds.

 

For example, if 30 samples are defined in the model, and a user requests 10 VM instances, this will produce a total of 300 samples if all simulations complete successfully.

 

Understanding Monte Carlo Sampling simulation structure and solution stitching

In PLEXOS Cloud, the Parent Simulation contains the information that is ultimately passed on to the Child Simulations. Read more to understand the difference between parent and child simulations, and how solution stitching is done for Monte Carlo Sampling.

Parent Simulation

    • The initial simulation splits the sampling but does not run a simulation using the PLEXOS engine.
    • Splits into multiple simulations depending on model configuration and requested sample count (divisions)
    • Produces generated input data, which is uploaded and associated with the Parent Simulation
      • No other solution data is produced by this simulation
    • Sample divisions are determined by PLEXOS, not PLEXOS Cloud
    • Does not wait for children to finish, will exit when done
      • Short running process, same VM will be used to run Child Simulations after they're enqueued
      • No automatic stitching

Child Simulation

    • Uses the generated input data from the parent as its input
    • Produces solution data per simulation
      • Parquet, Hybrid, Raw data available for each Child Simulation
    • No automatic stitching

Solution Stitching

    • Explicit actions from the user are required to run stitching. It does not run automatically.
    • The stitched solutions selected by user. Note: User defined order for stitching can be set through the PLEXOS Cloud API.
    • Produces Parquet and Hybrid stitched data.
    • Stitching is only available on Hybrid and Parquet based solution data.

3. Monte Carlo

This distribution method is used for user-defined samples. The total number of samples run is equal tot the Object Sample Count from the Stochastic Object. This means that if the model defines a Stochastic Object with 30 samples, any combination of how samples are split among the VMs will result in 30 total samples.

 

For example, if 30 samples are defined in the model and a user requests 3 VM instances, each VM will run only 10 samples, with the total being 30. 
 

NOTE: For Monte Carlo (Auto-sample) and Monte Carlo function, a Stochastic object must be enabled in the model.

 

Users can stitch the Solutions in two ways: 

 

  1. Before the Simulation Run, on the Run Configuration panel, users can click the checkbox of Stitch Solutions once Simulation completes to stitch the generated solutions.  
  2. Alternatively, after the Simulation Run, users can navigate to the Simulations page, locate the respective model, and use the Stitch Solutions option from the Kabab Menu.

4. Parallel Models

The parallel model distribution method in PLEXOS Cloud's Distributed run type is a technique that allows multiple models to be run simultaneously. This is achieved by selecting multiple models using the Ctrl + Click or Shift + Click method and then choosing a single virtual machine (VM) type to run these models on.

 

When the models are selected and the VM type is chosen, each model is assigned to its own VM and is run in parallel, thereby reducing the time taken to complete the simulation. This is particularly useful when dealing with large and complex models that would otherwise require a significant amount of time to run.

 

Using this method not only saves clicks by allowing multiple models to be selected and run simultaneously but also ensures that the execution is completed in a timely manner. It is an efficient and effective way to distribute the computational workload across multiple processors, making it easier to perform simulations and gain insights.

5.  Monte Carlo Outage

The Monte Carlo Outage Distribution Method operates on Outage sample draws and enables users to split their Simulation based on Outage sample draws defined in the Stochastic Object of Model. 

 

This method is beneficial when users wish to divide the Simulation into outage samples, executing each outage sample concurrently on separate Virtual Machines (VMs), rather than running all outage samples on a single VM. Users can select the Monte Carlo Outage from the Distribution Method dropdown and specify the required Number of Divisions to Run the Monte Carlo Outage for their respective Simulations. 

 

For example, if the Model is defined with 30 outage sample draws and a user requests 3 for the Number of Divisions, then each Virtual Machine will Run only 10 Outage samples, with the total being 30  
Outage samples. 

 

NOTE: To use the Monte Carlo Outage method, ensure that you have a Stochastic Object enabled in your model.

 

Users can stitch the Solutions in two ways: 

 

  1. Before the Simulation Run, on the Run Configuration panel, users can click the checkbox of Stitch Solutions once Simulation completes to stitch the generated solutions.  
  2. Alternatively, after the Simulation Run, users can navigate to the Simulations page, locate the respective model, and use the Stitch Solutions option from the Kabab Menu.

6. Planning

The Planning Distribution Method is beneficial when users want to split the Planning Horizon of the Model with the Year Horizon Units.

 

Users can  select the Planning method from the Distribution Method dropdown and specify the required Number of Divisions and Horizon Units such as Year to Run the Planning method for their respective Simulations.    

 

For example, if the Planning Horizon for a Model is set to 10 years, and a user requests 2 Divisions with Horizon Units set to Years, then each Virtual Machine will Run for 5 years. 

NOTE:  The Planning Distribution Method now only supports the Year Horizon Unit, and until other intervals are supported, the   Day, Month, and Week Horizon Units will not be available.
