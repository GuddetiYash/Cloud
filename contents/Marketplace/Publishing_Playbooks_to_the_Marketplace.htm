Overview

A Playbook is a use case-driven dataset or Study that includes content to accelerate and simplify the Modeling experience. It includes pre-configured Models, Solutions, Solution Reports, BI Reports, and other supplementary content, providing a ready-to-use package for in-depth analysis and decision-making.

The Marketplace provides playbooks across various commodities, including:

  • Electrical
  • Gas
  • Universal
  • Water

 

You can create a Playbook using one of the following methods:

 

  1. Create New Submission: Allows you to create a new Playbook from scratch to meet your analysis needs.

  2. Add to Existing Submission: Allows you to create and publish new versions of existing Playbooks. This functionality is particularly useful when updates or modifications are required to existing Playbooks. For instance, you might need to enhance a Playbook by adding new content, revising data, or refining existing information. Using this feature, you can efficiently manage multiple versions of Playbooks while maintaining a consistent structure and history.

 

Both methods involve selecting the appropriate Study Version, Executive Views, Solutions, Reports, and other relevant components to ensure an effective dataset for in-depth analysis and decision-making.

Things to Consider Before Creating a Playbook

A Playbook is a use case-driven dataset or Study that provides a ready-to-use package for in-depth analysis and decision-making. While creating a Playbook, you can select various components such as Executive Views, Solutions, Map Layouts, Grid Layouts, Solution Reports, and BI Reports.

 

Each of these components plays a crucial role in enhancing the Playbook's effectiveness. Selecting the right elements ensures your Playbook delivers maximum value and usability.

 

The following points explain the value addition of each component and why you should consider including them in your Playbook:

 

  1. Select an Appropriate Study Version: The Study Version determines which Executive Views, Solutions, and Reports are available for selection in your Playbook. Each version contains changes made to your Model at a specific point in time, ensuring consistency in data analysis. Selecting the right version helps maintain accuracy, track modifications, and align Simulations with the latest updates. By default, the system will choose the latest Study Version while creating a Playbook.

    You can manage all your Model versions from the Versions tab under the Data page on PLEXOS Cloud.

 

➤ Why is this important?


  • Ensures data consistency: The selected Study Version defines the dataset and results used in your Playbook, avoiding inconsistencies across different versions.

  • Maintains alignment with Model evolution: If multiple versions exist, selecting the right one helps retain the intended assumptions, configurations, and simulation outcomes.

  • Facilitates accurate comparisons: Using different Study Versions enables historical tracking, helping you compare changes in model assumptions, inputs, and results over time.

 

  1. Executive Views: An Executive View is a collection of Business Intelligence (BI) reports that provide high-level summaries of key results. These views allow executives and decision-makers to review insights in a structured and concise manner.

 

➤ Why include it?


  • Summarized insights for executives: Instead of navigating multiple Reports, executives can access a pre-curated set of insights for quick decision-making.

  • Facilitates collaboration and feedback: Executive Views allow stakeholders to review, comment, and provide input on key findings.

  • Streamlined Reporting:  Helps prioritize Key Performance Indicators (KPIs) and track critical metrics efficiently.

 

  1. Solutions: A Solution is a fully processed Simulation output that captures the results of a Single Model Run. It includes key performance metrics, operational insights, and system-wide data for evaluation.

 

➤ Why include it?


  • Captures full Simulation results: Ensures that the Playbook retains all outputs from a Single Simulation run, providing a comprehensive dataset for analysis.

  • Reference for comparisons: Allows you to compare different Modeling runs over time to track performance changes and these Solutions serve as a reference point to investigate variations in results. 

 

  1. Map Layouts: A Map Layout represents a saved visualization of the Model's geographical data, including selected Properties, Objects, and any other selected parameters. It provides an interactive way to analyze spatial relationships within the system.

 

➤ Why include it?


  • Preserves custom geographic configurations: Saves predefined settings for quick access to essential geospatial data that includes the selected Properties, Objects, and any other selected parameters.

  • Saves time in repeated analysis: Instead of manually configuring Maps each time, predefined layouts streamline the process.

 

  1. Grid Layouts: A Grid Layout is a customized tabular view that stores selected Properties, Objects, Filters, Pivot Views, and configurations for organizing Model results. It helps in structuring large datasets for better readability and focused analysis.

 

➤ Why include it?


  • Reduces redundant effort: Instead of manually adjusting filters each time, saved Grid Layouts automate data presentation.

  • Custom filtering and sorting: Helps users extract targeted insights based on selected properties, phases, and periods.

 

  1. Solution Reports: A Solution Report is a structured report that focuses on specific analysis scenarios. It includes custom columns, formulas, and calculations, making it an essential tool for evaluating Model outputs.

 

➤ Why include it?


  • Standardized and structured reporting: Presents organized results with calculated insights, making data easier to interpret.

  • Pre-built insights for common scenarios: Helps users quickly analyze system behavior using preconfigured reports.

  • Customization for advanced analysis: Users can modify reports further to suit specific analytical needs.

 

  1. BI Reports: A Business Intelligence (BI) Report is an interactive, data-driven visualization that transforms complex datasets into meaningful insights. It provides dashboards, charts, and tables to help users interpret trends, monitor KPIs, and compare scenarios.

 

➤ Why include it?


  • Transforms complex data into visual insights: Uses graphical representations to improve decision-making.

  • Facilitates trend analysis and KPI tracking: Helps users monitor key performance metrics and identify patterns.

  • Enhances decision-making with structured analytics: Allows users to compare scenarios and analyze system performance efficiently.

 

  1. Default Content in Playbook:: When creating a Playbook, the system automatically includes the available Models along with their Scenarios and input data from your selected Study Version.

    Even if you do not select Executive Views, Solutions, or other Reports while creating a Playbook, the Playbook will still contain the Models associated with your selected Study Version. This ensures that every Playbook has a foundational dataset, allowing you to build upon it by adding relevant reports, layouts, and analyses as needed.

Publishing Playbooks to the Marketplace

Refer to the following step-by-step instructions on creating and publishing Playbooks to the Marketplace: 

★ [Step 1]: Create a New Submission

  1. Sign In to PLEXOS Cloud using your credentials.

  2. On the Overview page, click CREATE PLAYBOOK and select + Create New Submission. 

 

★ [Step 2]: Select an Appropriate Study Version

  1. In the Playbook Wizard, select an appropriate Study Version from where you wish to add your  content.   



    NOTE: The system displays Executive Views, Solutions, and Reports corresponding to the selected Study Version. You can switch between versions to view their respective contents.   However, a Playbook can only contain content from a single Study Version. You cannot create a Playbook by selecting content from multiple versions.

★ [Step 3]: Choose Content to Add  

  1.   Select the content you wish to add to your Playbook, such as Executive Views, Solutions, Map Layouts, Grid Layouts, Solution Reports and BI Reports.



    NOTE: When creating a Playbook, the system automatically includes the available Models along with their Scenarios and input data from your selected Study Version.

  2.  Click Next.


★ [Step 4]: Submit / Publish the Playbook

If your Role is User (without any publishing permissions):

 

  1. On the Submitter panel, provide the following information:

    • Select the Content Visibility that determines who can view your submission (e.g., Private, or Tenant).

    • Enter the Playbook Title, Description, and Publisher Name.

    • Enter Version details and Category type (Gas, Electric, Water, Universal).

    • Upload images to support your Playbook (if any).

    • Upload Release Notes in .pdf format to state the updates.

    • Upload Miscellaneous Files (if any) in .xlsx, .xls, or .csv format.

 

  1. Click SUBMIT.

  2. The Playbook will be submitted to the Content Moderator for review.   The Moderator evaluates the content for accuracy, completeness, and compliance. If necessary, the Moderator may edit details like playbook names, descriptions, and SKUs. Once the review is complete, the Moderator approves and publishes the content to the Marketplace.

 

If Your Role is Content Publisher: 

 

  1. On the Publisher panel, provide the following information:

    • Select the Content Visibility that determines who can view your submission (e.g., Private, Tenant, or Public).

    • Enter Playbook Title, Description and Publisher Name.

    • Enter Version details and Category type (Gas, Electric, Water, Universal).  

    • Upload Playbook-related Images to support your Playbook (If any).

    • Upload Release Notes in .pdf format to state the updates.

    • Upload Miscellaneous Files (if any) in .xlsx, .xls, or .csv format.


  2. Click PUBLISH.  

  3. The Playbook will be published on the Marketplace.

  4. Go to the Marketplace to preview the published content. The content published at the Tenant level is exclusively visible to users within that Tenant on the Marketplace.

    To quickly filter and view content based on its accessibility level, use the Visibility Status dropdown in the Marketplace.
