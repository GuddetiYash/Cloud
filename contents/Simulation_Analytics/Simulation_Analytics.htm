Overview

The Simulation Analytics page serves as a centralized hub where you can monitor and track all Simulations running across your Tenant. It consolidates all essential data into a single interface, enabling you to easily monitor and track Simulation performance, resource usage, and outcomes. It eliminates the need to navigate through multiple pages to gather critical Simulation metrics. 

 

The Simulation Analytics page is accessible to all users within their Tenants. This page provides insights into key performance indicators such as core usage, credit consumption, memory efficiency, Simulation statuses, error details, and more. This empowers you to make data-driven decisions and optimize Simulation performance. 

 

Steps to Access the Simulation Analytics Page [For All Users]

 

  1. Sign in to PLEXOS Cloud using your credentials.
  2. In the left navigation panel, click Simulation Analytics.

Key Benefits & Features 

The Simulation Analytics page offers the following benefits that improve visibility into simulation performance: 

Centralized Access: All essential Simulation metrics are available in one place, reducing the time spent navigating between different interfaces. This central access point enhances productivity by making it easier to monitor multiple Simulations. 
 

  1. Real-Time Monitoring: You can track active and completed Simulations with real-time updates, helping to identify performance issues or successes immediately.  

  2. Enhanced Collaboration: As the page is accessible to all users, it promotes transparency, improving teamwork by making essential data visible to everyone. 

  3. Data-Driven Decisions: The metrics provided on the Simulation Analytics page help you make informed decisions regarding resource allocation and Simulation management, avoiding unnecessary Credit consumption and inefficiencies. 

  4. Enhanced Sorting Options: Flexible sorting and filtering options allow you to view specific Simulation data based on your needs (e.g., by Study type, time range, or triggered simulations). This allows you to focus on relevant Simulations and optimize their analysis. 

 

The Simulation Analytics page provides the key metrics, such as Core Efficiency, Memory Efficiency, Credit Usage by Study, Credit Usage by Users, details of the Total Simulations Triggered, Failed, Successful, Average Run Time, Active Users, and more.

Sorting Options 

The sorting options on the Simulation Analytics page allow you to customize the data based on your specific needs. You can sort the Simulation data by Studies, Timeline, or specific users. 

 
 

[A] Sorting by Studies  

  • All Studies: Displays Simulations from all available Studies in your Tenant, providing a comprehensive overview of all Simulation activities. This helps you monitor Simulations across the entire Tenant. 

  • Shared with Me: Displays Simulations from Studies that have been shared with you, allowing you to track Simulations relevant to your collaborative work. 

  • My Studies: Displays Simulations only from Studies that you have created or triggered. This option helps you focus on Simulations where you are directly involved, simplifying the tracking of your work. 

[B] Timeline Sort Options

  • You can sort the Simulation Analytics dashboard by Last 30 Days, Last 7 Days, Last 90 Days, or Show All. This allows you to focus on recent activity or get a long-term view of simulation trends, depending on your needs. 

[C] Triggered By

  • This option allows you to sort the simulation data by All Users or a Specific User. It helps you track simulations triggered by the entire team or focus on the activity of individual users. 

All Simulations 

 
The All Simulations section provides a comprehensive list of all Simulations for the selected timeline. It offers critical details in a tabular format, such as Model name, Status, Study name, Credits used, Engine Version details, and more. This helps you track ongoing Simulations and analyze their performance. 

 

Additionally, this section allows you to view Diagnostic details, Simulation Messages, and Log Files to help analyze Simulation Errors and take prompt action. 
 

[A] Column Details 

This section contains the following columns that help you gather the details of your Simulations.

 

 

  • Status: Shows the current status of the Simulation(s) triggered by you. 

  • Model: Shows the Model Name executed for each Study. 

  • Study: Displays the name of the Study that the Model or Simulation belongs to. 

  • Study ID: Shows a unique Study ID for reference. 

  • Created: Displays the creation date of the Model. 

  • Started: Shows the date and time when the Simulation was triggered. 

  • Elapsed Time (mins): Shows the total time, in minutes, from when the Simulation was triggered by the user to when it completed. For a completed Simulation, the Elapsed Time equals the completed time minus the Simulation-triggered time. For running Simulations, the Elapsed Time equals the current time minus the Simulation-triggered time. 

  • User Name: Displays the name of the user who triggered the Simulation. 

  • Completed: Shows the date and time when the Simulation was completed. 

  • Engine Time: Shows the actual time it took to run the Simulation on the Engine. While the Elapsed Time includes the time waiting in the queue, spinning up the machine, and deploying the engine, Engine is only the actual Run time of the Simulation. 

  • Engine Version: Shows the type of Engine Version and revision (e.g., 9.1 R01) selected for the Simulation. 

  • CPU (core): Shows the number of cores used for triggering the Simulation. 

  • Memory (GB): Shows the Memory size of the Virtual Machine (VM) in GB (Gigabytes) for the Simulation triggered. 

  • Simulation Type: Shows the Simulation Type (Single or Distributed) selected by a user for triggering the Simulation. 

  • Simulation ID: Displays the unique identifier for the Simulation. 

  • Tags: Shows the metadata generated for each Simulation. You can attach these pre-defined Tags while enqueuing Simulations from the API. 

  • Rec Provided: Shows the total count of the recommendations provided. 

  • Rec Accepted: Shows the total count of the recommendations accepted. 

  • Cores Saved: Shows the Cores saved, if any, from the selected recommendation. 

  • Custom Arguments: Shows any custom arguments passed during the Simulation using the PLEXOS Command Launcher. 

[B] Kebab Menu 

The Kebab Menu (three-dot option) provides the following actions based on the selected Simulation: 
 

  • Add to Simulation Group: A Simulation Group is a set of Simulations that runs together in a batch. This option allows you to add the selected Simulation to the new Simulation Group, or you can create a new Simulation Group and add it to that group.  For more information, refer to the detailed Simulation Groups help article.  

  • Simulation Messages: This option allows you to view Simulation Messages that contain details of the Cloud and Engine logs captured during the Simulation. These logs are useful for analyzing Simulation performance and diagnosing issues. 

 

  • Download Artifacts: This option opens a panel on the right side of the screen, allowing you to view the logs, download the logs, or download the solution (PLEXOS Compatible). These logs are useful for analyzing Simulation performance and diagnosing issues. 

  • Diagnostic: This option allows you to view the Solution Information that contains the Study ID, Simulation ID, Execution ID, Changeset ID, and other relevant Solution details.

    To view this option, ensure that you have enabled Diagnostic from the Preferences page. If not, navigate to Profile > Preferences > Diagnostic and enable it.

  • Access to Study: This option allows you to request access to the Study. 

  •  

NOTE: If you do not have access to the Study, the Kebab Menu will only display the Simulation Messages and Access to Study options. Other actions, such as Add to Simulation Group, Download Artifacts, and Diagnostic, will not be available. 
 

[C] Export Option 

The Export option, located on the top right of the All Simulations table, allows you to export Simulation details in CSV or Excel format for further analysis. 
 

[D] Filter Option 

The Filter option allows you to narrow down the Simulation data based on specific criteria. This feature is particularly useful when you are managing large data and want to focus on certain types of Simulations or results without scrolling through the entire list. 

 

  • To apply filters, hover over any column and click on the filter bar to sort the data according to your needs.  

  • To clear all filters, click the CLEAR FILTERS button located at the top right of the All Simulations table.

General Stats 

The General Stats section provides key performance metrics that offer valuable insights into the overall activity and efficiency of your Simulations within the selected timeline. This section includes metrics such as Average Run Time, Active Users, Top Engine Version, Active Studies, and Most Active Study. These metrics help you analyze the resource utilization and performance trends within your Tenant. 

 
 
Refer to the following Key Metrics in General Stats: 

 

  • Average Run Time: Displays the average duration of Simulations in minutes for the selected timeline, helping you monitor how long simulations typically take to complete. 

  • Active Users: Shows the number of users actively running Simulations within the selected timeline, offering insights into team engagement and resource usage. 

  • Active Studies: Displays the number of active Studies from which the most Simulations were triggered during the selected timeline.  

  • Top Engine Version: Highlights the most frequently used Engine Version during the selected timeline, ensuring consistency and optimal performance across Simulations. 

  • Most Active Study: Identifies the Study with the highest number of Simulation Runs during the selected timeline, providing insights into the most frequently used Study for Simulations. 

 

Simulation Triggered Stats 

The Simulation Triggered Stats section offers a summary of the total number of Simulations executed within the selected timeline. It provides you with a quick glance at the overall Simulation activity, showing how many Simulations were successfully completed, how many failed, and how many are still in progress. By monitoring these metrics, you can stay updated on the status of your Simulations and take any necessary actions. 
 
 
 
Refer to the following Key Metrics in Simulation Triggered Stats: 

 

  • Total: Displays the total number of Simulations triggered during the selected timeline, giving you an overall view of the Simulation activity. 

  • Successful: Shows the number of Simulations that were successfully completed. It also includes the percentage of successful Simulations in relation to the total number of Simulations Run during the selected timeline, helping you assess your success rate. 

  • Unsuccessful: Displays the number of Simulations that failed during the selected timeline. It also provides the failure percentage compared to the total number of Simulations, helping you monitor and address any issues that may require attention. 

  • Running: Shows the number of Simulations that are currently in progress, allowing you to keep track of ongoing Simulations. 

 

Credit Usage by Study 

The Credit Usage by Study section provides insights into how credits are consumed across different Studies within the selected timeline. This feature helps you track and monitor the percentage of Credit usage for each Study, giving you a clear understanding of which Studies are utilizing the most resources. By analyzing this data, you can manage resource allocation more effectively, ensuring that Credits are used optimally across all Simulations. 

 

 
Refer to the following key points for the Credit Usage by Study metric: 

 

  • Minimized View: When the Chart is in its minimized state, it displays a pie chart showing the Credit usage percentage for the top 4 studies that consumed the most Credits during the selected timeline. The 5th section of the chart is labeled as Others, representing the remaining Studies grouped together. 

  • Maximized View: When the pie Chart is expanded, it displays the Credit usage for all Studies within the selected timeline, giving you a detailed breakdown of how resources were allocated across different Studies. 

  • Download Plot as PNG: You can download the Chart as a PNG image for reporting or further analysis outside the platform. 

 

Core Efficiency 

The Core Efficiency section provides insights into how efficiently CPU cores are being utilized by different Studies within the selected timeline. This metric helps you monitor the distribution and effectiveness of core usage, allowing you to identify which Studies are using resources most efficiently. 
 



Refer to the following key points for the Core Efficiency metric: 

 

  • Minimized View: In its minimized state, the chart shows the core efficiency in percentage for the top 5 Studies in ascending order. This quick view allows you to see which Studies are using CPU cores most efficiently during the selected timeline, providing a snapshot of core utilization. 

  • Maximized View: When expanded, the chart displays the core efficiency in percentage for all Studies within the selected timeline, providing a detailed breakdown of core usage. This allows you to analyze core efficiency across a broader range of Studies and make informed decisions regarding resource distribution. 

  • Download Plot as PNG: The chart can be downloaded as a PNG image, allowing you to use the data for reporting, presentations, or further analysis. 

 

Errors 

The Errors metric provides details of errors that occurred during Simulations within the selected timeline. This feature helps you identify the issues that may be affecting your Simulations. By monitoring the occurrence of errors, you can improve the reliability and performance of your Simulations and take proactive steps to prevent similar issues in the future. 

 

 

Refer to the following key points for the Errors metric: 

 

  • Minimized View: In its minimized state, the chart displays error details for the top 4 Studies with the most errors, presented in percentage. The 5th section of the chart is labeled Others, grouping the rest of the Studies with fewer errors into one category. This provides a quick snapshot of the Studies that are encountering the most issues. 

  • Maximized View: When expanded, the chart shows the error details for all Studies within the selected timeline. This view provides a detailed breakdown of errors, allowing you to analyze where issues are occurring across all your Simulations and make data-driven decisions to address them. 

  • Download Plot as PNG: The chart can be downloaded as a PNG image, allowing you to store the data for reporting purposes or further analysis. 

 

Memory Efficiency 

The Memory Efficiency section provides insights into how efficiently RAM is being utilized by different Studies within the selected timeline, displayed in percentages. This feature helps you track memory consumption across Simulations, ensuring that your simulations are making optimal use of available resources. By understanding memory usage patterns, you can improve resource management and enhance the overall performance of your Simulations. 

 

Refer to the following key points for the Memory Efficiency metric: 

 

  • Minimized View: In its minimized state, the chart shows the memory efficiency in percentage for the top 5 Studies, arranged in ascending order. This view gives you a quick snapshot of which Studies are consuming the most memory within the selected timeline.

  • Maximized View: When expanded, the chart displays memory efficiency in percentage for all Studies within the selected timeline. This detailed breakdown allows you to assess memory usage across all Simulations and make informed decisions on how to allocate memory resources more effectively. 

  • Download Plot as PNG: The chart can be downloaded as a PNG image, making it easy to include in reports or presentations for further analysis. 

 

 

Credit Usage by User 

The Credit Usage by User section provides insights into how Credits are consumed by individual users within the selected timeline. Displayed in percentages, this feature allows you to track resource usage by each user, ensuring that credits are being distributed and consumed efficiently across your team. By understanding how different users are utilizing credits, you can optimize resource allocation and promote better accountability. 
 

 

Refer to the following key points for the Credit Usage by User metric: 

 

  • Minimized View: In its minimized state, the chart shows Credit usage in percentage for the top 4 users who consumed the most credits within the selected timeline. The 5th section, labeled Others, groups the remaining users together. This gives you a quick snapshot of which users are consuming the most Credits. 

  • Maximized View: When expanded, the chart displays Credit usage in percentage for all users during the selected timeline. This detailed breakdown helps you assess how Credit resources are distributed across all users, enabling you to optimize Credit allocation. 

  • Download Plot as PNG: The chart can be downloaded as a PNG image, allowing you to use the data for reporting, presentations, or further analysis.
