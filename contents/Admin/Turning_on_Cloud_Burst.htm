Turning on Cloud Burst

Cloud Burst is a flexible and inexpensive way to get powerful compute capacity without buying high-end servers that need deployed and maintained in your environment. It is additional capacity that can be accessed to handle large and complex simulations, run parallel simulations across many virtual machines, or speed up processing times to hit deadlines.

NOTE: You must have administrator access to PLEXOS Cloud to manage Cloud Burst.

 

To turn on Cloud Burst:

 

  1. Open PLEXOS Cloud and navigate to the Admin > Usage page. The Admin option is only available if you have administrator access.


  2. Click the Provision Cloud Burst. The Provision Cloud Burst wizard will appear.

  3. Set the Cloud Burst time-frame using the Start Date and End Date fields. Cloud Burst is activated in minimum increments of 24 hours (1 day).


     NOTE: If there are simulations in progress during the last 24 hours of a Cloud Burst period, 24 more hours will be added to the period. This is so active simulations will not be slowed down or stopped unexpectedly. Administrators should be aware that this could use up more Cloud Burst days than they want.


  4. Click on Provision to continue.


Additional virtual machines will now be available when configuring a simulation run. For example, if your organization owns 16 base cores and turns on 64 cores of Cloud Burst, you will now also see 32 core and 64 core virtual machines available during run configuration.
