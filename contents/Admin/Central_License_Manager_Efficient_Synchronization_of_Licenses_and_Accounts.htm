Introduction

In today’s environment, where software licenses represent a significant investment for organizations and security is crucial, Energy Exemplar’s Central License Manager (CLM) stands as a strategic asset. CLM enables efficient, secure, and uninterrupted software license management by centralizing license administration. This ensures you maximize your software investment while maintaining compliance.
 

 

CLM manages software licenses and entitlements from Energy Exemplar. The critical function is to provide control of and visibility into the status of user licenses, enabling you to limit licensing overages, reduce the risk of license non-compliance, and avoid misreporting of commercial agreements.


The CLM's built-in dashboards offer ongoing visibility into license usage and inputs, aiding in software asset management. CLM supports tracking Energy Exemplar software that is licensed based on physical cores, datasets, named users, virtual cores (vCPUs), sockets, or the number of machines.   Admins can monitor which licenses have been issued to specific users or keys, including the dates of issuance, the associated products, and the expiration dates of the licenses. They can also access additional details like machine names and core counts. 


Presenting all this information enhances visibility into the status of every license generated for that tenant.  It encompasses the total number of named user licenses purchased, the remaining quantity, and all cloud entitlements and their current balances. Additionally, it includes licensed product modules in PLEXOS and Aurora. All this data is consistently tracked and monitored for any updates or changes. 
 

Central License Manager (CLM) Overview

Central License Management (CLM) is a licensing tool that helps assign and manage licenses, provides a single platform for efficiently allocating licenses to users, and maintains records of license assignments and changes for accountability and review.

 

Energy Exemplar products including the PLEXOS   Platform (Desktop and Cloud), Aurora, and PLEXOS Connect, fetch and synchronize licenses using CLM. CLM is essential for efficient license management and enables smooth and accurate access. 

 

Core Features and Benefits:

  • Subscription: A Subscription in the Central License Manager is an agreement granting users access to specific software features for a set period. It involves:

     

    • Access Rights: Specifies features and services available to the subscriber.
    • Duration: Defines the subscription's validity, typically monthly, annually, or multi-year.
    • Usage Monitoring: Tracks usage and compliance with license terms.

      Subscription details are vital for managing software licenses across an organization in a central license manager (CLM). They help track and identify licenses, while real-time usage data provides insight into license availability and compliance. Subscription details offer a clear view of assigned licenses, including how they are pooled, distributed, and tracked based on the user roles.

 

  • User Management: In Central License Manager (CLM), user management is essential for controlling access to licensed software within an organization. The system allows administrators to view a list of active and inactive users, add new users, assign licenses, and set user roles based on their responsibilities. Admins can also send welcome emails to new users, providing them with the necessary access information.  CLM enables admins to manage user status, allowing them to mark users as active or inactive, which is helpful for temporary or suspended access. If a user is no longer needed, their license can be freed up for reallocation. This ensures efficient use of licenses and keeps the system organized by removing unnecessary users.

  • Single Sign-in (SSO) Support: Single Sign-On (SSO) enables licensed users to access multiple systems or applications with single login credentials. Users can access the CLM and associated applications with a single set of credentials, eliminating the need to remember multiple passwords.

  • Access Keys: The Access Keys page in the CLM allows users to generate or regenerate essential login keys, such as Client ID, Primary Secret, and Secondary Secret. These keys enable seamless access to Energy Exemplar services without requiring users to log in to PLEXOS Cloud directly. Located on the Access page, these generic keys can be utilized to access Energy Exemplar services such as PLEXOS Connect, the Command Line Interface (CLI), and the Application Programming Interface (API). Users can regenerate and copy these login keys and secrets multiple times, ensuring smooth service access.

  • Enhanced Security and Control with SSO Integration in CLM: CLM is seamlessly integrated with Single Sign-On (SSO) systems. It enhances enterprise-level security and gives organizations granular control over software access and usage. Administrators can effortlessly manage user accounts, assign roles and permissions, and monitor license allocation, ensuring a secure and compliant software environment.  

  • Instant Transfer Named User Licenses: This feature enables smooth license transfers between users, eliminating delays and costs typically associated with traditional license acquisition or switching processes. CLM ensures seamless operations and uninterrupted workflows, maximizing productivity and minimizing operational disruptions.

  • Offline License Checkout: This capability allows users to check out a license for up to 30 days, enabling continuous work in remote or low-connectivity areas. This feature ensures productivity and workflow continuity regardless of location or connectivity constraints.

 

Functionality Overview

  • Account Synchronization: Account synchronization in the Central License Manager involves maintaining consistent updates and alignment of user accounts and their associated licenses across all systems and platforms. This process includes synchronizing user details, roles, and permissions to ensure accuracy and consistency.

    It also involves reflecting any changes in license status, such as additions, removals, or transfers. This process ensures that contract and license data remain current and aligned with the user's environment. By continuously tracking and monitoring key information such as the number of purchased and remaining named user licenses, cloud entitlements, and licensed product modules in PLEXOS, and Aurora Admins can quickly identify any discrepancies or changes. 

    This proactive approach helps maintain compliance, optimize resource allocation, and prevent potential issues related to expired licenses or overconsumption of entitlements. Essentially, it keeps the organization’s licensing in sync with its evolving needs and infrastructure.
      
  • User Management: CLM facilitates the creation of new users on the PLEXOS Platform, assigning appropriate license types based on stored user data. Users can be categorized as Analysts, Reviewers, Executives, Admin Only, or Cloud Analysts, ensuring they have the necessary permissions and access levels to perform their roles effectively.  

  • License Refresh and Subscription Renewal: 
     
    • License Refresh: License refresh refers to the process of updating or renewing the PLEXOS Platform licenses to ensure continued access to its f  eatures and services. This can involve reactivating or extending the validity of a license, ensuring compliance, and maintaining uninterrupted use of PLEXOS. CLM automatically refreshes the licenses for the PLEXOS Platform after 30 days. Users can manually refresh their licenses using a few Command Line Interface (CLI) commands available in the Central License Manager article. 

    • Subscription Renewal: Based on your Subscription, you will have access to multiple Energy Exemplar applications, such as Aurora, PLEXOS Desktop, PLEXOS Connect, etc. Tenant Admins can contact their Customer Success Manager or Account Manager to renew their Subscription at expiry.
       

 

Data Collection and Transmission Overview for Energy Exemplar

CLM (Central License Manager) collects licensing-related information to enhance customer support and improve the product. The data collected includes a limited number of licensed user data, to be specific - User name, email address, machine identifier, and IP address. No study data or Sensitive Personal Information (SPI) other than mentioned above is collected, stored or processed. 

 

Please refer to the Energy Exemplar’s Privacy policy, which complies with GDPR and other privacy-related requirements. The data in transit and “At Rest” is encrypted. For PLEXOS Connect customers, no user-specific information is gathered. However, simulation-related information such as run duration and features used is sent back to Energy Exemplar.

 

When a user signs into the Cloud platform, they must use the Identity service, which logs authentication attempts. These logs are stored in a database, but the information is not sent to Energy Exemplar. Instead, it involves publicly available client IP web requests made to the API. This allows the system to identify the authenticated user and associate their IP address with license requests. 

 

The identity server audits all login attempts, with the data recorded in a SQL database. Long-term historical records, which are used for compliance and event reconstruction, are stored in a secure, cloud-based storage solution. Additionally, the Central License Manager (CLM) stores computer names and identifiers in a SQL database for license enforcement. All data is securely stored within the environment selected by the tenant. It is important to note that the IP captured is the customer's external IP address, not the IP of any individual machine. Energy Exemplar cannot monitor activities behind a customer’s firewall or track traffic routed through another external connection. 

 

The logged IP is essential for identifying the origin of a request, and Energy Exemplar has started exploring the option of blocking requests from specific geographical areas based on this information. Knowing the connecting IP helps ensure that they can monitor who is attempting to connect at any given time.
 

 

Air Gapped Machine Access

In scenarios where PLEXOS is installed on machines that are completely isolated from external networks, obtaining a license via CLM is not feasible. These Air Gap machines, which are entirely disconnected from the internet, cannot meet the basic requirement of internet connectivity necessary for fetching the license through CLM. 

 

 

The Central License Manager is essential for securely managing and accessing Air Gapped machines systems isolated from external networks for security purposes. It allows you to connect to these machines by serving as an intermediary, ensuring sensitive data remains protected. You can securely transfer licenses and updates by exporting them online and manually importing them into the Air Gapped system. 

 

The Central License Manager ensures licenses are correctly applied, keeping your software operational. Additionally, it provides a user-friendly way to track and manage licenses across multiple Air Gapped machines, making it easier to maintain compliance and functionality in high-security environments. Cloud Operations Standards

Cloud operations standards ensure efficient, secure, and compliant management of cloud environments. Key elements include:

Data Access Controls and Auditing

Energy Exemplar employees do not have access to customer data by default. Access to customer data is restricted. We use several layers of controls for monitoring data access, including administrative and technical controls. Customers may choose to authorize access to support or cloud operations staff for testing or validation purposes.

 

General Operating Stance

Energy Exemplar's Cloud offerings are ISO 27001 and SOC 2 TYPE II certified and undergo annual 3rd party audits. The General Operating Stance refers to its overall approach to managing operations, including principles, policies, and practices that guide its day-to-day functioning.

To ensure the highest levels of security, compliance, redundancy, and fault tolerance, our Data Centre Operations are conducted with best-in-class infrastructure services from top-tier public cloud providers. We operate in geographically dispersed SOC2 TYPE 11 and ISO 27001-compliant facilities, which are certified in public records by each cloud provider.

 

Physical Security

Physical security is essential for safeguarding the infrastructure that supports CLM systems, ensuring the protection of sensitive license information, and maintaining overall system integrity. Proper physical security measures complement logical and cyber security practices to create a comprehensive security posture for CLM. 

The public Cloud Infrastructure Provider provides physical security and is audited under its compliance standards. Energy Exemplar has no physical access to any cloud infrastructure. A copy of the Provider’s compliance report can be provided upon request.

 

Transport Security

Transport security is vital for ensuring that data associated with CLM systems is protected from unauthorized access, tampering, or interception during transmission. By implementing strong encryption, authentication, and network security measures, you can safeguard the CLM data and maintain the integrity and confidentiality of their license management processes.

PLEXOS Cloud transfers data securely through the industry-standard method of Windows Communication Foundation with TLS 1.2 or above encryption. To support back-end integration from your network, Energy Exemplar can provide VPN connections using industry-standard AES (128 or 256) IPSec encryptions.

 

Data At Rest in the Cloud

Data at Rest in the cloud refers to data that is stored in a cloud environment, such as in databases, storage services, or file systems, and is not actively being transmitted or processed. Data that is uploaded into the cloud is, by default, encrypted at rest.

 

Customer Environment

Energy Exemplar operations establish a standard and secure tenant network for each customer. Securely housing customer data in a separate network ensures tenant data confidentiality. Additionally, application-specific controls protect data from unauthorized access across multiple application layers.

 

Authentication and Authorization

All access to the Cloud Service is authenticated and authorized. Energy Exemplar offers customers several options to simplify the user login experience without compromising security. We use Local Authentication where passwords are stored inside the Cloud environment, maintained separately from your network credentials. Customers can also choose to integrate with their in-house Microsoft Active Directory.

 

Data Privacy

Data privacy involves protecting personal and sensitive information through proper classification, encryption, access control, and adherence to legal and regulatory requirements. Energy Exemplar’s data privacy policy can be found at https://www.energyexemplar.com/privacy

 

 

Data Backup and Disaster Recovery

The service utilizes the backup services provided by the Cloud Provider. These backups are replicated in multizone, which enables data restoration (data resiliency). Application Release and Quality Controls

To ensure consistent quality and seamless experience, the development team goes through multiple integrated stages of testing including but not limited to the following:

  • Code reviews are performed on all application source changes: All source code is peer reviewed/architect reviewed before merging to ensure security, quality, and consistency.

  • Continuous automated testing during the development cycle: Our team runs continuous tests during the development cycle to identify defects early in the development cycle and resolve them early in the development process.

  • Backward compatibility testing: We maintain a large repository of models with which we test for backward compatibility and increased model accuracy. Our support and QA teams collaborate with customers to increase this repository of models.

  • Security testing: As part of our DevOps release pipeline, we go through static source code analysis for security issues consistent with industry best practices.

  • OWASP Top Ten Security testing: Our architects are trained in OWASP security testing and perform security reviews prior to every merge and release.

  • Security Penetration testing by independent vendor: PLEXOS is security tested and certified by an external vendor annually.

  • Defect Queue Reviews: We perform reviews of the defect backlog internally before every release with a policy to not release with any open severity 1 or 2 issues (major functionality).

  • Close monitoring of Escape Defects: Our support team tracks and reports on customer defects after a release to ensure the quality of the release is consistent with expectations.

  • Our release process: We have several stages with an open Beta to allow customer exposure late in the development cycle to validate release quality and product performance.


Application Security and Testing

Application Security is a strong consideration during every step of our deployment process and ongoing maintenance as well as key steps within our application development and release processes. As standard practice we conduct the annual third-party security testing on the application to resolve all high and medium security incidents.

 

The objectives for Web Application & Application Programming Interface (API) security testing are:

 

  • To verify the input validation functionality of Energy Exemplar application.

  • To verify the access control and authentication-related security flaws of Energy Exemplar application.

  • To test the security of Energy Exemplar application in its business workflow level.

  • To analyze the security of all the API endpoints associated with Energy Exemplar application.

  • To analyze whether transactions and all other data of the application are encrypted (both At Rest and In Transit) with updated encryption algorithms.

  • To analyze the security of the backend server system of web application portals.

  • To identify whether it is possible to tamper with the transaction amount processed by the application API.

  • To identify configuration level security vulnerabilities, present in the web application server and API server.

  • To verify that the Access Control Policy is enforced in the Energy Exemplar Application.

  • To identify application data breach possibilities with respect to OVVASP ASVS.

  • To identify if an application is exposing sensitive files like source code, API code, configuration files, etc., to public access.

  • To identify if an application has a rate-limiting feature to block Denial of Service attacks.

  • To analyze Access logs and identify if logs are exposing sensitive information.

Guidelines for Access Control and Security at Energy Exemplar

Access control and security are crucial for protecting sensitive information and systems. Here are some guidelines:

  • User Credentials
    An account will be created for staff with appropriate access as requested by their reporting/line manager. Login credentials are to be kept secure and are not to be shared with another individual. Staff must change their password when automatically prompted. If an individual believes their login credentials have been compromised, they must notify IT services immediately.

  • Minimum Password Requirements
    EE has implemented strong password policies and guidelines in line with ISO 27001.

  • Multi-Factor Authentication
    All staff, contractors, or third-party vendors assigned an Energy Exemplar email address or access Energy Exemplar platforms must have Multi-Factor Authentication enabled. This provides an added level of protection in the event user credentials are stolen. Users have the option to configure their mobile numbers to receive a verification code via SMS/TXT or download and install Microsoft Authenticator on their Mobile Phones.

  • Email
    All emails are tracked and archived for retention and backup purposes. This can only be accessed by Energy Exemplar IT Services in the event of a restore process or security breach, or when requested by a reporting/line manager. Mimecast’s Email Security Platform has been implemented to appropriately protect Energy Exemplar. IT Services receive alerts if a message is in breach of a security policy and can reroute, delete, or block an email message without the end user’s knowledge.

  • Remote Access
    Staff who require access to infrastructure, such as but not limited to Local Storage, Testing, and Simulation Servers whilst away from an Energy Exemplar office, can remotely connect to these resources via a company-provided secured VPN connection. The currently approved method of remote access is via VPN client - FortiClient. Access to local infrastructure via any other connection method is strictly prohibited such as, but not limited to TeamViewer, GoToMyPC, and ConnectWise.

  • Facilities
    Energy Exemplar offices require the use of a swipe card/pass to gain access to the premises. Each pass is assigned to a staff member and access to the building is logged for security purposes. Swipe cards/passes are not shared by employees, nor are they allowed to be used by unauthorized personnel. If a pass is lost or stolen, staff must immediately contact IT Services to have the pass disabled. For Energy Exemplar offices that use Key access, under no circumstance are keys given out to external parties and or copies made. Each key has a unique number that is assigned to each employee and is to be handed back at the end of their employment at Energy Exemplar.

  • Company Issued Smartphones
    All devices that connect to the Energy Exemplar network and or emails must be password protected. The operating system on the device must not be tampered with or changed. iPhones are not to be Jailbroken and Android devices must not be Hooted. Only official firmware updates can be applied to mobile devices.

  • Desk phone - Landline
    Staff will be provided a fixed-line desk phone if required to perform their role. The use of landline services is monitored and tracked and may be reviewed as part of an audit, security breach, or at the request of a reporting/line manager.

  • Information Security Awareness Training
    Staff are required to undertake mandatory information security awareness training.

  • Reporting of Security Incidents
    It is the responsibility of all employees to report any incident, or suspected incidents, involving a breach of Information and or a breach of IT security where information or data has been externally obtained. These incidents are to be reported directly to the Head of Global IT for investigation without delay.

  • Portable Storage Devices
    Portable storage devices are not to be used unless issued by Energy Exemplar IT Services. Issued USB devices will be encrypted, and passwords are protected to ensure that in the event they are lost or stolen, information is not able to be recovered. USB ports on all workstations will be disabled for portable storage devices unless it is deemed a requirement for the employee to perform their role. Ports will continue to operate other USB devices.

  • Loss or theft of Equipment
    When a device (laptop, smartphone, iPad) is lost or stolen, the employee must notify IT immediately to report the incident at any time of day or night. The IT department will, among other things, reset passwords and block all access to network resources including email until the issue has been resolved.

The release deployment strategy for the Central License Manager (CLM) in PLEXOS Cloud ensures a seamless transition with regular updates.
