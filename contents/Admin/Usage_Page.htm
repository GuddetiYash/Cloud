Overview

The Administration > Usage page provides a structured and insightful view of Core or Credit usage, helping Admins manage resources proactively and make informed decisions. This update offers a clear overview of allocation, consumption, and balance, enabling better optimization and planning for future needs.

 

Credit Usage Metrics

The top section of the Usage page features key metrics that give Admins immediate visibility into Credit allocation and consumption details, empowering better monitoring and forecasting.

 

 

Refer to the following details about each metric:

  • Credit Start Date: Displays the date of the earliest Credit purchase transaction (active), helping to identify when Credits first became active. If multiple transactions are present, the oldest transaction date is shown. 

    For example, if you have two Credit Purchase Start dates, 1 Jan and 1 Feb, 01 Jan will be considered the Credit Start Date.

    NOTE: The date format follows user preferences set under the Profile Name > Preferences page.
     
  • Credit Expiry Date: Displays the date of the most recent Credit purchase, reflecting when Credits will expire if they are unused. If multiple transactions are recorded, the latest transaction date is shown. 

    For example, if you have 2 Credit Purchase End Date made respectively on 1 Oct and 1 Dec, the n 01 Dec will be considered the Credit End Date.

    NOTE: The date format follows user preferences set under the Profile Name > Preferences page.
  • Purchased Credits: Displays the total Credits purchased by the tenant during the current Credit tenure.
     
  • Consumed Credits: Displays the total Credits used, including both Purchased and Overage Credits to provide a clear breakdown of consumption and coverage:
    • Purchased Credits (displayed as positive values).
    • Overage Credits (displayed as negative values in red).
       
  • Credit Balance: Displays the remaining Credits based on the following logic:
    • If the Consumed Credits are lesser than Purchased Credits, the Credit Balance shows as positive value.

    • If the Consumed Credits are greater than Purchased Credits, the Credit Balance shows as negative value  in red.

    • If your Credit Balance is negative and you buy new Credits, it will be reflected as positive value under Credit Balance and the previous negative Credit Balance value will move under Overage Credits that you need to settle separately.


       
  • Overage Credits: Overage Credits represent the Credit deficit incurred when Consumed Credits exceed Purchased Credits. These are recorded separately after the purchase of new Credits and is marked red with a "-" sign for easy identification.
     

 

Usage Trend Tab

For Credit-based customers:

The Usage Trend tab in the Admin > Usage page provides a comprehensive view of Credit Usage and Credit Balance over time, offering a clearer picture of your tenant’s credit activity. This tab includes interactive tools to enhance data visualization and trend analysis.

To further enhance your experience, the chart view includes new tools as follows:

  • Markers – Enable or disable data point markers for better trend identification.
  • Grid Lines – Toggle grid lines for improved readability of the trend graph.
  • Legends – Show or hide the legend to clarify different data series.

 


 

Additionally, you can select a custom date range (up to one year) to analyze your usage trend data, allowing for greater flexibility in monitoring Credit consumption over time. 

 

Chart Customization Options

These Chart Customization Options allows you to download the chart in PNG format, incorporate shapes onto the chart, implement zooming and autoscaling functionalities, and more. 



 

  • Download plot as a PNG
    This option allows you to download the chart in PNG format. You can utilize this option to easily download and share their customized charts with colleagues and clients, or integrate them into presentations, reports, or documents, facilitating seamless collaboration and communication.  

    Click on the Camera icon to download the Chart in PNG format.


     
  • Zoom Button 
    This option allows you to zoom in to the desired area of the Chart. You can utilize this option to focus on specific data subsets for deeper analysis.   

    To zoom in on a particular area of the chart: 
    • Click on the Zoom icon and select the area you wish to zoom in on the chart. 
    • To return to the original view, double-click anywhere on the chart.   


       
  • Pan Button 
    This option allows you to move or scroll across the chart horizontally or vertically.  

    To pan across the chart: 
    • Click on the Pan icon and click and hold your mouse to explore the data on the chart. 
    • To return to the original view, double-click anywhere on the chart. 


       
  • Box Select 
    This option allows you to select a specific range or subset of data points within the chart for further analysis of the data. This is particularly useful when dealing with large datasets or complex visualizations where isolating specific data points or regions is necessary for deeper analysis. 


     
  • Lasso Select 
    This option is another way like box selection to select a specific range or subset of data points within the chart for further analysis of the data. Instead of drawing a box to select data points, you can use the lasso tool to draw a freehand shape around the desired data points. This is particularly useful for selecting non-linear or irregularly distributed data points.  

    Lasso select is beneficial when dealing with complex data structures, such as scatter plots with overlapping points or line charts with multiple series, where traditional box selection may not be sufficient. 


     
  • Zoom In/Out 
    These options allow you to zoom in or out on specific areas of the chart to focus on details or to get a broader view of the data. 


     
  • Autoscale 
    This option automatically adjusts the axes of the chart to fit the range of the data being displayed. It ensures that all data points are visible within the chart's boundaries, optimizing the visualization for better clarity and readability. You can use this option to quickly adjust the scale of the chart without manually adjusting the axes, especially when dealing with datasets with varying ranges or outliers. 


     
  • Reset Axes 
    This option resets the axes of the chart to their default scales and positions. It provides you with a convenient way to revert any changes made to the axes, such as manual adjustments or zooming, and return the chart to its original state. You can use this option to reset the chart for a fresh analysis or to restore the initial view after exploring specific regions or subsets of data.

 

For Core-based customers:

The Cloud Burst Usage chart in the Admin > Usage page provides improved visibility into resource consumption for core-based customers. The enhanced Cloud Burst Burndown chart now displays both Burndown and Actual Usage, enabling a clearer comparison of allocated vs. consumed resources over time.

 

Additional tools have been introduced to enhance the data visualization:

  • Markers – Highlight data points for easier trend analysis.
  • Grid Lines – Improve readability by toggling grid lines.
  • Legends – Enable or disable the legend for better clarity.
     

 

 

These updates help you to track resource depletion more effectively, ensuring better planning and optimization of cloud burst capacity.

Transactions Tab

The USAGE and TRANSACTION HISTORY tabs have been merged and renamed TRANSACTIONS. This new tab provides you with the Credit usage and transaction-related details in one place, making it easier to track current transaction activity.


 

 

Refer the following description of each column:

Column Name

Description

Study Name

Displays the name of the study associated with the transaction.

Models

Indicates the model type used in the transaction (e.g., High, Low, Base).

User

Shows the name of the user who executed the transaction.

Transaction Date

The date and time when the transaction was recorded.

Transaction Type

Displays the type of transaction. Only active transaction types are shown: Purchased, Refunded, Simulation.

Simulation Type

Specifies whether the simulation was Single or another type.

Start Time

The timestamp when the simulation or transaction started.

End Time

The timestamp when the simulation or transaction ended.

Simulation Status

Indicates the completion status of the simulation (For example, ✅ for successful completion).

Credits

Shows the number of credits consumed for the transaction.

Execution Time (Mins)

Displays the actual time (in minutes) taken by the Virtual Machine (VM) to complete the simulation.

CPU (Core)

Shows the number of cores selected for the simulation, as configured in the Run Configuration panel.

Memory (GB)

Displays the memory allocation chosen in the Run Configuration panel for the simulation.

Description

Provides details for Purchased and Refunded transactions. This field remains blank for simulations.

Simulation ID

Displays the unique ID for each simulation executed within the tenant. This remains blank for Purchased transactions.

Transaction ID

Displays the unique ID assigned to each transaction, helping in tracking and referencing.

 

Export Data

You can export historical credit utilization data for your tenant using the Export icon. This feature allows you to select predefined date ranges (3 months, 6 months, 9 months, or the past year) or specify a custom range (up to one year) for retrieval. Exporting data in .CSV format enables easier tracking, analysis, and reporting of credit usage, helping you make informed decisions and optimize resource allocation.
