Study Level Permission

In this article

  • Provide Study Acces to other users 
  • Pushing and Pulling Study Changes through CLI 
  • Remove yourself from a Private Study

 

Prior to the February 2022 release, Studies were by default public and could be viewed by all users in the PLEXOS Cloud interface but only visible to the user who uploaded it in Cloud Bridge and the Excel Plugin. With this new release, Studies are now by default private, and only the user who uploads them (the creator) can view it. The creator can add other users to their study, thus permitting them to access it in the PLEXOS Cloud interface, CLI, and Excel Add-In.


 NOTE: Only the Study Owner and Admin can Archive, Rename, and edit the description of a Study. Previously, all members of a Study would be able to perform these actions.
 



 

Users with a Modeler license, who are part of the team for a Study, can access the Study through Cloud Bridge to view and edit directly on their desktop using PLEXOS. They can also use the MS Excel Add-In to edit data for the same Studies. 


Provide Study Access  to other users

You can add members to a study from the Study Overview page. Only active users can be added to a Study. Inactive users cannot be added. 

 

  1. Navigate to the Overview page.

  2. Click on the MANAGE TEAM option in the top right corner of the Study Overview page.
     


     

  3. Select the user from the drop-down, and click on the ADD button.


     

  4. Access to your study is granted to the users you have added.

 NOTE: Only active users can be added to a Study. 

 

Pushing and Pulling Study Changes through CLI

If changes have been made to a local Study, they can be pushed using the command. This command requires a studyId and a message (description of changes). For example:

 

plexos-cloud.exe study changeset push --studyId --message "Sample Message"

 


Changes since the most recent changeset are required, if there have been no changes you will receive an error message. If there are changes to push then you will only get a successful "Changes Pushed" message at the end of the push operation. You will also be informed of the new ChangesetId as shown below.
 


The new changeset can also be confirmed by using the study changeset list --studyId command. If there are no changes found, the CLI will output "Study has no changes to push."

If there are pending changes on the server that have not yet been pulled down locally, they can be pulled using this command:
 

plexos-cloud.exe study changeset pull --studyId

 

Remove yourself from a Private Study

NOTE: This change is applicable for both Simulate and Analytics tier customers.


Users can now leave a Private Study if they no longer want to be part of it or receive any notifications (notifications can be managed from the Preferences page). This will help users better focus on the Studies they are working on and avoid distractions by being added to unnecessary Studies.

 

 

If you are the Owner of a Private Study, you can leave a Study only after you assign someone else as the Owner of that Study. Please note that users cannot leave a Public Study. 


NOTE: If you leave a Study marked as Favorite, that Study will be removed from the Favorites section of the Study dropdown.

 

To change an Owner of a Study:


NOTE: The steps below are applicable only for the Study Owner. Only the current Study Owner can assign someone as a new Owner. An Admin or Read-Only user cannot view these options if they are not the Owner of that Study.

 

  1. Navigate to the Overview page and select the Study for which you want to change the Owner from the Study dropdown. 
     
  2. Click on the MANAGE TEAM option in the top right corner of the Study Overview page.


  3. Click on the kebab menu next to the Member Name you want to add as an Owner and select the Make Owner option.


  4. You will see the confirmation pop-up as shown:


  5. Click on the CONFIRM button to add the selected user as a new Study Owner.

 

To Leave a Private Study:

 

  1. Navigate to the Overview page and select the Study for which you want to change the Owner from the Study dropdown. 

  2. Click on the MANAGE TEAM option in the top right corner of the Study Overview page.


  3. Click on the kebab menu next to your name and click on the Leave Study option.


  4. You will see a confirmation pop-up as shown:



    NOTE: Once you Leave a Study, you will not have access to that Study, this includes from the PLEXOS Cloud interface, PLEXOS desktop, the CLI, or the Excel Add-In.

  5. Click on the CONFIRM button to continue.

  6. The following pop-up will appear to confirm that you have successfully left the Study. 


  7. You will then be re-directed to the first Study in the Study dropdown. If you don’t have access to any other Study, a message “There are no studies to show in this tenant, Clone a Playbook from Marketplace to start using PLEXOS Cloud” will appear.
