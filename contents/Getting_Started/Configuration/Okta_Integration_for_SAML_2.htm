Okta Integration for SAML 2.0 idP

This is an overview for creating an idP application in Okta for SAML 2.0 authentication fo PLEXOS Cloud.

In this article:

  • Create/Configure the SAML 2.0 idP App
  • Authentication from Okta Application

 

Read below for step-by-step instructions on creating an application in Okta for SAML authentication, and identity provider (iDP) initiation authentication using Okta.

 

Refer to the region-wise URLs below for each of the environments listed in the Configuration table.
 

  • APAC:  identity-au.energyexemplar.com 
  • EMEA: identity-eu.energyexemplar.com 
  • NAPROD : eeprod-na.energyexemplar.com

Create/Configure the SAML 2.0 idP App

Creating the Application 

1. To start, you will need to retrieve your Energy Exemplar (EE) Identity Web Server URL, which will be used to configure the idP. This will be provided for you by an EE Support Team member.

 

2. Next, create an Okta account on the Okta website to configure a SAML 2.0 idP. Once you have made an account, go to Applications and select Create App Integration.

 


3. When prompted to Create a new app integration, select SAML 2.0 as the method.
 


4. Give the application a name, and select Next.
 


5. Configure the General settings for SAML. Use the Identity URL that was shared with you as the Single Sign-On (SSO) URL to fill in the settings as shown in the below table. Set the Name ID format to EmailAddress and Application username to Email.

Add and Assign Users to the Application  

Under the Directory tab, go to People and click to Add person and specify the required credentials.

 


Return to the Application and go to Assignments. Place the cursor above assign and click Assign to People to select the Users who have access to the application.

 


Follow the steps listed in a related article to Configure/enable SAML 2.0 in PLEXOS Cloud .

 

Authentication from Okta Application

For idP-initiated authentication, follow the following steps.

  1. Log in to Okta and idP.
  2. Click on the Application in idP dashboard.
  3. You should then be redirected to PLEXOS Cloud.
