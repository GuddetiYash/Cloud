OpenId Connect (OIDC) Integration

Overview

The OpenID Connect (OIDC) is a protocol built on the OAuth 2.0 framework, designed to facilitate secure authentication and authorization between an Identity Provider (IdP) and a Relying Party (RP) typically the application or service a user wants to access. As an identity layer on top of OAuth 2.0, OIDC enables applications to verify a user's identity based on the authentication performed by the IdP, supporting authentication (verifying who the user is) and authorization (determining which resources the authenticated user is permitted to access).

Configure Azure IdP

Azure IDP (Identity Provider) is a cloud-based service that enables authentication and authorization for users to access various applications and resources within the Azure ecosystem. It allows users to authenticate with their existing credentials, such as usernames and passwords and provides a single sign-on experience across multiple applications and services.


It supports various authentication protocols, including Security Assertion Markup Language (SAML), OpenID Connect (OIDC), and OAuth 2.0, allowing it to integrate with multiple applications and services.

Register New Application in Azure IdP

Registering a new application in Azure Identity Provider (IdP) is essential for enabling secure authentication and authorization for applications and services.


To register a New Application in Azure IdP, perform the following steps:

 

  1. Open the Azure Entra ID, switch to the App registration from the primary (left) navigation menu and select New registration.

  2. In the Name field, enter PLEXOS Cloud.

  3. Select Account in this organizational directory only (Default Directory only – Single tenant) in the Supported account types.

  4. After the User is created, open the Overview tab from the primary (left) navigation menu, capture the client ID and tenant ID.

  5. Navigate to the Authentication tab and ensure that the ID Tokens option is selected.

  6. Navigate to the Token configuration tab and open Add optional claim.

  7. Select ID under the Token Type field.

  8. Enable email, family_name, and given_name under Token Types, then click the Add.

  9. Navigate to the API permissions tab and click Add a permission.

  10. Enable email, openid, and profile under Microsoft Graph, then click the Add permissions button.

 

Configure and Enable Single Sign-On (SSO) in the Central License Manager (CLM)

Single Sign-On (SSO), is a mechanism for logging in to multiple applications or systems using a single set of credentials. Enabling SSO involves several steps and can vary depending on the specific technologies and applications involved.


To enable SSO in PLEXOS Cloud, you must integrate the Energy Exemplar Identity Web Server URL and configure the OIDC Identity Provider (IdP), PLEXOS Cloud SSO, and Validation.


NOTE:  Ensure you meet Okta and Azure IdP Configuration prerequisites before proceeding with the SSO setup.

 

  1. Open the Central License Manager (CLM) and switch to the SSO page from the primary (left) navigation menu.   

  2. In the Protocol field or dropdown, select the OIDC option.

  3. In the Authority field, enter the OIDC Metadata URL given by the OIDC provider.

    NOTE: The OIDC Metadata URL is one of the prerequisites mentioned above under Okta/IdP Configurations.

  4. In the Client ID field, enter the valid Client ID provided by the OIDC.  


     
  5. Set the Post Logout Redirect URI, Redirect URI, and Scope as NA.

    NOTE: The Client ID can be found in the prerequisites mentioned above under Okta/IdP Configurations.

 

Validation

You can validate your SSO using the following steps:

  1. Log out and return to the Central License Manager.
  2. Enter your Email Address, which will redirect you to the OIDC IdP.
  3. Log in to the IdP and you will be redirected back to the Central License Manager.
