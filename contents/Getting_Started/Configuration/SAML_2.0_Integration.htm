Overview

SAML (Security Assertion Markup Language) is an XML-based standard for exchanging authentication and authorization data between parties, specifically between an identity provider (IdP) and a service provider (SP).

And, Authentication is the process of verifying the identity of a user or device, while authorization is the process of granting or denying access to a resource or service based on the authenticated identity. 

SAML works by allowing an identity provider to authenticate a user and generate a SAML assertion, which is a digitally signed statement that contains information about the user, such as their identity and authentication status. The identity provider then sends this assertion to the service provider, which can use it to grant access to resources or services based on the information in the assertion.

Configure and Enable SSO in PLEXOS Cloud

SSO, or Single Sign-On, is a mechanism that allows you to log in to multiple applications or systems using a single set of credentials. Enabling SSO involves several steps and can vary depending on the specific technologies and applications involved.

To enable SSO in PLEXOS Cloud, you need to integrate the EE Identity Web Server URL and configure the SAML 2.0 Identity Provider (IdP), PLEXOS Cloud SSO, and Validation.

 

 NOTE : Before proceeding with the below setup for SSO, please ensure that you have checked the prerequisites for Okta and Azure IdP Configuration.

After setting up the prerequisites, follow the steps below to proceed with the SSO setup: 

 

1. Go to the Single Sign On Settings page in PLEXOS Cloud Admin panel and click the Add New Settings button.

 

  

 

2. For the Protocol, set it to SAML2. Set the Authority as the "SAML2 Metadata URL" given by the SAML provider.

 

NOTE: The "SAML2 Metadata URL" can be found in the prerequisites mentioned above under Okta/IdP Configurations.

 

 

3. Set the Post Logout Redirect URI, Redirect URI, and Scope as NA.
 


4. For the Client ID, set it to the Entity ID given by the SAML provider. Click Save to enable SAML 2.0 in the application.

 

NOTE: The Client ID can be found in the prerequisites mentioned above under Okta/IdP Configurations.

 

 

Validation

You have two different options for validation: Service Provider (SP) initiated or idP-initiated.

For SP-initiated login, follow the following steps.
 

  1. Log out and return to PLEXOS Cloud.
  2. Enter your Email Address, and you should be redirected to the SAML 2.0 idP.
  3. Log in to the idP, and you will be redirected to PLEXOS Cloud.


For IdP-initiated login, log in to the IdP and follow the directions given by the provider so that you are redirected to PLEXOS Cloud.
