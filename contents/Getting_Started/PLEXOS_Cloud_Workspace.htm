Overview 

This document describes the PLEXOS Cloud Workspace offering. This guide will help you get started with PLEXOS Cloud Workspace and provide essential information to make the most of this service. It will guide you through the features and benefits of PLEXOS Cloud Workspace.

 

With PLEXOS Cloud Workspace, you can perform the following activities: 

  • Create and edit the Models with the PLEXOS Desktop application.  
  • Upload and Sync database from PLEXOS Desktop to PLEXOS Cloud. 
  • Use PLEXOS Cloud with the Chrome browser.  
  • Use PLEXOS Cloud Command Line Interface (CLI). 
  • Integrate with OneDrive for moving data from your company network to PLEXOS Cloud Storage.

Cloud Workspace Setup 

The PLEXOS Cloud Workspace environment is provisioned and set up by Energy Exemplar’s Cloud Operations Administrators. If there are any issues with the setup, please contact the Energy Exemplar Support Team at support@energyexemplar.com. Once user accounts are created, Energy Exemplar will share the login credentials to access the PLEXOS Cloud Workspace. 

Access to the Cloud Workspace App 

When you sign into the web client, you will see your workspace. A workspace combines all the applications made available to you by your admin. Follow the below mentioned steps to sign in: 

  1. Open your web browser. 
    NOTE: Ensure that your microsoft account is not logged in to the  below mentioned URL. You can try opening the URL in incognito mode to avoid such confusions.

  2. Copy and paste the following URL into the browser.
    Azure Cloud: https://windows365.microsoft.com/ent#/apps

  3. Enter the account Username provided by the Energy Exemplar. Click Next. 

  4. Enter the temporary Password provided by the Energy Exemplar. Click Sign in. 

  5. When prompted to set a new password, enter the new password and click Sign In.

  6. Next step allows you to setup the Multi-Factor Authentication (MFA). Click Next to proceed with MFA for your account. 

  7. Download the Microsoft Authenticator App on your mobile device and click Next to proceed with the installation. 

  8. Open the Microsoft Authenticator App on your mobile device and tap the (+) icon to add a new account.
  9. Select "Work or School Account" and click Scan a QR code. Then click Next in the browser where the authenticator page is open.
  10. Scan the QR code displayed on the setup screen using the Authenticator App and click Next.


  11. A pop-up window will appear on the screen, displaying a two-digit number. Enter the displayed two-digit number on your mobile device and click Yes.

File Upload 

  1. A maximum of 255 MB can be uploaded per file or folder from a local computer to the portal using Cloud Workspace. To upload files, click on the upload arrow available on the right side of the user interface. For larger files, use OneDrive or any other cloud-based storage system. 

  2. Once the upload is complete, go to File Explorer > This PC > Remote Desktop Virtual Drive on RDWebClient > Uploads. 

File Download 

  1. To download a file, go to File Explorer > This PC > Remote Desktop Virtual Drive on RDWebClient > Download. 

  2. Paste the file you want to download into your local system. This will copy the file to Remote Desktop Virtual Drive on RDWebClient > Downloads.

  3. After the transfer is completed, a pop-up will appear in the browser with a Confirm checkbox. Click Yes to confirm the download.

  4. The file will be downloaded to your Downloads folder or the location where your browser files are typically saved. 

User Password Update

If you have forgotten your password, you can request Cloud Ops to reset it. Follow the steps below to change your password:

  1. Click on the User Profile icon and double-click My Account (this will redirect you to another page).

  2. Click Change password.

  3. Enter the old password, then enter and confirm the new password. Click Submit.

Session Timeout

  1. A session will remain active for up to 24 hours, with the possibility of variation based on the customer requirements. A pop-up will appear 2 minutes prior to the timeout.

  2. You will have 2 minutes to save your work before the session disconnects. After logging off, users will need to reconnect to the session.

Constraints

  • The maximum file size for upload is 255 MB. To upload larger files, use OneDrive or any other cloud storage-based solution.
  • The default storage in the setup is limited to the 'C' drive. If more storage is required, users can connect to OneDrive, which will be enabled in the environment.
  • Users will not be able to install any new applications in the environment and can use only the pre-installed applications.
  • A session will remain active for up to 24 hours (variations are possible based on customer requirements). A pop-up will appear 2 minutes before the timeout, and after that, the user will need to reconnect to the session.
