Changes Coming to Reports in December

The BI Reports tab in PLEXOS Cloud will be retired starting December 9th.

  • December 9th: Publish to Reports will be disabled. You will no longer be able to generate new Reports, but will be able to view previous Reports.
  • December 16th: You will neither be able to generate new Reports nor view previous Reports. The feature will be disabled.


Solution Reports will take the place of the BI Reports tabs. You can learn more about creating and modifying Solution Reports in PLEXOS Cloud .

Solution Report Enhancements

Save Solution Data as a Report from the Data Grid 

You can now save your ad hoc query from the Solutions Data Grid as a Solution Report. This feature allows you to capture the Solution Data as a Report, which can then be used later if you wish to analyze the same reported Properties for a different Solution. You can also generate a Report without having to go into a JSON file. 
 

This can be done through the Save as Report function. Navigate to the Data Grid and query the desired data from your Solution, then click Apply. You can further adjust your query and once happy with the results, in the top right-hand corner of your window, you can select the button for Save as Report. 
 

  NOTE: This functionality is currently available for only Daily, Weekly, Monthly, Quarterly, and Yearly reported data. 

 

After clicking Save as Report, you can enter the Report Name and Group Name. After entering the values, click Save. After clicking Save, the following popup will appear. Click on View Reports or the Reports icon to be directed to the Solution Reports workspace. 
 


 

You should now be able to view your Report in the My Reports section as shown below: 
 

 

To learn more, refer to the following article about Solution Reports in PLEXOS Cloud. 


Drilldown into Period Data in Solution Reports 

Users can use the Drilldown functionality to gain further insight into Solution Report data. If you are analyzing Generation for Monthly or Daily data and want a better understanding of when the Generation is occurring on a more granular basis, Drilldown can fulfill this purpose. 
 

After selecting a Report (from EE Reports or My Reports) from the Reports tab, right-click on a cell and Drilldown will appear as an option. 
 


If you click Week, you go can go down one level and view the weekly data for April 2022. If you click on Year, you can go up one level and view data for 2022. 
 


After clicking Year, you see the data for 2022 summarized in the table. You also have additional options: Back and Reset. Back will take you to the previous data view, which would be Month in this case. Reset will take you to the initial data view for the Report, which would be also monthly in this case.
 

View Included Output Properties on a Solution Report with View Details

You can now gather more information about the properties enabled in your Solution Report (JSON file) and Model. The View Details table will list the properties enabled in your Solution Report. If a specified property from your Solution Report is not present in the properties reported in your Model, the property will appear as red text in the View Details table.  
 

Right-click on a Report > View Details as shown below: 
 


The values in Red are Properties that have not been Reported in your model. To export these Report Details (the Child Class Name, Collection Name, and Property Name) as a .xlsx file, click Save.

 


 

New Supported Period Types

Daily, Weekly, and Quarterly Data is now supported for Solution Reports. This can be specified in your JSON configuration file with the following values for the periodTypeId: 

  • Daily: 1 

  • Weekly: 2  

  • Quarterly: 7 


These new period types are now supported in addition to those that were already present: 

  • Monthly: 3 

  • Yearly: 4

Excel Add-In Enhancements 

The PLEXOS Cloud MS Excel Add-In has added a new Custom Column feature in the Schema tab. This enables the user to modify custom columns using the Excel Add-In.  
 


Modify Custom Column values 

  1. In the PLEXOS Cloud Excel Add-In, load an existing Template and Switch to the Schema tab. 
     

  1. Add a new row to the Schema tab for a desired class (Lines, Generators, Nodes, and Regions) and fill in the details shown below for modifying custom column values: 
     

  Note: Leave the Parent and Collection columns empty. 

Column Name Description
SheetName Provide a Sheet Name as applicable in a Template. (Line, Generators, Nodes, Regions)
TableName Provide the Table Name as applicable in a Template.
 
ColumnName A Column Name appears as a Header Name of the Custom Column when you Load the Data.
PlexosEquivalent Enter the name of the Custom Column as provided in a Study (In PLEXOS Desktop).
Type Enter “CustomColumn” to let the Excel Add-In know that this is a custom column.
KeyType Enter “CustomColumn” to let the Excel Add-In know that this is a custom column.

 

  1. Save the template for future use (File > Save).  
     

  1. Next, select the desired Study from the Study dropdown in the right-hand pane of the screen. 
     


 

  1. Click on the Load Data button to load the data for the selected Study. 
     


 

  1. Switch to the Object Class tab (Lines, Generators, Nodes, and Regions) for which you had added a Custom Column. 
     

  1. You can see the Custom Column available with the data configured in the Study. You can now edit the data for that Custom Column. Make the necessary changes to the data. 

 

  NOTE: Currently, only updating the value is supported. You cannot Add or Remove any value but can only update an existing value. 

 

  1. After making the changes, click on the Save Data button to save the changes to PLEXOS Cloud and create a new version in the Version History. 
     
     

Study Management Enhancements  

The Study Management tab under Administration focuses on managing storage across all Studies. This tab is configured to display the list of all Studies including archived and unarchived. It allows you to Archive, Restore, and Delete single or multiple studies (up to 5 at a time).

Users can now view the Study Size (GB) on the Study Management page. You can leverage this information as an Administrator to decide whether to delete the Study or not. 
 

To view, navigate to the Studies tab on the Administration Study Management page. 

 


See Managing Studies in PLEXOS Cloud for more information. 
 

Simulation Management Enhancements 

You can now view a single KPI for all Distributed Runs (Monte Carlo (Auto-sample), Chronological Split Execution, and Monte Carlo) on the Simulation Management page. 

 


 

Universal Class Updates 

The Output Data Grid can now report the Units for Universal Class objects. Previously, the Unit field was filled with ~, but now users will be able to see the units. 

 

 

  NOTE: The correct units will only appear for new simulations. For older run solutions, the ~ will continue to appear. 

Study Dropdown Enhancement 

The Study Dropdown in PLEXOS Cloud has been improved to split your studies into further categories.

The Recent Study section shows the Study you have accessed most recently. Studies that are starred will be stored under Favorites. Others will include Studies which you can access but have not marked as Favorites. Studies for which you do not have access will be categorized under No Access.

Additional Improvements and Bug Fixes

There have been updates to improve user experience with Congestion Analysis. Lines on the map will be highlighted when selected to draw focus. 
 

In the Tenant Preferences page, fixed a Logo positioning issue by adjusting the height and the position in the given frame as per user requirement. Learn more about Tenant Preferences page.

 

Fixed an error “Invalid Locale” and updated the Locale Settings dropdown (In the Advanced Configuration tab) with the appropriate selection from the dropdown. i.e., the default will be English {en (US)}, but if a user selects Afrikaans, then the object will be updated to Afrikaans.

 

In the Administration >Single Sign On Settings page, updated the Add New Setting button to display the Create Setting popup upon clicking on it. The admin can now enter the SSO details for a new user in the popup.

 

For a successful Solution completed using Monte Carlo Auto Sample as a Distribution method, fixed a Stitching issue to allow users to Stitch two or more successful Solutions together.
