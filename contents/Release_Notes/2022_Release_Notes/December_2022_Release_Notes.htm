Solution Reports Enhancements

Enhancements to Drill Into functionality: 

Drill Into functionality can be used to gain detailed insight into Solution data by easily moving between time periods. 

For example, if you analyze Generation and want a better understanding of when the Generation is occurring on a more granular basis, ‘Drill Into’ can fulfill this purpose. 

 

With this release, we have updated the options for Drill Into. 

 

After selecting a Report (from EE Reports or My Reports) from the Reports tab, right-click on a cell, and Drill Into will appear, hover over that to see the options, shown in the following images. 
 


 

To view day-wise data, click on Day, it will show day-wise data for the complete month. 
 

To view the total amount of data in a month, click on Month.
 

You can also see data for an interval of time in a day, right-click on the cell of any data and then click Drill Into, and then on Interval, it will show you data available for that period. 
 

Likewise, you can see data for the year too. 

And, if you want to go back to a normal state (original state), click Reset.

Excel Add-In Enhancements

Users can now filter the data they want to see based on date. This makes it easier to focus on just the data they want to view with the Excel Add-In.

In the Schema tab of the PLEXOS Cloud MS Excel Add-In, a new “IsDateRequired” column cab be added manually. It allows the user to configure and view the input data between a specific date range for the required property.

Configure the IsDateRequired Column

To access and view the data between a specific date range,


1.  Go to the Schema tab in the template.

2.  Add a new column and name the column as IsDateRequired.
 
3.  In the IsDateRequired column, type “Yes” for each of the required property rows that you want to fetch the data.
 

 
4.  In the PLEXOS Cloud panel on the right-side of the screen, select the required Study from the Study dropdown list.
 

 

 
5.  Click on the Load Data button and wait for the data to complete loading.
 
6.  After the data is successfully loaded, you will see an additional tab created next to the main property tab where you entered “Yes” in the Schema tab.
 

7.  For example, if you entered “Yes” for the Lines and Generators properties in the IsDateRequired column, you would see the Linesproperty & Generatorsproperty tabs available next to the Lines & Generators tabs as shown below:
 

 

8.  Click on the additional tabs to view the data for the selected date range in addition to the Property name, Value, BandId, and Scenario columns.

 
NOTE: Currently, editing of data is not supported in this tab you can only view the data here.

Run Configuration Panel Enhancements

To help users Identify the Capacity of a particular set of Virtual Machines (VMs) upfront and to reduce the simulation failures caused due to storage limits, the Run Configuration panel has been enhanced to provide information on cores needed, Available Virtual Machines  (VMs), and Disk Space (in GB). See more details below:


Cores Information

In the Run Configuration panel, we updated the grey box (Run Information box) with cores information and a message to display when the selected Virtual Machine (VM) type exceeds the available capacity.

  • Cores Needed: Shows the required number of cores to Run the Model.
  • Cores Available: Shows the current number of cores available in your tenant (environment).


In the above example, if you select Number of Division as 3 and Virtual Machine Type as 32Core-256GB RAM-1023GB Disk, you will see the Cored Needed to Run the Model  are 96 and the Cores Available in your tenant are 64.

 

In this scenario, you will see the warning message, “The selected VM type has limited capacity. This run will queue and execute as VMs become available.”


Available VM information

In the Virtual Machine Type dropdown, the new chips were added to display the number of available VMs for every set of machines. There are three types of categories- Green, Yellow, and Red.

 


The calculation on which the color category will be decided is as below:
 

  1. Green- If the available number of VMs are more than what is currently being requested + 4, it will show in Green.
     
  2. Yellow- If the available number of VMs are between requested VM+4, it will show in Yellow.
     
  3. Red- If the available number of VMs are less than the requested VM, it will show in Red.
     
NOTE: As the Number of Machines Requested by the user changes, the chip color for the available set of VMs will change.


Disk Space Information

In the Run Configuration panel, Disk space information has been added in the Virtual Machine Type dropdown. This provides detail on how much storage is available on this VM. This is helpful for large or long running simulations that require more local storage.

Command Line Interface (CLI) Enhancements

You can now cancel simulations and run the Gurobi Tuner using the CLI. You can also view the CLI version and environment in log files as shown below. 
 

Cancel Simulation Requests 


To learn more about cancelling simulation requests, enter the following command: 

plexos-cloud simulation cancel --help
 


The simulationId is required input to inform CLI which simulation request is being cancelled. 
 

Gurobi Tuner 

The Gurobi Tuner can now be triggered using the CLI in PLEXOS Cloud. Running the Gurobi Tuner is typically a time and computationally intensive process. If your machine is not powerful enough, running the Tuner in PLEXOS Cloud allows you to utilize the available cores and tune your model. 

 

The simulationType should be set to GurobiTuner, with the following payload data provided: 

StudyId, ChangeSetId, SimulationOptions, SimulationEngine, Source, RequestedCpuCores, MinimumMemoryInGb, and SolutionData.


Learn more about the PLEXOS Cloud Toolkit (CLI).
 

Archive and Delete Solutions within Administration Study Management

 To help administrators manage storage in their tenant, a new tab is available under Administration/Study Management called Solutions. The Solutions tab is configured to display the list of all Solutions including archived and unarchived with its associated information such as Triggered By, Triggered At, Storage, and Execution Id. It allows you to Archive, Unarchive, and Delete single or multiple Solutions (up to 5 at a time).
