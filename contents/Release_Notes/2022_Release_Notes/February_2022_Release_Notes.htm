View a Summary of a Study (Study Overview)  

The new Study Overview page shows key information for all models within the selected study, it provides an at-a-glance view that allows you to see progress and version changes and quickly navigate to Model or Solution.

 

The key components of the workspace are:

  • View of key metrics:

This section shows key metrics of the study along with the description at the top that allows you to communicate the key goals of the study to other users.

 


 

  • Creator and Member section:

This section shows who created the Study and who all have access to the Study.

 

  • Permission/Access to Study:

 

Permission to different features in a Study is determined by the user’s license in the system. A Modeler has full permission to a Study they are a member of, while a Report Viewer can only view Reports of the Study.

 

You can add members to access your study by clicking on MANAGE TEAM.

 


 

 

Study Overview

Models:

The first tab of the Study Overview page shows a list of all Models in the study, number of simulations that have been run for the model along with which user ran it last, and on what date. The Run Horizon is also shown along with a RUN button.


This allows you to run the Model directly from the Overview page. You can only select and run one Model at a time. To run models in sequence please use the Models Workspace.

 

You can also apply sorting and filtering to each of the columns.

 


 

 

Version History

The Version History tab shows the list of all changes made to any object settings in the study. This view is similar to the one in Cloud Bridge.


 

The list by default is grouped by date, it can also be grouped by a member. You can also sort the list by clicking the down arrow.

 




Each version shows data as per the changes all the users have made, such as Adds, Edits, Removes, and Timeseries.

 

  • ‘Adds’ shows number of models added to the study.
  • 'Edits’ shows number of models that have some edits.
  • ‘Removes’ shows the number of models removed.
  • ‘Timeseries’ shows number of changes made using Timeseries files(CSV files)

Study Level Permissions   

A new level of access and security are available now in PLEXOS Cloud Feb 2022 release. When you upload a study through Cloud Bridge, it is now private and no one else can see it in PLEXOS Cloud. Only the creator can add other users as members of the Team and give them access to the study in PLEXOS Cloud interface, Cloud Bridge and Excel Plugin.

 

This means when you upload a Study through Cloud Bridge, it will be private, and no one else can see it in PLEXOS Cloud. Further, you can add other users as team members and grant them access to your study. The level of access is determined by the user’s license.

Access Another User’s Study through Cloud Bridge and the MS Excel Plugin

This feature is related to Study Level Permission. This ensures If you have permission to a Study, you can now access and download it from Cloud Bridge even if you haven’t originally uploaded it. This also means a user can view input and output data from a Study they didn’t upload using the Microsoft Excel Plugin (Beta). Note: Only users with a Modeler license have access to Cloud Bridge and the MS Excel Plugin.

 

Support for Hidden Parameters and Solver Tuner Files

Studies uploaded through Cloud Bridge and Simulations run on PLEXOS Cloud now support hidden parameters and solver tuner files. Now, you will be able to edit the "Undoc param files" (Hidden Parameter and Solver Tuner files) in PLEXOS on your desktop and can push the change using Cloud Bridge via Study Creation process (as initial upload) as well as versioning process.



 

We are making it more convenient than ever to review Studies and collaborate through all PLEXOS and PLEXOS Cloud channels. It's also a great way to segment Studies and keep data confidential. We hope you’ll find the new Study Overview and Study-Level Permission page an easier way to quickly see key metrics, study progress, and version changes for all Models.

 

Additional Bug Fixes and improvements

We have continued to fix bugs affecting user experience, stability, and performance, including fixes related to Cloud Bridge. Additionally, we have hidden the RUN button for in-progress Simulation on the Solutions page, added other usability enhancements in Cloud Bridge, and improved the PLEXOS Cloud user interface.
