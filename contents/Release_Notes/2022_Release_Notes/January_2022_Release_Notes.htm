Category Available in Models Workspace 

The Model Category is now available within the Models Workspace. This enables you to sort and filter your Models based on categories. By default, the Category column isn't visible from the Models Workspace. Click on the right arrow to view the Category column. 
 


 

 

When you Transform the view, the focus becomes Class and Scenarios on the right columns and the Categories will not be shown. This is by design so that when you switch mode, the classes become rows instead of columns and models become columns. Hence, adding categories to columns can be difficult. 
 



Clicking on the 'Lines' icon, as shown in the image below, provides additional options for Categories:

  • You can filter by Category or Hide certain Categories
  • You can manipulate the columns using Autosize, Pin Column, and/or Reset Columns
  • You can search for Categories and select the ones you want to use
  • You can also Sort Order on the Category column



 


 

You can create a Category, name it anything you want, and then click on Add and then Save.
 


 

Other users can select any Category from created categories as shown below.  There will be "-" as the default Category.


Congestion Analysis 

This tool helps you understand why congestion occurs on the system and how it impacts nodal pricing. You can drill down into a specific hourly (or sub-hourly) solution and query the data to show the sources and/or sinks that contribute to the loading on a given line.  Additionally, the contingency events that cause congestion can be highlighted, if needed.

After determining the causes of congestion, you can see how each congested component impacts the Locational Marginal Price (LMP) via the marginal cost of congestion at any desired node.

Enabling Cache Congestion Insight will inform the engine to create and store the data needed to support the Congestion Insight features.
 


After a successful run, you can see an icon highlighted in yellow (as shown below) to launch the Congestion Insight page. The Lightning Bolt icon highlighted in red indicates that this Solution has Congestion Analysis on it.
 


Once you click on the icon highlighted in yellow, you will be redirected to the Congestion Insight page, there will be three tabs: INDUCED FLOW DECOMPOSITION, CONGESTION CHARGE DECOMPOSITION, and NODE FLOW REPORT. On each tab, you can select a date, hour, and line object from the given drop-down list to view the reason behind the congestion.

Excel Plugin Enhancements 

Recently, we introduced the Microsoft Excel Plugin utility that allows leveraging Excel's many advanced tools for bulk editing. You can download the Excel Plugin directly from TOOLS in PLEXOS Cloud.
In this release, we continue to enhance the editing experience by adding new features, such as Scenario data editing and multi-banded editing.
Multi-banded variables are often used to model other variables that need multi-point data or to define a shape, such as 'offer quantity/price pairs' or a 'heat rate curve'. Similar to the PLEXOS desktop, the bands are displayed as a column of numbers in the data grid with the heading 'BandID'.

In the feature 'Scenario data edits', a new column named Scenario will be available in the data grid.  
The value in this column will be blank for base data. It will display the scenario name if it has the associated values for the variable. At this time, we do not support the creation of scenarios directly using the Excel editing grid.  
The priority order of scenario data is alphabetical unless overridden by using the Scenario priority attribute in the model setup. 




We also ironed out some bugs, performance issues, and improved the 'UI' to make the user experience more flexible, fluent, and familiar. For more details see Edit Input Data Directly in Excel with Tools Plugin.
