Analyze Output Data that is Outside a Normal Range using Event Detection  

Periods of a typical output data are called Events. Ideally, after a successful simulation, the user wants to quickly validate the output. This process can be quite labor-intensive and error-prone.

Event Detection can speed up this process by identifying and prioritizing time periods that may be of interest for further confirmation. We want to make it easier and faster to identify periods of time where output data falls outside of normal range.
Using Artificial Intelligence Machine Learning (AIML) techniques, we now offer Event Detection. From the Solution Summary on any successful Solution, you can view a heat map of Events and drill into them. This enables you to quickly identify interesting events that may need further study like Price Spikes, Price Drops, Unserved and Dump Energy.

Import and Export Shape Files   

We now support the modification of the geospatial data through shape files. Shape files allow you to edit the geospatial data to have accurate representations of zones, regions and pools, etc.

 

You can download a copy of the shape file and update the coordinates and upload the file back to PLEXOS Cloud. The update on shape files can be done by using any shape file editor (like GIS editor, Map shaper, etc).

 

The user can also upload a 3rd party shape file (Open-source shape files) and it’ll be rendered on the map.

 

To upload a shape file:

  • Go to Setup
  • Click on the Upload button on the far right.
  • Drag or browse to locate your file (Must be in zip format) and click Upload

 


 

To download a shape file:

Click on Download, you’ll get two options to download, Shapefile (Geocoded - contains shapefile which has geocoded data only) or Shapefile (All - contains shapefile which has non geocoded as well as geocoded data.)
 


 

Click on Edit to edit the uploaded shape file as per the requirement and then click on Save to save your settings.

 


 

 

Cloud Bridge Enhancements   

This release focuses on improving the workflow for users when moving data between PLEXOS and PLEXOS Cloud. Many new features make it easier and faster for users to accomplish key tasks such as running simulations and saving new versions.

 

Updated User Interface

Cloud Bridge now has 3 different parallel sections on the home page - Studies, Models, and Solutions. It will give you an overview of Studies, Models, and their Solutions altogether.
 


 

To see the number of Models in a study, click on any Study from the Studies column.

It will list all the Models associated with that study along with the date and time of the last run if available.
 

 

Once you click on any Model, it will show all the Solutions available for that model in the Solutions column with the status for each. This includes in-progress runs, canceled, failed, and successful runs.
 

   

  • If the Solution was successful, you can download the Solution log and Solution zip.
  • If you can't see models on any of the studies that means it has not been initialized earlier, to use that study, you need to download and initialize that first.
  • To do so, click on Download and Initialize project files.

 

  • Select the desired location on your machine to save the study and click OK.



 

 

  • Once the initialization is successful, you will see the following window, click OK to close.

 

  • You can also sort the model by date by clicking on the up arrow.

 

 

Run a Model Directly from Cloud Bridge

With this feature, you don't need to go to PLEXOS Cloud to update and run the study. You can do it all in one place. This feature will make the process of running a model easier and faster for you.

 

To run a model directly from Cloud Bridge, follow below mentioned steps:

  • From a selected study, go to Model and click on Run.

 

 

  • If a Model has a run history, it will prompt a window to run the Model using the previous configuration or customize the existing configuration.

 

 

  • Click on Run to run the model with the previous configuration, else, click on CONFIGURE if you want to change the configuration.
  • Once you click on Run, it will show the status of 'run' as shown in the screenshot below. Click OK to close the window.

 

 
  • If the Model has never been run, it will redirect to PLEXOS Cloud, where you can configure the selected Model and run it.

Create a Study from an Existing Version (Version History):

You can now create a new Study from an existing version of any Study.

Follow the below steps to create a Study from its existing version:

 

  • Click on Version in selected Study.

 
  • Click on the Create Study button in the version row from which you want to create the Study.

 

  • Enter the Study name and description of the new Study to be created and click OK.

 

  • A progress dialog box will appear that shows the progress of high-level steps that are taking place on PLEXOS Cloud.

 

  • Upon completion of the Study creation process, you will get an option to click on Launch Study which will open the Study Overview page on PLEXOS Cloud for this Study.

 

  • If you click Ok, it will take you back to the Version History page in Cloud Bridge.
  • To work locally (PLEXOS Desktop) with this newly created study, go to the Cloud Bridge home page and click on the Download and Initialize button of the newly created study.

Additional Bug Fixes and Improvements:

We have continued to fix bugs affecting user experience, stability, and performance, including fixes related to Cloud Bridge. With this release, we have enhanced the memory limitations on time series file changes. Cloud Bridge previously limited the size of time-series file changes (when updating a study) to 1GB. This limit now has been increased to 2GB. This should reduce upload errors for large updates.
