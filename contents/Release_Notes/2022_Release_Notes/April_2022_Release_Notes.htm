Integrated Resource Planning (IRP) Report

PLEXOS Cloud now includes the first out-of-the-box in application report that is based on a new configurable framework. This report is available in Solutions, known as Integrated Resource Planning or IRP, helps users better understand the economics of the LT plan, Capacity position, and Energy position. It is designed to provide analysts transparency into cost components, reduce post-processing, and streamline the IRP process. In addition to the IRP process in the United States, any LT plans can benefit from this report.


 

 NOTE: These reports are not supported for Monte Carlo or Stochastic runs.

Chronological Split Execution 

Chronological Split Execution gives you the ability to split the Model into pieces based on time such as year or month for parallel processing. This is an opportunity to leverage cloud compute to improve total simulation run time.

You will see two options in the Run configuration window – Single Run and Distributed Run.

 

To split the model simulation by horizon units, select the option ‘Distributed Run’

 

It will show the drop-down of the following methods:

 

  • Chronological Split execution

  • Monte Carlo samples

 

Chronological Split Execution:

It divides the splits by Horizon units– Months, Weeks, or Days.
With Chronological, select the number of splits that you need for the simulation. It can be any number above 1, but you should choose something that aligns with your Model and allows it to be evenly split. Then select the Horizon Units (Months, Weeks, or Days) and choose the machine configuration for Parent Virtual Machine and Child Virtual Machine.

 

Parent Virtual Machine coordinates the entire simulation and adds all the outputs together at the end.

 

And Child Virtual Machine supports each of the individual splits to run separately.

 

 NOTE: Chronological Split Execution is only supported with engine versions 9.0 R07 and above. It will not be enabled in the dropdown for earlier engine versions. Along with that, Solutions of Chronological Split Execution are not currently supported with the option "Publishing to Reports".


However, both of these limitations will be fixed in the upcoming release.

 




Monte Carlo samples:

It divides the machines to run a number of samples per virtual machine. It can be any number that varies from 2 to 999. In Monte Carlo samples you can select the virtual machine type and number of VM required to run independent samples per virtual machine. It enhances the overall simulation process.
For example, if you want to run 100 samples, you could provide 25 samples per virtual machine. In that case, you can select 4 virtual machines in parallel to make the execution faster.

 

 NOTE: For Monte Carlo Samples, a Stochastic Model must be enabled in the model.

 


 

 

Support for Read-Only and Report Viewer License Types

Administrators can now assign two additional license types to users. These licenses, Read-Only and Report Viewer enable access to certain functions within the tool that is different from the standard Modeler user.

 

The Read-Only user has access to view the Study Overview, Models, Solutions, and Reports for the Studies they have access to in PLEXOS Cloud. But they do not have access to run simulations or edit data.

 

A user with the Report Viewer license can only view the Reports section of the interface for the Studies.

Ability to view engine version and solver for all previous runs

The existing Previous Run Details feature now includes the ability to view all details across all previous runs for the selected Model. This is available from the Run Configuration panel launched from the Study Overview, Models, or Solutions workspaces.

 

When you click on Run, you will see the option to view previous run details, click on that.


 

You will get the details of all the previous runs of the selected Model.



 

Geocoding with the Excel Plugin

The PLEXOS Excel plugin includes a new feature to allow the user to review and perform bulk edits of the geographical coordinates (geocodes) of modeled objects. 

There is a new column labeled “Include Geocode” available in the template tab of the Excel plugin.  This column accepts the values “Yes” or “No” and toggles whether geocodes are retrieved for the specific table.  
When “Include Geocode” is set to “Yes” for a table and the user clicks on pull data, the resulting table will have an additional column called “Location” that contains the longitude and latitude information for the entity.  The user does not need to alter the schema tab to include these variables.  This data will be formatted in the format [±x.xxx, ±y.yyy], with longitude going before latitude.  The Location column can be edited manually or via intelligent Excel features such as lookup, concatenation, or autofill.  
The results can be saved directly back into the study when Push Data is called.  This feature will make setting up geographical information in the PLEXOS study faster and much more flexible.
