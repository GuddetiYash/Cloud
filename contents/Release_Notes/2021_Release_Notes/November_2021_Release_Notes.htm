Run Multiple Models in Sequence on a Virtual Machine

Select multiple Models, and then click RUN or right-click and select Run to run your Models in a specific order.
 

 NOTE: The ability to run multiple Models in-sequence is on a single (one) virtual machine.


The output of one Model can be used as input for the next Model.  Selecting Models is done by using Ctrl + Click or Shift + Click.
 



The Run Order column uses the Execution Order property from PLEXOS. It will be uploaded and shown as the first column in the Models workspace known as "Run Order." If you don't have a "Run Order" for your Models, you can use the "Fill Run Order" button. It will put a value from 1 to n based on the current order of your models in the grid. You can change the order by editing the run order cell, and once you save the new order, it will be used going forward. If you don’t care about the order, you can select Models and run them.

If you select multiple Models, the Models will run the smallest Run Order first and continue to the largest Run Order. They will run in the order defined in that column. Because this is a group of runs, Recommendations and Previous run details will not be shown. 

If you want to run multiple Models in parallel, you should continue to select them individually, and then run them one at a time.  A new virtual machine will be used for each simulation.  No parallel runs are supported. Stochastic/Monte Carlo runs will use their Model-specific risk sample count.  

Gurobi is the default solver. This can be changed in Advanced Settings. You can select multiple solvers. All runs will complete with one solver and then start with the next solvers and so on.  
   

 NOTE: A select model must have a run order to run. If you select multiple runs with blank or duplicate run order values, you will receive an error.

 

Compare Output of Two Monte Carlo Runs

From the chart analysis page within Solutions, you can now compare the output of two different solutions. These can be two solutions within one model or two solutions in different models within the same study. This allows comparing the metrics for different solvers, input data, scenarios, and more. This functionality is very helpful in risk mitigation.
 


 

You can select any successful Solution from any Model in the Study.
 


Simulation Recommendations with New Design and Cloud Burst Support

There is an improved look and feel to the Simulation Recommendations and Cloud Burst Support to enable simulation to run on Cloud with one click and to better summarize recommendations.
 



 

Run a Simulation Directly from Solutions

To improve your workflow, Solutions now lets you run a simulation for a Model that you previously ran. You don't need to go back to Models.  You can find the model, click RUN directly from the Solutions card, and fill out the configuration.



This feature uses the latest data (including any changes that were made after the run) and defaults to the last configuration.  Recommendations and other features will work the same way as they do in the Models workspace.

Diagnostic mode is an easy way to gather key details to share with the Energy Exemplar support team when you have questions about a simulation.  Diagnostic mode is turned off by default and can be turned on or off in User Preferences.

Consolidation of Commodities into Single Publish Reports

Now you can do a single click to trigger "Publish to Reports" for all commodities (such as electricity, gas, etc.). To simplify the process of publishing Solution data to reports, we have combined these different workflows into one.  



Map Enhancements in Setup and Solutions

Support for Regions and Pools

Setup and Solution maps now support Regions and Pools in the same way as Zones. In Setup, you can create polygons to visually represent PLEXOS Region and Pools on the map.  Region and Pool outputs can be displayed in Solutions as well.
 

 

Delete Geocodes for an Object Including Zones, Regions, and Pools

From Setup, to remove geocodes for an object, you can click on an object hit the delete key and the geocode will be deleted without deleting the object.  After clicking SAVE, you can find the object under non-geocoded in the filter panel.



You can also delete the geocoded polygon for a Zone, Region, or Pool. Delete either just a point in the polygon or the whole polygon.  Currently, polygon delete functionality is not currently supported through the csv export/import.
 

Improved Generation Output

We have optimized the Generation Output feature in Solutions by updating the bars to reduce the width and it now supports the ability to hover your mouse over objects to see detail.



You can zoom out to aggregate data and zoom in to see details. 
 



When generators are at the exact same geocode (latitude and longitude), the bar will show +n when n is the number of generators.  You can use Previous/Next to move through generators using the object inspector.


Global Share Button

The Share button enables you to easily share a link to PLEXOS Cloud.  The button shares a link directly to that page in the study. Security is enabled, so if a user is not licensed to view PLEXOS Cloud or that part of the application; or if they don't have access to the study, they will receive an "access denied" message.


 

New Active Licensed Users Login Chart for Administrators

In Administration-Subscriptions an Administrator can view a chart to see unique users that have logged into PLEXOS Cloud over the last 7 or 30 days broken down by license (Modeler, Report viewer, Read-only, Report Designer).


 

 NOTE: Report Designer is an add-on license. A user that logs in that is a Modeler and a Report Designer will show as 1 Modeler and 1 Report Designer on the chart as they are actively using two types of licenses.

 

Cloud Bridge Improvements

With every release, we continue to improve Cloud Bridge to make getting data back and forth to PLEXOS Cloud easier and faster. In this release, we made the following enhancements:

  • Improved validation flow to prioritize errors that would cause the upload to fail earlier in the validation process.
  • Adjusted file referential integrity so a user can upload only needed time series files for the Models they plan to run versus all the required files for the study. 
  • Improved upload progress reporting that shows both completed steps upcoming stages in order.
  • Enhanced performance and memory usage.


Microsoft Excel Plugin - Improved Performance and Ability to Analyze Output (Beta)

Performance improvements of up to 50% when pulling input data, and up to 80% when pushing times series changes. You can analyze output directly in Excel, and with a click of a button, you can load output data into Excel to analyze and leverage it using Excel tools and charts.  


Bug Fixes

We have made many bug fixes that affect stability, performance, and user experience, including fixes related to Cloud Bridge.  Additionally, failed simulation run will now show the log file from the Solution card.
