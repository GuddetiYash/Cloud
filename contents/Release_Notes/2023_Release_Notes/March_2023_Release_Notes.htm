Save Selected Properties and Format on Solutions Data Grid for Fast Reuse with Layouts

The Solutions Data Grid page allows you to save the current settings of your ad-hoc query for reuse. You can save the current view as a Layout and store the selected Properties, Objects, Phases, Periods, and more. 

Layouts will save time spent on selecting the same fields every time you evaluate a Solution.



Learn more about Layouts here.
 

Output Data Grid – Faster Performance and Major interface Improvements

We continued making improvements in the Output Data Grid page under Solutions in this release.


We have enhanced the data loading process in the grid. It now supports loading 100k records in just a few seconds. Additionally, we reduced the need for scrolling by adding the Show More Data button to load up to 100k more records at a time. The grid now supports up to 1 million records.

 





Learn more here.
 

Distributed Parallel Models

We have introduced the Parallel Models distribution method in the Distributed run type, which supports the running of multiple models simultaneously.


Select multiple models with Ctrl + Click or Shift + Click and then select a single VM type that will be used to run these Models. Each Model will be run on its own VM in parallel.

 

This saves clicks and ensures timely execution.


 

Enhancements in the UI of the Run Configuration Panel

In this release, we have made some exciting improvements to the Run Configuration Panel. We made the options for Run Configuration less complex and easy to use.


Now, you don’t need to switch tabs back and forth to see the Advanced Configuration and Input settings. All the settings will be shown in a single window.


In addition to that, now Tuning operations can also be performed by selecting various VM configuration types.


Next, we also enhanced the visibility of seeing previous run details by segregating the information individually as per the Run types.

Excel Add-in Enhancements

Editing on Attribute Data

The MS Excel Add-in now allows you to perform editing on attribute data.
To fetch attribute data and perform editing on it, you need to edit the Schema table as highlighted below.



After you click on Load Data, the Attribute value (Horizons) for the mentioned attribute (Step type) will be pulled into a new column with the header name being the columnName from Schema.
Click on Save Data once you finish editing and adding attribute values to save the new attribute value.

NOTE: Latitude and Longitude attributes will not be supported for any object.
 

Ability to modify Model-Scenario membership, Model-Horizon membership   

Users now can pull data from the Model, Scenario, and Horizon classes and also perform editing on them. 
For Model-Scenario membership a different sheet will get created with name = "ModelScenario" where '0' means there is no relationship between the Model and Scenario object and '1' means there is a relationship between them. 
If you want to edit these you can choose values from 0 and 1 which will create a relation between them.

For Model, Scenario, and Horizon there will be no BandId and Scenario columns which were created by default for the rest of the classes.

The Solutions tab now shows only successful Simulations. This keeps users focused on the Simulations where Solution files are created and can be evaluated using the map, output data grid, Solution Reports, and more. 
The Simulations tab continues to show Simulations in all states including in progress, failed, or canceled.

This update will help users easily find successful Solutions without using filters.

Enhancements to BI Analytics Dashboard

We have continued making improvements in the BI Reports. In this release we added some great options for BI Reports users including Save As(Copy), Delete, Rename, and Share BI Reports.
We added a ‘Shared Reports’ folder in addition to My Reports and EE BI Reports folders. The Shared Reports folder will contain all public BI Reports that were created by others.

Solution Reports - Split a Group by Category

In Solutions Reports, with the addition of the Auto-split feature, you can report each individual category or object in the system by just enabling a flag. 


You no longer needed to hardcode object names in reports. This enables Solution reports to be Study agnostic.

Previously, with Solutions Reports, users need to manually enter each individual filter name, such as objects, and categories, to create their report. 


And for a different study, they may need to change the names manually again. It also makes the OOTB (EE Reports) reports not generally applicable to the user's study since the filter names are manually entered.
 

CLI Enhancements

"createDirectory" command is no longer necessary

When downloading a solution file, you no longer need to use the "--createDirectory" command. Instead, solution data can be directed to be stored in the desired output directory without it having to be instantiated. This removes one more step and improves the workflow when downloading solution data. The command can now be called as shown below:
 

pxc solution files download --solutionId f22937c5-d263-4084-83ca-aa2cbd81ad5b --outputDirectory "PathToOutputDir”
 

Hybrid to Parquet Conversion Update

Previously, after hybrid files were converted to parquet format, the files in the source parquet directory were deleted. Now, the source files are successfully converted to the destination folder and the files in the source parquet directory are not deleted. To observe this behavior, create a folder to store the output and enter the command below:
 

pxc solution convert hybrid-to-parquet --sqlitePath “DatabaseFilePath” --parquetDir “PathToParquetDir” --outputDirectory “PathToOutputDir”

 

Viewing the Number of Geocoded Objects

You can also now view the number of geocoded objects in a study. This is helpful when trying to determine if you have forgotten to geocode an object in a specific class. Enter the following command to extract this information:
 

pxc study stats geocoded-objects --studyId f22937c5-d263-4084-83ca-aa2cbd81ad5b --changesetId cd437e2d-e84f-4df1-b11c-3927e3b4969c
 


The "studyId" and "changesetId" are both required input parameters for this command.
 

Live Data When Downloading Solution Files

When downloading Solution files, you can now view live status updates as your download progresses. This is helpful in the event that a file size becomes too large while downloading, or if you want to know which file is being downloaded and its size. A sample command is shown below:
 

pxc solution files download --solutionId 64e2498f-7641-4d9f-9041-79fcd2041d68 -d "PathToOutputDir" --type Log
 


The solutionId and outputDirectory are both required input parameters for this command.

 

Return a List of the Studies in a Directory

A new command has been added to return a list of studyIds in a given directory path. This can help improve workflows and reduce confusion when you are working with multiple XML files stored in the same folder. The command can be called as shown below:
 

pxc study list-local-studyIDs --studyDir "PathToStudyDir"
 


 

Solution Download Improvements

Previously, if you needed any single file or subset of files in a zipped solution file, you would need to download the entire zip archive to your computer and extract it just to get the files you need. There is now a new option on the `solution files list` CLI command, `--includeArchiveEntries`. This allows us to now expose what files are inside of zip files. For example:
 

pxc solution files list --solutionId 6c5406b9-0a24-4ad8-8a1e-2edea6a7de40 –type Raw –includeArchiveEntries




Using the existing `solution files download` command with the existing --file option, we can now download just a single entry from out of the zip archive by passing the fileName of the entry given back in the above `solution files list --includeArchiveEntries` command. For example, if all I want out of this sizeable Solution.zip file is the small PLEXOS_Param.xml file:
 

pxc solution files download --solutionId 6c5406b9-0a24-4ad8-8a1e-2edea6a7de40 --type Raw --file "Model Stigler Diet Solution.zip/PLEXOS_Param.xml" --outputDirectory . –-automation

API Updates

When querying simulations through ODATA, the expand retries option will return a sub-list of simulations that were the retry attempts for the parent simulation. This allows users to see if there were any related entries and improve troubleshooting workflows.  This option can be added by adding the "?$expand=Retries” query parameter to the URL. You can see a sample output below:



The id, status, execution, and retryCount of the retry attempts for the parent simulation are returned.
 

Additional Improvements and Bug Fixes

We have continued to make improvements in the UI and fix bugs affecting user experience, stability, and performance of the PLEXOS Cloud. 

With the continued improvements in Desktop to Cloud Integration, we have now provided the capability to work on multiple tabs while a study upload is in progress. Users can now switch to other open tabs such as Solution, log files, etc. Download PLEXOS 9.2 or newer to access this enhancement.

If you are using the parquet Solution output with PLEXOS Cloud, please note we have added one additional column in the “fullKeyInfo” table that may impact your workflow. The column added is named “PeriodTypeName” which denotes period types in PLEXOS and is used by BI Analytics.

With Excel Add-in, we have enhanced the capability of loading the data in one go. Now, you can be able to pull data from large datasets up to 8000 rows of data at once. 

The Solution Report Details screen now shows user-friendly names instead of just ID numbers.
The Diagnostics feature, accessible from the Solutions card under the kebab menu (three vertical dots) now provides additional details including:
 

•    tenantid
•    environment - URL of the page
•    changesetid
•    studyid
•    timestamp - when was the popup opened/copied to clipboard
•    userid


This information is helpful when communicating issues to the Energy Exemplar support team.

In bug fixes - A bug that changed the study solver from CPLEX to Gurobi when uploading to Cloud has now been resolved.  
The ‘PLEXOS-cloud simulation monitor’ command now correctly reports the status for simulations.
