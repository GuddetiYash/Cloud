Important Notice - End of Support for Cloud Bridge

Starting January 2024, Energy Exemplar will no longer support Cloud Bridge. This means the end of technical support, security updates, and ensured compatibility of Cloud Bridge with new releases of PLEXOS Cloud and related products. To facilitate a smooth transition, we encourage you to use the PLEXOS Cloud-Desktop Integration, which is equipped with all the features present in Cloud Bridge.
 

Solution Output Analysis (Map, Grid, Reports)

Support for All Classes on Map View

The Solutions Map Properties Panel and the Setup Map now accommodate all available classes, eliminating the previous limitation of displaying only a subset of classes on the Map. You can see the list of supported classes, which includes all classes in PLEXOS Cloud, from DATA > SETUP > Edit. 

 NOTE: This does not include Membership Classes (e.g., GasStorage.GasContracts).

The following image shows the Map within Setup showing the additional supported Classes.




You can also view these Classes on the Solutions Map. They are represented by the same icons that are visible on the Setup Map.

Seamless Solution Transition with Persistent Grid Context

Improvements have been made to the user experience on the Output Data Grid Page when switching between Solutions from the left panel. The Data Grid retains its context and smoothly reloads the updated data from the selected Solution.  The Data Grid will preserve the selections, the time intervals, sorting settings, filters, and orientation. This supports switching between Solutions in the same Model or different Models.

 NOTE: If the phase selected by the user does not contain any Solution data, the corresponding section will appear empty on the Data Grid.

 

Model Building and Editing (Desktop Integration and Excel)

Browse Cloud and Clone Study in Desktop

The new Cloud Study panel and Clone functionality in PLEXOS Desktop allow users to browse and find available studies on PLEXOS Cloud and Clone a Study for editing in PLEXOS Desktop. Cloning creates a local copy of a Study on a user’s PC, complete with version history, so that users can start to collaborate on the Study in PLEXOS Desktop with integrated version management features.

 NOTE: The Clone Study feature is available with PLEXOS Desktop 9.2 R05 and above.

Refer to the detailed article PLEXOS Cloud Integration with PLEXOS Desktop for more information.

 

Create a New Study from an Existing Study Version

Users can now save any version of a Study as a new Study. This action allows users to take a specific state, or version, of their Study and create a new one. This is useful if many changes were made that users aren’t happy with, and they want to revert to an earlier state. This functionality previously only existed in Cloud Bridge.

 NOTE: The existing Study will not be changed. This creates a brand-new Study from an existing version. If the existing Study is no longer needed, make sure to Archive or Delete it to manage storage.



Refer to the detailed article Managing Study Versions in PLEXOS Cloud for more information.

 

Logging Data from the Excel Add-In

The Excel Add-In now includes a Log sheet that allows users to track and view the modifications made to a Study when data is pushed or pulled. A new Log worksheet is automatically generated each day when a user loads a Study for the first time. The Log sheet provides information such as Timestamp, Type, Message, Version, Environment, and Operation for data pushing and pulling. This feature enables users to monitor, identify, and report any problems that may arise during their Excel Add-In activities.

 NOTE: The Log sheet specifically tracks the changes made in the DATABASE and DATA FILES tabs within the Input Data section.



Refer to the detailed article on the PLEXOS Cloud Microsoft Excel Add-In  for more information on Log sheets.

 

Tag a Datafile to a Property with the Excel Add-In

The Excel Add-in now provides support for Data File references for a Class Property, which is used to define the value of a Property during a PLEXOS simulation based on time-series data stored in the referenced Data File. Excel Add-in users can now define Properties that reference Data Files in the Template, and then add/update/remove Data File references in the Property tables.

This will help users improve their workflow while making bulk edits to Properties.

Refer to the detailed article on the PLEXOS Cloud Microsoft Excel Add-In  to learn more about how to tag a Datafile to a property.


 

Add a Row/Column to Existing Datafiles in the Excel Add-In

Users can now add multiple rows and columns in the Timeseries File sheet loaded under the DATA FILES tab of the Excel Add-In. Users have the flexibility to configure these rows and columns as required. They can manually set a Header name and assign property values, making the sheet more flexible and easier to read. Previously, users were limited to adding any new rows and columns to existing Data Files.

Refer to the detailed article on the PLEXOS Cloud Microsoft Excel Add-In  for more information on adding data rows and columns.


 

Model Execution

Simulation Groups

A Simulation Group is a set of Simulations that run together in a batch. Users can create and modify these groups, eliminating the need for manual selection of multiple Models each time they wish to run them. By simply selecting a predefined group and clicking "Run," users can execute all the Models within the Simulation Group effortlessly.

Each Simulation Group is equipped with its own settings, which define the Models, Run Configuration Settings (such as Run Type, Virtual Machine, etc.). When triggered, the Simulation Group executes each Model on separate Virtual Machines simultaneously or sequentially (based on Simulation Setting), utilizing the specified Simulation Group settings.






You can create a Simulation Group in one of the following ways: (1) through the "Classes" tab on the Models page or (2) via the Simulations page.

Learn more about creating and executing Simulation Groups in the article Simulation Group in PLEXOS Cloud.
 

Run Configuration Panel Changes to Show Recommendations

In this release, significant changes have been made to the UI of the Run Configuration Panel to enhance visibility. The panel now includes two cards to show the Previous Configuration, and Recommendation along with a Custom Run Configuration.
 

1. Previous Configuration - The Previous Configuration card displays the configuration details of the previous successful Run. Users have the option to select it and proceed with that exact configuration.



 
2. Recommendation - The Recommendation card provides a suggested configuration tailored to the user's Model, aiming to optimize core usage.
 
3. Custom Run Configuration - The Custom Run Configuration card functions as the traditional option, allowing users to view and adjust their settings according to their specific requirements.


 
Additionally, the Panel now offers the capability to add Models to a Simulation Group. Users can either create a new Simulation Group or add their Model to an existing one. For a graphical representation of the steps, please refer to the following GIF.



 

BI Analytics

 NOTE: The features described in this section related to BI Analytics are only available to users that own the BI Analytics add-on with the core-based licensing or the Analytics Tier with usage-based licensing.

BI Solution Comparison (Beta Release)

Under the Compare page users can compare different Solutions using BI Reports after completing the "Add to BI" operation for the Simulation Group.

To facilitate this comparison, users need to create a Simulation Group. Once the Simulation Group runs successfully, it will be available in the Compare page. After adding the Simulation Group to the BI, users can compare the data of different Solutions of Models within that Simulation Group. Each group allows for a minimum of two and a maximum of six Solutions to be compared. This powerful feature enables users to analyze and compare results across different Solutions.





Learn more about the new feature in the article BI Solution Comparison.
 

Remove a Simulation Group from BI Analytics

If you want to remove a Simulation Group from BI, simply click on the Remove Solution Group from BI icon.

 

Manage BI Solutions

Users can now better manage their Solutions that have been added BI. Users are limited to ten (10) Solutions added to BI Analytics at any time, and other users within the tenant can add different Solutions and all users can view all Solutions added to BI. 

This new feature empowers users to review the Solutions added to BI, and if they wish to remove them, they can do so by using the "Remove Solution/Solution Group from BI" feature. This allows users to reduce space and regain the option to add new Solutions to BI.



 

Administration (Simulation Management and Storage Management)

Retain Selected Filters in Study and Solutions Tabs in Storage Management

In the Studies and Solutions tab of the Storage Management page, applied filters will be retained even if the browser or grid refreshes. 

Previously, users had to manually reapply the filters after saving the data or when the browser or grid refreshed. With this improvement, users can seamlessly continue working with their preferred filters without repetitive manual adjustments.



 

Email Notifications for Saves as New Study

Users will receive email notifications indicating the Successful or Failed status of the Study creation process for the Save as New Study action. Additionally, users can view an activity “New Study Created Successfully" or "New Study Creation Failed” added in the Activity History panel of an existing Study (Source Study) under the Overview tab.





Integration and Automation - CLI/API

There have been no changes to PLEXOS Cloud APIs or the PLEXOS Cloud Toolkit (CLI) in the June 2023 Release of PLEXOS Cloud. Please return in July for any changes or enhancements.
 

Additional Improvements and Bug Fixes

We have continued to make improvements to PLEXOS Cloud, with a focus on enhancing the user experience, stability, and performance. 

The CI Select Node Object under the Congestion Insights page has the option to Search the Nodes and apply a filter on the results.

Additional design enhancements have been made in the BI Reports section, specifically related to changes in wording. The term "Add" has been used instead of "Publish," and "Remove" has been used instead of "Unpublish."

The Credit Usage Trend on the Subscription page will now have a "Show All" option in addition to the existing 7 days, 30 days, and 60 days trends for “Capacity-Based Licensing."

Important Note for PLEXOS Desktop Integration and Cloud Bridge
We recommend that users leverage the PLEXOS Desktop Integration and the CLI -OR- Cloud Bridge for interacting with their Study. Going back and forth between Cloud Bridge and CLI-based integrations for one Study (desktop and the command line) can create versioning issues in rare circumstances.
