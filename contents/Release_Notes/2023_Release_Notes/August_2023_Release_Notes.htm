Solution Output Analysis (Map, Grid and Reports) 

Executive View – Communicate easily with decision makers directly from PLEXOS Cloud

Executive View has been introduced in PLEXOS Cloud, it offers a user-friendly interface specifically tailored for managers and executives. This interface allows decision makers to conveniently review reports that are shared by Analysts within their tenant. The Executive View provides Analysts with a way to collaborate with the decision makers within their Tenant. This way they can focus more on modeling and take help from Executives on important decisions. The Executive View is included in the Analytics bundle of PLEXOS Cloud, it is not available with the Simulate bundle. There are two distinct flows for the Executive View – Publish and View (Review). In the Publish Flow, users with an Analyst license type are able to publish their work specifically for decision makers (Executives). The View Flow presents Reports shared by Analysts, which are accessible by the decision makers (Executives, Reviewers and Analysts) that they can utilize to make informed decisions.

Administration (Simulation Management and Storage Management)

View Consumed Compute Credits of your tenant

We upgraded the Subscription Details section under the ADMIN page to show the key metrics for your Subscription with the addition of Tenant Name, Purchased Compute Credits (Referred to as Owned Compute Credits previously), and Consumed Compute Credits. These metrics will help Tenant Admin to have better clarity on the number of Credits available in their tenant, so they can plan to order or purchase additional Credits in advance.

 

  • Tenant Name: Shows the name of the environment you are currently logged in.
  • Purchased Compute Credits: Shows the total number of Compute Credits purchased for the current license duration (For Compute Credits).
  • Consumed Compute Credits: Shows the total number of Compute Credits consumed till today's date from the start date of the current license duration (For Compute Credits).

 

 

NOTE

  • Credits deducted for any Simulation Run would be deducted from the Available Credits and added to the Consumed Credits.
  • Credits refunded for any Simulation Run would be deducted from the Consumed Credits and added to Available Credits.

 

View the Transactions of Compute Credits of your Tenant

NOTE: This section is only visible to Admin users.

 

To improve visibility into Compute Credits usage, we integrated the TRANSACTION HISTORY, USAGE, and USAGE TREND tabs under the Compute Credits Summary section of the Subscription page. This will help users better understand the usage of the Credits in their Tenant. Previously, these sections were separated, and users had to navigate to different areas to view the Compute Credits Data.

 

 

We have also added new columns in the grid and an Export feature to export the data in either CSV or Excel format. Please find the details as mentioned below:
 

  • Transaction Date:Shows the date when the Compute Credits were Purchased or Refunded.
     
  • Start Date: Shows the date when the Compute Credits transaction would come into effect.

  • Expiry Date: Shows the date that the Compute Credits will end and no longer be valid for use.

  • Type: Shows the type of transaction made such as Expired, Purchased, or Refunded. (Use the Column action to filter or sort)

 

 

NOTE:

If you have unused Compute Credits remaining after the Expiry Date, they are regarded as Expired Credits. These Expired Credits are treated as Expired transactions and showcased in the grid. For these “Expired” transactions, the Start Date and End Date columns will not show any dates. You can consider the Transaction Date as the Expiry Date in this case.

 

  • Transaction ID:  Shows the unique ID generated automatically for the specific transaction or a Support Ticket Number you raised with the Support team.

 

  • Export:  Click on the Export button in the top right corner of the Compute Credit Summary section to extract the data in either CSV or Excel format. You can use this data to share amongst your organization for analytics or historical reference.

 

 

Admin can assign the new Executive license to tenant users


NOTE: The Executive license is visible to customers on the Analytics Bundle only. Simulate Bundle users will not have access to this license type.

The Executive stakeholders have non-execution licenses available in blocks of five, with pricing tailored accordingly. Executives with this license cannot perform Simulations but have access to view all Output Data and Reports. However, Executives are unable to Edit the Input Data, Run Simulations, or Generate Reports.

A new Executive license is available in the Add License dropdown under the ADMIN > Users > Add Users pane. This new license type empowers Managers and Executives to assess and offer input on the reports that the Analyst shares directly through the Cloud platform. Users with the Executive license can now look at the shared reports with them, review and work with Analysts to improve them. This license further extends the capability for Managers and Executives to actively participate within the Cloud environment.

Currently, 5 Executive licenses are included with the Analytics bundle subscription.

 



 

Integration and Automation - CLI/API

Timeseries Changes to Support Large CSVs

The tracking and application of changes in timeseries files have undergone enhancements, resulting in a significant improvement in the efficiency of pushing and pulling changes from the cloud, especially when dealing with large timeseries files. Due to this, Studies created from Cloud Bridge or before CLI version 1.0.752 will go through an upgrade process when accessed through the August CLI release (version 1.5.417) which will render them inaccessible to older Cloud Bridge and CLI versions. Due to this upgrade process, Timeseries CSV changes that were pushed before the August release of the CLI will not be available for viewing change details on the website. If multiple team members are working on a study that was created before April 1st, it is recommended that all team members upgrade to the latest CLI version before continuing their work on the study.

New Command for Retrieving List of Solution Reports with Report IDs

A new command is now available to obtain a list of solution reports along with their corresponding report IDs.
To utilize this functionality, simply use the following command:

 

pxc solution list-reports

 

 

This command fetches the comprehensive list of solution reports, each accompanied by its unique Report ID and Report Name.

To learn more about this command and using the CLI, refer to the detailed article Introduction to the Introduction to PLEXOS Cloud Toolkit (CLI).
