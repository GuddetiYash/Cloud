Solution Output Analysis (Map, Grid and Solution Reports)

Maintain Time Periods When Switching Period Types on the Solution Map or Grid

NOTE: This change is applicable for Analytics Tier customers.

 

You can now easily navigate between various time periods when exploring Maps or Grids to analyze the output. 

This allows you to seamlessly switch between broader, comprehensive views and delve deeper into specific details. When transitioning between period types (such as hourly, daily, weekly, monthly, quarterly, or yearly), the system remembers your last selection. 

This means that as you move between periods on the Map or Grid, the system uses your previously selected time as a starting point, ensuring a smooth and consistent user experience even when switching between Map and Grid views.

 

For Example: If you were viewing Yearly data for Jan 1st, 2023, and decide to switch to Monthly  intervals, the system will not automatically adjust the date; it will remain the same so that the user can change their preferred date.


New Default Solutions Map State for Improved Performance

The Solutions Map now opens with no input objects selected or displayed. This design empowers users to choose precisely what they wish to view on the Map. This adjustment is particularly beneficial for large Models where loading objects can be time-consuming, resulting in a notable enhancement in page load time by not loading objects that a user may not want.

 

To make the most of the Map and load exactly what users need quickly, users can take advantage of Map Layouts. By saving their preferred views of the Map in a Map Layout, they can conveniently load all input and output objects with a single click, bypassing the need to individually select items from the Properties Panel.

Configure the Default Solutions Page Based on User Preferences

 

You can now set the default page that will load when you click on Solutions in PLEXOS Cloud. Users can select Map (the current default), Grid, Solution Reports, or BI Analytics. This new feature is available from the Display Settings under Preferences. This option gives users the flexibility to choose their desired view.


To change the default page –

 

1.Click on your username in the top right.

2.Navigate to Preferences and scroll down to Display Settings.

 

 

3.Select the desired page: Map, Grid, Solution Reports, or BI Analytics.

 

 

4.The change will be reflected in your next login. When you open the Solutions area, it will open the page you selected.

 

Editing Input Properties and viewing Connections for Map Objects on the SETUP page

NOTE: This change is applicable for both Simulate and Analytics Tier customers.


You can now edit the Input Properties for any object available on the map from the SETUP tab of the Data page. You do not have to Run the Simulation to view or edit the Input Properties and view Connections (Map or Bus diagram) for the selected object on the map. This feature saves time by allowing you to edit the Input Properties and view Connections on the SETUP page before running a Simulation. Previously, you could edit the Input Properties and View Connections for the selected map objects in the Map and Grid tab of the Solutions page.


NOTE: When you make changes to the Input Properties and click on the APPLY button, the new version of that Study gets created. You can view and manage the Study version in the VERSIONS tab.

 

 

Learn more about  Make Changes to an Input Datasets. 

Enhanced User Experience in Executive View

NOTE: This change is applicable for Analytics Tier customers only.
To optimize usability and provide a more user-friendly experience, we have made a series of improvements to the Executive View page. These improvements are designed to empower users with increased usable space and enhanced visibility of key information. Here are the key changes:

  • Increased usable space:
  • The View Description now appears only when you hover over the View name. Previously the View Description was always displayed above the tabs within the Executive View.
  • The Report Description is hidden by default and only displayed if available. Previously the Report Description panel was always shown even if empty.
  • The View Name is now collapsed by default.

 

Learn more about Executive View in PLEXOS Cloud.

BI Analytics

AutoSave BI Reports Periodically

NOTE: This change is applicable for Analytics Tier customers only.

Auto Save functionality is now available in BI Reports for a more seamless user experience. This feature ensures periodic saving of BI Reports to reduce the risk of accidental data loss or overwriting data. 
The default Auto Save interval is set to 10 minutes, automatically preserving your edits while in Edit mode.

 

 

Learn more about this feature in the article for BI Analytics in PLEXOS Cloud.

 

Model Execution

Support for Users to Remove themselves from a Private Study

NOTE: This change is applicable for both Simulate and Analytics Tier customers.

 

Users can now leave a Private Study if they no longer want to be part of it or receive any notifications (notifications can be managed from the Preferences page). This will help users better focus on the Studies they are working on and avoid distractions by being added to unnecessary Studies.

 

 

If you are the Owner of a Private Study, you can leave a Study only after you assign someone else as the Owner of that Study. Please note that users cannot leave a Public Study. 


NOTE: If you leave a Study marked as Favorite, that Study will be removed from the Favorites section of the Study dropdown.

Model Building and Editing (Desktop Integration and Excel)

Manage triggering new Simulations with zero Credits for Trial Tenants

NOTE: This change is applicable for both Simulate and Analytics Tier customers.


Trial Tenant users cannot trigger a Simulation when their available Compute Credits are zero. However, any Simulation Run before the Compute Credits are zero, will continue to Run and Complete, but users cannot Run a Simulation once that Simulation finishes. Tenant Admin cannot enable the toggle, but they can connect with the EE Support team or CSMs to purchase more Credits. 

 

NOTE: For Production customers, the checkbox will be enabled, allowing them to Run a Simulation even if their Compute Credits are zero or below.

For Trial Tenant customers, the checkbox will be disabled by default, and they cannot enable it.


For New Production Customers, the checkbox will be disabled by default, but they can enable it, if required to Run a Simulation even if the Compute Credits are zero or below.

 

 

Learn more about the Tenant Preferences Page. 

 

Additional Improvements and Bug Fixes

1. Activity history shows activities for Simulations and Simulation Groups. 

The Activity History panel on the Study Overview page now shows activities for both individual Simulations and Simulation Groups. We have enhanced the Activity History tab to display Simulation-related events such as Simulation triggered or completed along with Simulation Group activities including group creation and deletion.

 

 

2. ‘New’ tag is added to indicate new Model in the Simulation Group:

 

A newly added model is now highlighted with the tag ‘New’ in the Simulation Groups list under Solution Comparison to help differentiate whether the Model has been added to BI in the existing Simulation Group or not. 
The new tag will be there until user removes the Existing Simulation Group from BI and again with the newly added Model into the BI.

 

 

Announcements

Cloud Bridge End of Support

Starting January 2024, Energy Exemplar will no longer support Cloud Bridge. This means the end of technical support, security updates, and ensured compatibility of Cloud Bridge with new releases of PLEXOS Cloud and related products.
To facilitate a smooth transition, we encourage you to leverage the PLEXOS Cloud Toolkit (CLI) with the Desktop-To-Cloud Integration. This offers a seamless experience for users to build and edit their models on the PLEXOS desktop while running simulations and analyzing output in the PLEXOS Cloud.
