PLEXOS Cloud Marketplace

The PLEXOS Cloud Marketplace is a new feature that provides a library of content for users. It allows users to access pre-built Excel Add-in templates, BI Reports and Solution Reports that can be easily copied into their own tenant.


The Marketplace provides users with a convenient way to access pre-saved layouts for reports and templates, saving them time and effort in developing their own. New content will continue to be added, so make sure to check back frequently.

 

 

Learn more about PLEXOS Cloud Marketplace.

 

Model Building and Editing (Desktop Integration and Excel)

Support Output Data Query Through Saved Layouts 

The latest update to the Excel Add-In enables the loading of large volumes of output data in a single operation, in the chosen layout or format. The user can select previously saved layouts built in the PLEXOS Cloud web interface to use in the Excel Add-In.

NOTE: Output can only be queried from successful Solutions.


To learn more, refer to the detailed article on Microsoft Excel Add-in.

The Solutions Panel Filter has an updated look and feel to let users filter Solutions on a more granular basis. It allows you to filter the data by Model Name, Specific Status of Solutions (such as BI, Congestion, Latest, and Older version), and specific Date range.

Output Data Grid Enhancements

  • Category Column in Output Data Grid

The 'Category' column is now available in the grid for showing the category of the selected objects.  The ‘Category’ column will be shown in the List view pivot type only. 
 

 


 

  • Save Output Data Grid Layout with all the selected settings

In a previous release, the ability to save the objects and properties selected on the Properties Panel as a Layout has been added to PLEXOS Cloud. Now with this release, you can save the Layout with all available settings and filter options. This includes saving adjusted column height and width, pinned and resized columns, applied sorting/filtering, and more.
 

 

 

  • Enable Drop Zeros by default

Drop Zeros will now be enabled by default to save the number of records loaded at once. It will automatically discard the rows with zero values.

Users can now manually Delete and Archive Solutions from the Solutions card. This will help users manage storage and eliminate unnecessary Solutions so they can focus on the Solutions that matter. An Admin user or the Study Owner can use the Study dropdown to manually Archive a Study and delete its associated Solutions. This option is designed to make it faster for users to free up storage space.

Administration (Simulation Management and Storage Management) 

Renaming the Admin Study Management tab to Storage Management 

We have renamed the “Study Management” tab as “Storage Management”. This change was made to better align the tab name with its actual function.


Please note that this change only affects the name of the tab, and all of the features and functionalities within the tab remain the same.


Export Storage Data for Studies and Solutions

We added a new Export icon in the Admin > Storage Management tab (Above the Action column). It allows Admins to export the data from the grid in either ".csv" or ".xlsx" format. It is helpful to get this data to sort filter, group, etc. Also, you can use these data as a backup for future reference or can share it within your internal teams for analytics purposes. 

NOTE: You must switch between the Studies and Solutions tab to export data for each type of data.



Select more Studies and Solution at once in Storage Management tab 

We have enhanced the Storage Management tab to allow Administrators to select up to 10 records (Studies or Solutions) at a time to perform bulk Archive, Unarchive, and Delete actions. This is done to save your time in performing bulk operations. Previously, the limit was 5 records at a time. If you exceed the maximum record selection limit, you will see a notification, “Only 10 Studies or Solutions can be selected at a time for any bulk action to be performed”. Integration and Automation - CLI/API

CLI Updates

Improved Output Messaging

The Command Line Interface (CLI) now provides clearer output messaging for the study changeset push command. If no changes are found, the CLI will output “Study has no changes to push”. This improves workflows and minimizes confusion if you are working with the most up-to-date version of your model.

The study repair-local command attempts to repair the cloud sync of a local study when the data has been corrupted or the study has been moved. If you enter the command, and PLEXOS cannot find your database in the correct file path and there appears to be no corruption, the following message will be output by the CLI: 


“The local Study folder for Study Id “studyId” was found in location “FilePath”. The Study is an active study in cloud environment and is tied to input project file at location “FilePath”. If study is not working correctly, recommend using the study delete local command and recloning to a new folder, followed by replacing local study data.”
 

New Options and Filtering Capabilities

The Simulation list, Solution list, and Study list commands can now list 1000 records. These commands previously only reported '250' records each, but the new larger number of records presents users with more data that can then be filtered.
Two new options have been added to the simulation monitor command:

--Raw and --Utilization. The --Raw option allows you to view live updates to the log file, and the --Utilization option updates on resource utilization in addition to outputting the parsed step data.The simulation monitor command previously only supported the --StepData option.

In this release, we have also added more flexibility to the "FileFilter" property in the SolutionData object by allowing only downloading desired files from a solution into a simulation's input data. This new feature works by allowing users to specify file name patterns to target only specific sets of matching files. For example, if a CSVOutput.zip file includes multiple folders with large quantities of data, but you only wanted to download the Daily Generator Generation files, you could specify the “FileFilter” as “*Daily/ST Generator*.Generation.csv”. This is especially useful for very large outputs where only a small subset of output files are needed.

Read more about using the CLI and automating workflows with these new features in Introduction to the PLEXOS Cloud Toolkit (CLI).
 

API Updates

Changes to Parquet Files 

With this release, the partitioned Data Parquet files will pack multiple “seriesIds” (one or more) within a row group for all new solutions produced.  Earlier, each “seriesId” data was packed in a one-row group. The "info" data will now have “1:N” relation between “seriesId:rowgroupId”, instead of 1:1. 

 

Additional Improvements and Bug Fixes

In this release, we have continued to make improvements to the PLEXOS Cloud, with a focus on enhancing the user experience, stability, and performance. 

As part of these efforts, we have improved the usability of Solutions Reports by retaining Report selections when users switch between different Solutions. 

We have also added horizontal scroll arrows for KPI Metrics cards on the Simulation Management page, specifically on the tabs mentioned below: 

  • Admin > Simulation Management 
  • Username > Simulation Management 
  • Simulations (within a Study)
  • Admin > Storage Management tab 

You can use these arrows to view hidden metrics cards in the list and can easily navigate to the required metric. 


  
The Properties panel now includes an indeterminate checkbox, which shows a partial selection of both properties and objects. This enhancement allows users to more easily identify which properties have selected fields.
