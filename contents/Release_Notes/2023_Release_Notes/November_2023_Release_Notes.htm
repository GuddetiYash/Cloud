Solution Output Analysis (Map, Grid and Solution Reports)

Date Range Filter for Solution Reports

 

NOTE: This change is applicable only for Analytics Tier customers.

 

In the Solution Report configuration, we have added a new filter property —a "date range". The availability of the Date Range Filter will improve your data analysis and reporting with Solution Reports. This helps you customize your Reports by specifying start and end dates in the YYYY-MM-DDT00:00:00 format directly in the Report's JSON file. It is a simple and flexible way to customize your reports. The advantage of this feature is it allows you to filter out any data that is not relevant to the time period of interest.

 

For example, if you wish to retrieve the data for only January 2023, you can set the date range as:


“date range”: [“2023-01-01T00:00:00”, “2023-01-31T23:59:59”]

Model Execution

View multiple Simulations at a time using the Simulation Monitoring Dashboard

Note: This change is applicable for both Simulate and Analytics Tier customers.

 

The Simulation Monitoring Dashboard enables you to monitor real-time progress updates, including status, key metrics, and logs for ongoing Simulations. Access this feature through the Simulations page by double-clicking on the Status icon for any In Progress Simulation (other states are not clickable). The monitoring dashboard opens in a new tab, displaying the Model's name as tab label. You can now open 5 tabs simultaneously, with each tab displaying a different run’s Simulation content. The content of each dashboard displayed in a tab will remain dynamic, allowing you to view the latest update for each widget and each piece of information. If you aren’t happy with the output, you can cancel the Simulation. This feature empowers you to efficiently oversee various aspects of the Simulation without constantly switching between different views.

 

 

Learn more about Simulation Monitoring Dashboard.

 

Icons to help identify the associated Study Version for Models in Compare/Simulation Groups

 

NOTE: This change is applicable only for Analytics Tier customers

 

Icons to help a user easily identify if a Model in a Simulation Group is associated with the latest Study version are now available on the Compare and Simulation Groups pages. The goal behind incorporating icons into the Simulation Group is to provide clear differentiation between the latest and previous Study Versions.

 

 

 

Model Building and Editing (Desktop Integration and Excel)

 

Quick and easy configuration of Template and Schema using the new Template Builder in Excel Add-In

The Template Builder feature enables users to interactively create or modify a Template, defining various Classes and Properties to be synchronized for editing/updating within the Excel workbook.
This feature offers a user-friendly interface for constructing or editing Templates, given the extensive range of Classes and Properties that can be edited in a PLEXOS Model. Users can browse Classes, and select their Properties, Categories, and Membership Classes to construct a Template that specifies the properties synchronized via the Excel Add-In. Utilizing the Template Editor enhances the process of finding and selecting the correct properties to include in a schema, eliminating the potential for manual data input errors that can occur when manually editing the Template. This saves time and minimizes errors for users creating Templates for themselves or sharing them with their team.

Integration and Automation - CLI/API

New CLI commands added for Executive Views

NOTE: This change is applicable only for Analytics Tier customers.

a) Command to create and publish Executive Views

Users can now create and publish Executive Views via the Command Line Interface (CLI). This improvement provides several benefits, such as the efficient publishing or automation of Views, making the sharing of critical reporting and data easier. It optimizes resources, streamlines the process, and reduces manual efforts.


Users can utilize the Command-Line Interface (CLI) to publish Views that incorporate Solution and BI Reports, by using the following CLI command:

pxc solution view publish --file {filePath}

 

 

The above command includes path to a JSON file which contains View name, View Description, tenant ID and Study Id, etc. as shown in the image below -

 

Definition of the Parameters used in the above shown JSON sample:

 

  • Name: This is the unique name of the View in the tenant
  • Tenant ID: This is the company’s tenant Id  
  • Study ID: This is the Study ID of the Study which is used for publishing the View.
  • ShowCreatedBy : This is the flag to show or hide the name of the Analyst who created the View.
  • ShowCreatedOn: This is the flag to show the date on which View is created. It should be ‘True’ always.
  • DatasetID: This is the Solution ID 
  • BIReportDatasetId: This can be empty. Not required to pass any value in this parameter.
  • DatasetType: This determines the type of the Dataset.  ‘0’ - Used for Solution, ‘1’ - Solution Comparision; Please use only ‘0’ as the value here, as we currently do not support any other values. 
  • Type: This is the type of report (BI or Solution Report) that you are publishing. ‘0’ is for BI Report and ‘1’ is for Solution Report.
  • Model Name: Model Name is the Model on which this Solution was triggered.
  • Original Report Name: Solution/BI Report name that you are using for publishing the View)
     

 

Note:

Views that contain a Solution Report, cannot be created or edited in the PLEXOS Cloud user interface at this time. Creating and Editing Views with Solution Reports can only be done via the CLI. When an 'Analyst' selects Edit/Save As from the Action Menu, a popup will appear, displaying the message: "Unable to perform this action as this View contains Solution Reports. This functionality will be supported in a future release."

 

 

Additionally, the ability to preview Views containing Solution Reports is currently not accessible. Clicking on the View name will display a message on the View page stating: "Solution Reports can't be previewed." 

 

 

b) Command to get the download details of a Solution Report 


By executing the following command, users can obtain details about the Solution Report    available for download. This information can then be used to retrieve Solution data and download the Report in .csv format.
 
pxc solution view get-downloadable-report-details --viewId [ ]      

 

 

c) Command to fetch the Solution/BI Report Details 

 

This CLI command can be used to fetch the Solution or BI Report details of a View which then can be used to download the Report in .csv format.
This command will return ViewID, reportId, name of the View, datasetId and type of Report.

 

pxc solution view get-report-data --outputDirectory {DirectoryName} --file {fileName} --reportId {reportId} --solutionId {solutionId} --viewId {viewId}

Announcements

Cloud Bridge End of Support


Starting January 2024, Energy Exemplar will no longer support Cloud Bridge. This means the end of technical support, security updates, and ensured compatibility of Cloud Bridge with new releases of PLEXOS Cloud and related products.
To facilitate a smooth transition, we encourage you to leverage the PLEXOS Cloud Toolkit (CLI) with the Desktop-To-Cloud Integration. This offers a seamless experience for users to build and edit their models on the PLEXOS desktop while running simulations and analyzing output in the PLEXOS Cloud.
