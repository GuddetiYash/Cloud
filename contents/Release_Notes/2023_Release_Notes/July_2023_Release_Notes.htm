Customize Icons for Classes and Categories on the Map

This new feature provides users the ability to select and customize icons for their Classes and Categories using the Icon Palette.

  

With the new Icon Palette, you are no longer limited to pre-defined or generic icons. Instead, you have the freedom to choose custom icons that align with your preferences and specific needs.

Model Building and Editing (Desktop Integration and Excel)

Excel Add-In: Add New Categories

The Excel Add-In now enables adding a new Category for any Class as required. This will make the data more precise for users and they can see the particular category data as required. Once Saved, these newly created Categories are available in the Category column drop-down for your selection. You can add different Categories to different Objects as required. 
 

     

Learn more about PLEXOS Cloud Microsoft Excel Add-in



Tag Properties to Scenarios

The Excel Add-In now provides support for tagging Scenario Objects to any Property data.
The Scenario column in the ClassName{Property} sheet allows user-friendly editing of Scenarios, by providing a drop-down input with the list of available Scenarios.

Using the drop-down, the users can browse the available Scenario names and select them, avoiding the need to remember names or accidentally enter an invalid name. Users previously had to enter Scenario Object names manually, so this improvement reduces the required keystrokes for users.
 

     

 

Learn more about PLEXOS Cloud Microsoft Excel Add-in


Create and Modify Categories

Excel Add-In now allows users to edit the Category of a PLEXOS object within a Model Database.
Users can now also create new Categories to assign objects to via the Add-in.
The new Category column available in any Class sheet now shows a list of all Categories that are available for the ClassName you selected in the Template.
You can modify the Category values as required or create a new Category if it is not available in the dropdown list. 
 

     


Learn more about PLEXOS Cloud Microsoft Excel Add-in


Excel Add-In: Time slices Support 

Excel Add-In now provides Timeslice support for Timeslice Files for a Class property. The two new columns get created when data is loaded. Users can attach the Timeslice tag to any property in the Timeslice Tag column to identify variations and add the Timeslice path in the Timeslice Text column. 

The Timeslice reference can be defined for a property using the Timeslice Tag and Timeslice Text columns. These columns are editable.
Refer to the description of the columns below:
 

  • Timeslice Tag:  Users can Add, Edit, and Delete Object Names from the Timeslice Class. 
  • Timeslice Text: Users can Add, Edit, and Delete any existing file path (For example, data, values, .csv, etc.)

We improved the handling of PLEXOS Desktop Out of Sync messages to provide more information about the Cloud version history.
An Analyst can now view the Version Number on Cloud when a local Study on the desktop is out of sync with PLEXOS Cloud. This helps users determine who made the changes and when they were made.  

Click on the Go to Version History button to view the PLEXOS Cloud VERSIONS page in a browser. Here you can view additional details on the changes for that version.

Users can now search for any Study in the Input Databases > Synced to Cloud tab on PLEXOS Desktop. A new Search bar allows you to search for the Study name. You can enter the characters matching your Study Name and the resulting list of Studies will be displayed in the list above.

Enter Personalized Commit message while pushing changes from PLEXOS Desktop

PLEXOS Desktop now supports adding a personalized Commit message when changes are being pushed from Desktop to Cloud.
When a user makes any changes in the existing Study and clicks on the Save dropdown > Save Study to PLEXOS Cloud button, a Commit Message pop-up appears, prompting the user to enter details about the changes, then Click Save to save the changes.

Previously, changes were being saved directly with no option to attach a message to the commit. 

 

NOTE: By default, {Pushed on dd MMM yyyy} format message will be saved if a user doesn’t customize their message.

Model Execution

Run One or More Models in a Simulation Group

Users can now run Models individually within Simulation Groups with this release of PLEXOS Cloud.
With this update, when a new Model is added to an existing Simulation Group, there is no longer a need to run the entire group. Instead, you can now execute the Model on its own using the Actions menu located in the row for that Model. 

 

This enhancement provides users with greater flexibility and efficiency in their Simulation workflows. You can focus on running a specific Model without the overhead of executing the entire group.  

With individual Model execution, you can save time and computational resources by avoiding the need to run all the Models together. 

 

Simulation Monitoring Dashboard (Beta Release) 

The new Simulation Monitoring Dashboard feature offers real-time progress updates on the Status, Key metrics, and logs for ongoing Simulations. It provides users with the necessary information to assess whether the Simulation is progressing as anticipated.  

If deviations are detected, users can make informed decisions, including the option to cancel the simulation. 

 

NOTE: The Simulation Monitoring Dashboard will appear only with PLEXOS Engine and not with Aurora Engine.

 
The dashboard presents information such as Estimated Time to completion, Generation, Price, Load, Memory and Time Utilization, Engine Details, and other relevant metrics. 

 

  
 

Users can see the Dashboard for only one Simulation at a time. The Simulation Monitoring Dashboard is only available for Simulations that are In Progress state.
  
Learn more about Simulation Monitoring Dashboard

 

BI Reports

Enhanced BI Solution Management 

To manage Solutions added to BI effectively, users now have the option to view a detailed list all Solutions in BI Analytics.  

This helps the user decide which Solutions to remove when the limit is reached.  

This allows users to review the list and make informed decisions about which solutions they can remove from BI analytics. 

   

Users are restricted from adding or removing a solution when there is already a solution in the "In Progress" state. 

   

When the user level limit is reached for adding Solutions to BI, you can refer to the list and remove any unnecessary Solutions to make room for adding a new one.  

By checking the list, you will be able to see the total number of Solutions you have added, which will assist in determining how much of your limit remains. 

    

BI Solution management enhances usability and helps users manage their Solutions effectively, ensuring an optimal experience when working with BI analytics. 

 

Administration (Simulation Management and Storage Management)

Set your preferences for Email Notifications

All Users now have a choice to manage their PLEXOS Cloud Email notifications. The Preferences page, available by clicking on your username at the top right of the interface, allows users to configure email notifications for the Simulation and Compute Credits sections. This is done to avoid getting unnecessary emails or spam emails or overcommunication through email. Users can select the one they want to receive and skip the other emails.  
 

NOTE: The Email Preferences section is not visible to Reviewers or Read-only users. The Compute Credits section is only for Usage-based licensed customers. 
 

    


Manage your Email Preferences:
 

  • Simulation (For Analyst/Modeler only)

 

Enable or disable email notifications for different statuses of the Study creation process.

 

  • Compute Credits

 

NOTE:

Core-based users will not have access to this section.
 
  • Compute Credit Refund Alert
Enable or disable email notifications when Credits get refunded to your account against a Failed Simulation.
 
  • Compute Credit Usage Alert (For Tenant Admins only)
Enable or disable email notifications when your Compute Credit Usage limit crosses your desired threshold.

 

Notification When Credits are Refunded

Users will now receive email notifications when they receive a refund against the raised ticket in a Failed Simulation Run. Previously, users were not notified about refunded credits.  

This improvement ensures that you stay informed of the requested refunded credits resulting from unsuccessful Simulation runs. By receiving these email notifications, you can easily track and manage your credits, promoting transparency and enabling you to make informed decisions regarding future Simulations. Users can also make the choice to unsubscribe from these email notifications on the User Preferences page if they wish.  

 

Integration and Automation - CLI/API

Filter Study List by UserId

Changes have been made to the study list command that allow users to filter by the logged in user Id . The purpose of the study list command is to gather data about the available studies in your environment such as the Status, Name, Id, CreatedDate and more. The new --filterByUserId option allows you to filter the data returned by the command so it is specific to your studies with no extraneous user data.
 

Download Solution Reports as CSV

A new command has been added to support the download of Solution Reports as CSV files. The pxc solution solution-reports command allows you to download Solution Reports from the command line, and easily automate processes that interact with Solution Reports. The output directory is a required input for this command. To learn more about this command and using the CLI, refer to the detailed article Introduction to the Introduction to PLEXOS Cloud Toolkit (CLI).

Additional Enhancements and Improvements  

  • Timeseries Patches Enhancements

A new download button is available in the DATA >VERSIONS > Timeseries section of PLEXOS Cloud. Users can now download any Timeseries or Ancillary Files (.csv format), including those with sizes exceeding 5MB (>5MB), and view the changes on their local machine. 
 

  • Remove Timeseries Changes in Version History (Cloud Bridge) 

Effective starting this release, Timeseries Changes will not be available for users on the Version History page of Cloud Bridge.
Previously, users could see the Old and New values for the modified Time series file, but now only the FileName and Status sections are visible. As we are deprecating Cloud Bridge , we encourage users to use PLEXOS Cloud to view and download the Timeseries changes. You can find view changes under DATA > VERSIONS > Timeseries section of PLEXOS Cloud. 
 

  • Duplicate Study Error 

To ensure a smooth experience in the Study Creation process, an alert is shown to notify you of duplicate Study names. If you enter an existing name while creating a new Study, an error message stating," Study name already exists" will be displayed.

  • Environment name will be shown in the Sign In button

The Sign In button now displays the specific Environment name, allowing you to easily identify the environment you're logged into. After Signing in, your username will also be prominently shown, adding a touch of personalization to your PLEXOS experience.
