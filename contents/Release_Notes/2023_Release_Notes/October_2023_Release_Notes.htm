Solution Output Analysis (Map, Grid and Reports) 

Retrieve Data for All Sample IDs in Solution Reports

In Solutions Reports, alongside “reportByCategory” and “reportByObjects”, users can now retrieve data by Sample Ids by using the flag "reportBySamples". There is a new flag under "auto populate children" called "reportBySamples". At once only one of them can be true for one property, others will be false.

For instance, consider the following configuration:

The Auto sample generation flag in Solution Reports as shown below

{

  "Formula": "Generator_Generations_Generation",

   "reportBySamples": true

}

  • "Formula" corresponds to the formula or data source you want to analyze.
  • "reportBySamples": “true”, is the flag that enables the auto-generation of sample data.

In the above example, “reportBySample” is set to true then for Monte Carlo studies with 20 samples, in this case, there will be 20 columns generated and with each column will fetch report data against that sample as shown in the below image:

 

Generation (Sample1) Generation(Sample2) ... Generation(SampleN)

 


This enhancement offers users the ability to create comprehensive reports tailored to their specific needs, ensuring a more versatile and efficient reporting experience.

Executive View: Edit and Save As Options in View Management

From the Overview Page, Analysts can create tailored Views of output data for managers, executives, and Decision Makers. The View Management grid displays all the Views for the currently selected Study. The Action menu in the View Managements grid has two new options alongside the existing "Delete" option. Users can now "Edit" or "Save As (Copy)” a View, providing enhanced flexibility to Analysts for customizing their Views.

 

Edit:


The "Edit" action opens a wizard for the selected view. With Edit, users can update the following fields: Report Name, Report Description, View Name, View Description and update the list of shared users. It is important to note that, under specific circumstances, such as when a user unpublishes a Solution used in a View or deletes an original BI Report, some of the Reports may become unavailable within that View. When this scenario arises, a popup/dialogue will appear informing the user about the unavailability of the reports, then they can proceed with the Edit action by clicking CONTINUE WITH EDIT.

 

 

Save As (Copy):

 

When clicked, the action "Save As (Copy)" launches the wizard with the latest published Solutions for the Study in the existing View template.  A new View will be created with the name 'copy of previous name.' (This name can be changed.)

This new View includes all previously selected reports and details from the source View, along with the list of pre-populated users. Analysts  have the flexibility to edit any of the pre-populated fields according to their requirements.  

 

 

NOTE:


The View Management grid has been updated to sort entries by “Created On” field, ensuring that the most recently shared View is displayed at the top of the list.
Please note that the 'Edit' and 'Save As' options are exclusively available for new Views only. These options will be disabled for the Views created before this release. 
 

Learn more about Executive View in PLEXOS Cloud


Reviewers now have access to the View Management Grid

Reviewers now have access to the View Management Grid, providing them visibility into existing Views. Since Reviewers have read-only privileges in PLEXOS Cloud, they do not have access to the Action menu options within the View Management Grid.

 

Views now accessible by all the Analysts

Views can now be accessed by all the Analyst irrespective whether they have access to the View or not. From the View Management Grid, click the View Name to open the View in a new tab. Analysts can perform all the actions in the View such as Edit, Save As and Delete.

 

Show or Hide the Analyst’s name that Created the Executive View

Analysts now have the option to hide the 'View Created By' field from users accessing the View, allowing users to concentrate more on the content rather than additional details. While these details will still be recorded and displayed in the View Management Grid, by default they will remain hidden from users accessing the View from the simplified Executive View interface.

Model Execution

Allow Output to be added as an Input to the other Simulations

Users can now select the output of a previous Simulation as the input for the current Simulation. That means you can select a Simulation by choosing the associated Study, Model, and Solutions under ADD INPUT option.
You can add multiple outputs from various sources as input to a single Simulation, saving time by streamlining the data integration process, ensuring data continuity, and maintaining accuracy.

 

 

For detailed step-by-step instructions, please refer to the article Run Configuration Panel.

 

Model Building and Editing (Desktop Integration and Excel)

Quick and easy configuration of Template and Schema using the new Template Builder in Excel Add-In

The Template Builder feature enables users to interactively create or modify a Template, defining various Classes and Properties to be synchronized for editing/updating within the Excel workbook. 

This feature offers a user-friendly interface for constructing or editing Templates, given the extensive range of Classes and Properties that can be edited in a PLEXOS Model. Users can browse Classes, and select their Properties, Categories, and Membership Classes to construct a Template that specifies the properties synchronized via the Excel Add-In.

 

Utilizing the Template Editor enhances the process of finding and selecting the correct properties to include in a schema, eliminating the potential for manual data input errors that can occur when manually editing the Template. This saves time and minimizes errors for users creating Templates for themselves or sharing them with their team.

Integration and Automation - CLI/API

Introducing Version 2 of the Parquet Solution Output Schema

In this release, we have introduced the option to request a new Output schema version for Cloud queryable Parquet Output Files. Version 2 builds upon the existing Parquet Output Schema, ensuring backward compatibility for all querying use-cases. 

 

There are several advantages of this which are listed below:

 

  • Compatibility with PLEXOS 10: Version 2 allows you to open a Parquet database just like the native PLEXOS zip Solution in PLEXOS desktop, enhancing interoperability between desktop and cloud.

 

  • Support for Block Reporting Querying: It introduces a phase table that links reporting blocks to interval periods, facilitating block reporting querying use-cases.

 

  • Expanded Information: Version 2 includes additional data from the PLEXOS Solution ZIP file to support more querying use-cases. This encompasses membership information beyond reported properties, extra tables, and columns for master schema data. This reduces the need to join output data with input data to gain a comprehensive system view.

  • Enhanced Compression: Version 2 supports newer Parquet compression encodings, notably ZSTD compression combined with Parquet delta encoding. This results in significant storage size reductions for Parquet data with longer period horizons and improves the Simulation time spent creating Parquet files.
     

 

Learn more about Parquet Version 2.

 

Additional Changes in PLEXOS Cloud Toolkit (CLI) Simulation Enqueue Request

The ability to enqueue Monte Carlo Simulations set up to automatically stitch all sample databases after the execution is completed has now been added to request JSON for the Command Line Interface (CLI). This can be utilized with an optional property called “IsAutoStitchEnabled” placed on the existing ParallelizationOptions object, with a value of 'True' or 'False' as shown below:
 
  "SimulationOptions": {
    "Locale": 1033,
    "EnableRealTimeLog": true,
    "SimulationRunType": "Stochastic",
  },
  "ParallelizationOptions": {
    "InstanceCount": 5,
    "IsAutoStitchEnabled": true
  }
 

Support for running “CustomPlexos” Simulations using the CLI 

For Simulations using the PLEXOS engine, the Simulation platform now supports enqueuing Simulations with custom arguments, which are executed using PlexosLauncher with those specified arguments.

 

Example JSON Changes for Enqueuing: 
{ 
      ... 
   "SimulationType": "CustomPlexos", 
    "simulationOptions": { 
        "customArguments": "-s P,6,3", 
         ... 
    }, 
   … 
} 

 

CustomPlexos Limitations: 

 

The following PlexosLauncher options are not supported directly by custom arguments as shown below: 

 

                

 

  • This type of Simulation does not explicitly account for the various configurations possible with PLEXOS Launcher and does not impose additional constraints beyond the already mentioned validated options (above). 
  • CustomPlexos Simulations will run on a single machine and are not designed for distributed environments.
  • If the Simulation or defined models require parallelism, the VM size should be chosen appropriately to accommodate the desired workflow. 
  • Simulations will not perform ETL on solution data; only raw Solution data and logs will be available as uploaded artifacts. 
  • Success is not guaranteed, and Simulations may fail if the command-line arguments are invalid or misconfigured for the model."

Show categories icons on Map (Setup and Solutions):

Class category icons now appear on the map in both the Setup and Solutions pages unless the user has customized category icons.

Announcements

Important Update: Retirement of the Compare Option

Starting with this release, the Solution Panel's "Compare" option has been substituted with the enhanced BI Solution Compare feature. The new Compare feature empowers users to comprehensively analyze and compare results among various Solutions. Explore further details about the BI Solution Comparison.

Cloud Bridge End of Support

Starting January 2024, Energy Exemplar will no longer support Cloud Bridge. This means the end of technical support, security updates, and ensured compatibility of Cloud Bridge with new releases of PLEXOS Cloud and related products.
To facilitate a smooth transition, we encourage you to leverage the PLEXOS Cloud toolkit (CLI) with the Desktop-To-Cloud Integration. This offers a seamless experience for users to build and edit their models on the PLEXOS desktop while running simulations and analyzing output in the PLEXOS Cloud.
