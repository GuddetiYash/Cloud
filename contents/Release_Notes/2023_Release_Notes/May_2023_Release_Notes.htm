Solution Output Analysis (Map, Grid and Reports)

Enabling Pivot view in Output Data Grid Page (Beta Release) 

The Data Grid page in Solutions now has a Pivot View to include customization options for users. Previously, Data Grid supported List, Name, and Property view options. The List View remains, and the Name and Property Views are replaced with the new Pivot View.

 

The Pivot View can be enabled using the Pivot View Toggle in the Output Data Grid Page.   

It lets you switch from a list view to a more powerful Pivot View. With Pivot View, you can easily analyze large amounts of data and gain insights that were previously hard to discover. 

You can drag and drop columns to the group and aggregate your data.  

For example, you have a dataset that tracks the performance of different power generators across different regions. With Pivot Mode, you can quickly create a Pivot table that shows how generators are performing in different regions and compare their performance against other generators. 

For customization in Pivot View, simply click the Pivot Mode switch in the Grid toolbar. There is a sidebar that provides options for row grouping and pivoting, allowing for more flexibility and customization. 




Pivot View with Ad-hoc Charting

You can continue to create ad-hoc charts by selecting cells and doing a right-click. Additionally, selected cells will show key metrics at the bottom right including Sum, Average and others.

Save Map Layout with Output Properties and Configurations 

The new Save Layout button on the Map view enables users to store the current layout (configuration) of the Map. The Map Layouts are user-specific and can be used across other Solutions. It allows Modelers to select and save specific objects, output properties, and visualizations on the Map.
  The Map Layout feature allows users to save a variety of selections as mentioned below:
 

  • Classes 
  • Objects 
  • Output Properties 
  • Output Configurations (Visualization Type, Colors, etc.) 
  • Time 
  • Zoom Level

Enable Solution Comparison for Solutions Reports (Beta Release) 

The latest update of PLEXOS Cloud's Solution Reports now allows you to easily compare Solutions. With this new feature, users can compare two solutions at the same time by comparing PLEXOS direct output properties and Solution Reports' columns.

The comparison results are presented in a clear and user-friendly way, with both chart and grid views available for each Solution.  

Additionally, the delta view highlights any differences between the two Solutions, making it even easier to analyze and understand your data. 




To access the Compare Solutions option, click on the Compare Solutions icon located within the Solutions tab.

Introducing the Activities Panel Under the Overview Tab

The Activities panel, which is accessible from the Overview page, allows users to view updates and modifications made to that Study. This panel shows a list of all changes made in the last 30 days. The Activities panel provides information about the action that was triggered along with additional information such as the Model Name, the person who made the change, and the day or time the change was made. It tracks various types of modifications, including:
 

  • Simulation Triggered 
  • Simulation Cancelled 
  • Version Created 
  • User added/removed from the Study 
  • Study changed to Private/Public 




 NOTE: Please note that to see the changes reflected in the Activities Panel, you'll need to do a browser refresh. This will ensure that you have the most up-to-date version of the page and can view the changes properly. The panel will only display Activities that occurred after the upgrade date, activities performed on the Study prior to the May 2023 release will not show.

Administration (Simulation Management and Storage Management)

The Storage tab in the Admin > Tenant Preferences page now allows Administrators to define rules to automatically delete archived Studies and Solutions after a specific number of days selected in the Days dropdown. Integration and Automation (CLI/API)

New Command for Solution Stitching 

Solution stitching can now be triggered using the CLI. To start, make sure that the Run Type is set to Distributed and the Distribution Method is set to Monte Carlo in PLEXOS Cloud. After executing the model, the following inputs are required to execute the command: "executionId", "cores", "memory". The command can be entered as shown below: 
 

pxc simulation solution-stitching --executionId "Parent simulation executionId" --cores "cores" --memory "memory" 

 

Triggering Simulation Group Operations Using the CLI

A Simulation Group is a collection of Simulations that can be run together as a batch. Users can create and edit these groups so that they don't have to manually select multiple Models each time they want to run them, and can instead simply run a predefined group. 

Users can trigger operations related to Simulation Groups using the "pxc" study settings command. Currently, the only type enabled in the study settings command is the SimulationGroup type. 

Users can create, delete, and list Simulation Groups, and add configurations to existing Simulation Groups. Enter the following command in the CLI and you will be directed to the help options available for the command:
 

pxc study settings -? 


Users can also execute Simulation Groups via the CLI using the command below. The groupId and studyId are required inputs to execute the command.  
 

pxc simulation run-group --groupId "groupId" and --studyId "studyId" 


Learn more about Solution Stitching and running Simulation Groups using the CLI in the article Introduction to PLEXOS Cloud Toolkit (CLI). 
 

Additional Improvements and Bug Fixes

We have continued to make improvements to PLEXOS Cloud, with a focus on enhancing the user experience, stability, and performance. 

As part of these efforts, we have improved the usability of Gurobi Tuner runs, it now allows users to select a specific phase to optimize (ST, MT, LT, Pasa).  

User Level Limits have been added for publishing BI Solutions. Moving forward, this limit will be applicable to all PLEXOS Cloud users. Each user will now have a maximum limit of '10' published Solutions. If you reach your limit, you will see a warning message indicating that you have exhausted your limit. To regain access to the limit, you'll need to unpublish some of your existing Solutions. This update will help ensure that resources are used efficiently while still allowing you to create and publish the Solutions you need. 

The Excel Add-In has now been modified to reflect changes in the format of the output date column based on the selected period type. The available options for period type include Quarterly, Monthly, Interval, Hourly, Yearly, and Daily.
 

  • Quarterly or Monthly - 'MM YYYY' - 07-2019 
  • Interval or Hourly - 'DD MM YYYY: hh:mm:ss' - 25-07-2019: 15:14:30 
  • Yearly - 'YYYY' - 2019 
  • Daily - 'DD MM YYYY' - 25-02-2019 


Solution Id will be removed from Diagnostics Effective from this release, the Solution Id will not be a part of your Diagnostics Id. Please make use of CLI to query the Solution Id as mentioned below: 

 

pxc solution list --simulationId 9cae0cca-48c2-4d66-a79b-409023d50205

pxc simulation list --studyId de6b9dda-5c70-4cc6-8736-8ac83d9d6a8a

pxc simulation list --executionId 611c005d-5577-4044-a29f-77353fd20467


 
Diagnostics are not something users should reference for automation. The CLI shows actual data for use, whereas the Diagnostics are for users to send when raising support cases, so we have the information we need to understand the context. 

For Bug Fixes, in the Excel Add-In, the Out of Sync error has been resolved when loading data. Now a confirmation popup is displayed to the user asking them to reload the data or time-series files upon clicking the Load data button in the Database tab and Data tab.   

Known Issue: If your display name is showing incorrectly in the PLEXOS Cloud UI, particularly with special characters, simply update the value and save it to correct.
