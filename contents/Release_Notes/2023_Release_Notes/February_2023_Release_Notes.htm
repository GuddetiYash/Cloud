Create and Edit BI Analytics Dashboards directly in PLEXOS Cloud UI

With this release, we have enhanced the look and feel of BI Analytics Dashboard. We have added editing for BI Reports directly in the PLEXOS Cloud interface. 


In this release, the BI Analytics Dashboard includes the following capabilities.

 

Show Out of the Box (OOTB) Reports and Custom Reports

In this update, we have made the BI Dashboard's layout the same as the Solutions Reports layout to make it more user-friendly. Now in BI Reports, you can see, EE Reports and Custom Reports (My Reports). 
 

 
 

Edit and Publish Custom Reports

You can now edit your own reports and make them public or private (hidden from others) 

 

Ability to maximize and minimize the Reports page

You can maximize the Reports window to have a larger view of the page. 

 

Add New Reports

You can now add any existing reports or create a new one under My Reports (Custom Reports)

 

Search New Reports

Reports can be searched by entering the required keywords in the search tab. Learn more about the above-added functionalities BI Analytics Dashboard .

 

Addition of Aggregation and Transpose in the Solutions Reports

We have continued to enhance Solutions Reports. In this release, we have provided the 4 Aggregation functions ( SUM, MIN, MAX, and AVG) to be used in the Report data and an option to apply Transpose in the Report table.

 

Aggregation

The Aggregation method allows you to fulfill the requirement for getting the values for Sum, Average, Min, and Max.
In this method, we have provided the functionality to edit the “Json” file of the report. In that file, you can add the required Aggregation function (SUM, AVG, MIN, and MAX).

 

Transposing the data of the Report

Transposing the data in your Report means swapping the rows and columns. It is very useful when you need to rearrange the data for rows in columns or vice versa. In this release, we have introduced this feature so that you can quickly switch data between columns and rows.

 

To know more about the Aggregation method and Transpose, refer to the detailed article on Solution Reports .

 

Improve Simulation Run time with Gurobi Tuner in the PLEXOS Cloud Interface

The Gurobi Tuner helps tune different parameter values in your model to improve the performance by reducing the overall simulation duration.
The Gurobi Tuner can now be used from the PLEXOS Cloud UI as well. In the previous release, it could only be done via the Command Line Interface (CLI).
 


 

Follow the steps mentioned below to trigger the Gurobi Tuner.

1. Go to Models page and select the Model you want to tune.
2. Click on Run.
3. In the configuration window, Select the Run Type as ‘Tuning’ from the drop-down.
 

         

 

4. Select the Tuner type as “Gurobi Tuner” and then select the values for each of the other fields including “Engine Version”, “Tuning time”, “Tuning Attempts” and “MIP Gap Target.”

 

         
 

 

The MIP Gap Target parameter controls the minimal quality of the returned solution. The default value for MIP Gap Target is “0.00001” which can be changed as there is no upper limit.

 

Tuning Time is the amount of time it will take to tune in your data. By default, it is set to 7200 mins. However, you can change it with the desired value.

 

Tuning Attempts is the number of trials used by the Gurobi Tuner to figure out the right parameters for the simulation. '5' attempts is the default, you can change that too.

 


5. Now go to Advanced Configuration and select the required Input for – Study, Model, and Simulation. Study and Model are selected by default. You can select simulation from the drop-down. It will take solution files from the selected simulation as input.

 

       


       
 

Excel Add-In Enhancements

The MS Excel Add-In for PLEXOS Cloud now supports the DateFrom and DateTo values of model properties by having the “IsDateRequired” column value set to “Yes” in the Schema tab of the template. You can now edit data in the Value column of any <ClassName>Property sheet for the selected Study and click Save to upload the changes to your Study in PLEXOS Cloud.



 

 


Persistence of Study and Data Files Connection

Previously, the changes made to the Data Files had to be pushed before closing the Excel but the Persistent Data File Functionality eliminate this effort by allowing the Data File to load automatically with the recent changes saved when you save the Data File and re-open it again. It will select the associated Study and Data Files by reading the data from the Config sheet. The MS Excel Add-In preserve the connection to the last study used, as well as any data files that was open during the last editing session. When the user re-opens the Excel editing template file, the Study selection will be populated automatically, and any data file tabs will be re-opened.
 

Learn more about Microsoft Excel Add-in

 


 

View Versions and Manage Geocodes on the New Data Tab in the Primary Navigation

In the Primary Navigation on the left side of the screen, a new item labeled Data has been added. Data combines the previous geocode Setup functionality with the Version history. This allows users to track and view the Versions of the current  selected Study along with input data geocodes for viewing map data, editing the layout, and downloading, and uploading the geocodes and shape files.



 


The Data tab consist of two tabs under it as mentioned below:

 

It shows the list of versions for the currently selected Study on the left side of the screen and tracks the Adds, Edits, Removes, Timeseries and Ancillary data changes made under the currently selected version on the right side of the screen.

 

 

 

Setup: For the currently selected Study, it allows you to set up the map data, move around objects, and edit the layout of objects, download or upload map data in CSV or Shapefile format.

 

 


 

New location for accessing Administration functions

Administrators now have easier access to key functions as there is a new Admin button at the bottom of the left navigation bar. This moves it from under the Username and adds an updated Administration panel to allows users to navigate back and forth quick between Studies and admin functions.

 

 
 

New Study specific Simulation Management tab

The Simulations tab in the primary navigation pane shows all Simulations triggered for the Study selected in the Study dropdown. All users that have permission to the selected Study can access the Simulation Management page. It allows users to monitor in-progress and completed Simulations for a particular Study. Also, it shows all key details for the runs during the selected time period as well as key metrics at the top and a trend chart to show simulations over time.

With this new page there are now three ways to view Simulations:

 

  1. Study page- Accessible from the left navigation. Shows simulations for all users for the selected study.

  2. User page- Accessible from the Username dropdown. Shows all simulations across all studies for the logged in user.

  3. Admin page- Accessible from the Admin page. Shows all simulations across all studies and all users. Available only to Administrators.

 
               

Learn more about Study Specific Simulation Management.

Change in Re-activation period for Users

In the Users page of Administration, any a user set to Inactive, can only be active again by an Admin after 30 days. Similarly, the re-activation period for user license is set to 30 days. This is done to prevent license sharing between users.
If an administrator accidentally set a user to Inactive and needs to adjust it before 30 days, they can contact support and provide the User ID that is available at the top of the user detail page (use the copy button when opening the support issue)

 

 
 

Enhancements to Study Permissions

Permissions have been updated to only allow Study Owners and Admins to Archive, Rename, and edit the description of a Study. Previously, all members of a Study would be able to perform these actions.

Managing Storage in PLEXOS Cloud- Enhancements

The Study Management page under Administration is updated with four new storage metrics that provide a high-level summary of the storage across all the Studies and Solutions. These four metrics are:

  1. Total Storage

  2. Remaining Storage

  3. Active Storage

  4. Archived Storage


See the metrics details as shown below:

 

 
 

  1. Total Storage: The first number shows the total storage consumed across Studies and Solutions tab including Active and Archived Storage. The second number shows the total storage allocated for all users in a tenant.

  2. Remaining Storage: Shows the sum of total available storage minus consumed storage.

  3. Active Storage: Shows the total storage consumed by all Active Studies and Solutions.

  4. Archived Storage: Shows the total storage consumed by all Archived Studies and Solutions.

 

Learn more about Managing Storage in PLEXOS

Improved performance and loading of data in the Solutions Output Data Grid

We have added major improvements to the Output Data Grid within Solutions to improve performance and allow you to select and load significantly more data.

Faster and Easier loading of data in the Grid

We have implemented a solution for fast and easy loading of data in the Grid. This process will break down the large number of records into smaller chunks or you can say pages to make the data loading process easier and faster.

Now, you can see a certain amount of records on one page, and thereafter, the Show More Data button will be enabled for the data that is there on the other half of the page.

You can also see the total number of data present along with the total number of records shown.
We have also added Infinite scrolling to scroll through the records on a page.

 



 

In the above example, the Show More Data button is disabled. It gets enabled once all ‘37290’ records are loaded. For example, refer to the image below.

 

 

Remove limitations for selection of properties & objects:

In this update, we have removed the limitation on the selection of properties and objects. Now, you can select multiple or all the properties at once. Previously it was restricted to selecting only 5 properties at once.

 

 

 

Show Popup for Huge data

When the number of records exceeds 10 million rows then a popup will be shown to encourage you to adjust the query to reduce the amount of data.
 

 

Sort and filter data only when all data is loaded on the grid

The sorting and filtering options will only be enabled when all the available data is loaded. This is done to ensure the accuracy of the result you get. You can also see the message for this on top of the Grid.

CLI Enhancements

Support for ML.Net training models is now included for PLEXOS Cloud Studies. This functionality was not previously supported.

The push changeset command now returns the changeset ID of the changeset just created. This allows for easier access to simulation data for future reference. The command below allows you to learn more information about pushing study changes:


pxc study changeset push -?


You can now enqueue simulations using solution archives from previous runs. This improves the workflow if you have archived a study solution and still want to run a simulation based on a previous request.


There have also been general bug fixes related to switching environments and logging in. Learn more here in the Introduction to PLEXOS Cloud Toolkit (CLI). .

 

API Updates

There have been no updates to PLEXOS Cloud APIs in this release.

 

Additional Improvements and Bug Fixes

We have continued to make improvements in the UI and fix bugs affecting user experience, stability, and performance of the PLEXOS Cloud.

Changing the name of a study, the description, or archiving it can now only be done by Study Owners and Tenant Admins. This is the same for Public and Private Studies.
