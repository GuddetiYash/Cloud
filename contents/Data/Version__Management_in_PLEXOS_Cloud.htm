Version Management in PLEXOS Cloud

Understanding the Process of Pushing and Pulling Changes from PLEXOS Cloud

What is Versioning?

The process which involves creation and management of multiple releases of a product referred as Versioning.

Mostly, all of the versions have same general function but are improved, upgraded or customized.

In software development, versioning allows dev teams to keep track of the changes they make to the project code. It may include new features, enhancements, or bug fixes.

How does versioning work in PLEXOS?

One of the key features of PLEXOS Desktop is the ability to save and sync studies to the cloud, making it easy for multiple users to collaborate on a single study.

Users download a master copy of the study from PLEXOS Cloud to edit on their local PLEXOS Desktop (Known as downloading/initializing a study).

Changes made by any user are then saved to PLEXOS Cloud, with only the differences uploaded and merged.

Any changes made to the master copy on PLEXOS Cloud are also downloaded and merged into the local copy, ensuring synchronization between local and cloud versions.

Further In this article, we will explore the two-step process of syncing studies to the cloud from desktop, the concept of sync status, and how versions are created and viewed in PLEXOS.

 

Syncing Studies to the Cloud

There are two ways to push a new version of a study to PLEXOS Cloud. The first is to check the Sync status, which indicates whether the study is in sync with the cloud or not.

If the sync status shows that the study is out of sync, it means that changes have been made to the local copy of the study and need to be pushed to the cloud to make it in sync again.

To push the changes to the cloud, simply click on the Save Study to PLEXOS Cloud button, which initiates the process of syncing the local changes to the cloud.

 

Sync Status and Versions

Sync Status indicates whether a study is up to date with the changes made by other users or not. When changes are made to a study by one user and saved to the cloud, other users need to pull those changes to their local copy of the study to ensure they are working with the latest version.

Sync status can be checked by clicking on Check Sync Status, which is located under cloud icon in PLEXOS desktop.

If the sync status shows that the study is out of sync, it means that changes have been made by other users and need to be pulled to the local copy of the study.

Versions in PLEXOS

Versions in PLEXOS are created automatically when changes are saved to the PLEXOS Cloud or pulled from the cloud to the local copy of a study. When changes are saved to the cloud, a new version is created with all the details, which allows users to view and compare different versions of the same study.

However, it's important to note that versions are not visible in PLEXOS desktop, but only on the cloud. This means that users can only view and compare versions of a study on PLEXOS Cloud.

The version history in PLEXOS Cloud can be visible under Data tab.
