Introduction

The Microsoft Excel Add-in allows you to review and edit Model Input Data and synchronize changes to  PLEXOS Cloud directly from Excel. Also, you can fetch the Output Data for analysis and sharing purposes. The Microsoft Excel Add-In replaces the now deprecated Excel Plugin. The Add-In is lightweight and supports both Model Data and the Data Files associated with the Study. Small, targeted changes or complex bulk changes based on formulas can be done easily and saved back to the PLEXOS Cloud with just a few clicks.

 

NOTE: This applies to Analytics Tier customers only.

Managing Excel Files from the PLEXOS Cloud Interface

The Excel Files tab on the PLEXOS Cloud Overview page allows you to manage Study-linked Excel files directly through the web interface. This integration streamlines collaboration and version handling between the Excel Add-In and the PLEXOS Cloud platform.

 

You can duplicate, rename, or delete Excel files directly from the interface, eliminating the need for manual downloads or reuploads.

 

  1. Duplicate Excel File: The Duplicate action allows you to create a copy of an existing Excel file within the same Study or in a different Study that is accessible to you.

    1. This Study: Selected by default. Creates a copy of the file in the current Study for immediate use. This is especially useful when you want to create a personal version or backup of an existing file. Since versioning is not currently supported within Excel files, duplicating the file allows you to retain an earlier version for future reference or local backup, ensuring you don’t lose access to previous configurations or changes.

    2. Other Study: Opens a dropdown menu with all Studies that you can access. Once selected, the file is duplicated in the chosen study. To ensure file name uniqueness and prevent conflicts, the copied file is automatically suffixed with a unique random number


  2. Rename Excel File: You can rename Excel files by clicking the pencil icon next to the file name.


  3. Delete Excel File: The delete icon allows you to remove Excel files that are no longer needed from the selected Study. This helps keep the Study file lists clean and organized.

Bulk and One-Time Load for Input Data

The Input Data tab supports both Database and Datafile inputs. This offers two distinct options for loading timeseries files efficiently:
 

  • One-Time Load: Add and load individual timeseries files as needed, ideal for selective, file-by-file loading. 

  • Bulk Load: Use the Data File Query Configuration sheet to save a set of timeseries files that require frequent or iterative loading. The Bulk Load functionality allows you to load all added timeseries files from the Data File Query Configuration table to an Excel sheet in a single action, saving time and simplifying setup when working with multiple files. 

 

These options streamline the data loading process, boosting analyst productivity and enabling faster model setup by supporting bulk actions directly from a centralized configuration table.

 

 

 

Error Tracking and Data Load Validation in the Excel Add-In 

The Logs sheet in the PLEXOS Cloud Excel Add-In provides clearer insights into input/output operations and helping you troubleshoot issues more efficiently. 

➤ Validation Messages for Input and Output Data Loading 

When loading data using the Input or Output tabs, the Add-In logs specific status messages to help you quickly assess the result of the operation: 

 

✅ Success 

  • Message: Output Data loaded successfully. 
  • Indicates that the data was loaded completely without any issues. 

 

⚠️ No Data Loaded 

  • Message: No Data loaded. Check the Logs sheet for more details. 
  • Triggered when the data could not be loaded due to a configuration or connection issue. 

 

⚠️ Partial Load 

  • Message: Output Data loaded partially. Check the Logs sheet for more details. 
  • Indicates that only some of the data was loaded into the Excel sheet, and further investigation is needed. 

 

➤ Logs for Study-Agnostic Output Query Errors 

The Output Query Configuration sheet supports Study-Agnostic behavior, allowing you to retain diagnostic settings when switching between Studies.

 

To support this functionality, the Logs sheet displays validation messages when invalid or unmatched configurations are detected during a Study switch. 
 
Example: 
 
If you previously added a Solution from Model A, and upon switching to a new Study, Model A does not exist: 

 

  • The Model and Solution columns in the Output Query Configuration sheet will be left blank. 

  • The Logs sheet will display the following error message: 
    • "Model Name 'X' from the previous Study was not found in the selected Study. It has been left blank. Please fill in the correct details." 

 

 

These validations help ensure you’re always aware of data load status and potential mismatches when working across multiple Studies, reducing confusion and improving overall reliability within your Excel-based workflows.

While creating new Objects, the Memberships can be created only with already existing Objects.

For example: If I try to create a new Line A running from Node1 to Node2 cannot be new Nodes that are being added in the same version. So to create Line A, first create Node1 and Node2 and push your changes, then pull the latest data and create Line A and push your changes again.

  • Property data addition to multiple bands is not supported.

  • For Attribute Data, Longitude and Latitude will not be pulled. The template sheet will have includeGeocode column use this.

Persistence of Study and DataFiles Connection

Previously, the changes made to the Data Files had to be pushed before closing the Excel but the Persistent Data File functionality eliminate this effort by allowing the Data File to load automatically with the recent changes saved when you save the Data File and re-open it again. It will select the associated Study and Data Files by reading the data from the Config sheet. The MS Excel Add-In preserve the connection to the last study used, as well as any data files that was open during the last editing session. When the user re-opens the Excel editing template file, the Study selection will be populated automatically, and any data file tabs will be re-opened.

Perform the following steps to use this feature:

1.  Select the desired Study from the Select Study dropdown.
 


2.  The Datafiles associated with the Study will show in the Select Timeseries File dropdown.Browse and select the desired Timeseries File.


3.  Click on the OPEN FILE(S) button to view and edit the data.

NOTE: All columns in the Timeseries File are editable.


4.  A new sheet called Configs gets created. This sheet shows Study ID and Study Name for the currently selected Study. Do not delete this sheet or change the data, it will be used by the Add-In and API to communicate with PLEXOS Cloud.
 


5.  Make the necessary changes in the data file and click on File menu > Save As to save the Data File in .xlsx format. So, next time when you open the Data File, you will see the previously saved changes.
 


6.  Close the Excel Workbook and re-open it from the path where you have saved it earlier.

7.  Navigate to the Data tab and click on the PLEXOS icon (Energy Exemplar).
 


8.  PLEXOS Cloud automatically reads the data from the Config. sheet (Study ID, Name) and loads the associated Study and Data File.
 

9.  Make the necessary changes to the values of the Datafile.

10.  Click on the Save Data button to save your changes.
 

Configure and Manage Multiple Timeseries Files with New Data File Query Configuration Sheet

The PLEXOS Cloud Excel Add-In includes Data File Query Configuration sheet that enables you to manage and extract multiple timeseries files within the same Study more efficiently. 

 


 
This configuration table streamlines how you work with timeseries files by allowing you to add and manage multiple files in a single, centralized view. It enables you to selectively extract specific files or load all of them at once into Excel using the Bulk Load functionality, removing the need to process each timeseries file individually. 


➤ What It Does?

The Data File Query Configuration sheet lists your timeseries files in a structured, editable table and works seamlessly with the Bulk Load functionality.  


 
The table consists of the following columns: 

 

  • Row ID: Auto-generated row number for tracking individual file entries. 

  • Timeseries: Name of the selected timeseries file from your Study. 

  • Sheet Name: The target Excel sheet where data from the timeseries file will be loaded. 

  • Extract: Set to YES or NO to determine whether the file should be loaded when using the Load All option in the Bulk Load section.

Viewing Study Version and Commit Information in Excel

The Excel Add-In enables you to view the Study Version and Commit Message associated with each Simulation Output directly within the spreadsheet interface. This information provides enhanced traceability and helps contextualize simulation results without needing to refer back to PLEXOS Cloud or configuration logs.

 

  • Version Number: Indicates the Study change set version used to run the simulation, helping you identify which version of the Study was executed.

  • Commit Message: A custom description provided by the user when changes are pushed to a Study. This message helps explain what updates or modifications were made in that specific change set.

 

This visibility allows you to track version history and understand model changes directly within the Excel interface,  without switching to PLEXOS Cloud or inspecting configuration logs. It simplifies auditing, debugging, and collaboration by surfacing this important data alongside your output data.

 

➤ Key Benefits

  • Improves traceability by clearly showing version and change context alongside each Solution.

  • Saves time by reducing the need to trace back changes from the PLEXOS Cloud manually. 

  • Supports version comparison across multiple Solutions or Studies.

  • sheetName: sheetName is the name of the sheet where you want the table to be created. For example, we have entered Categories as sheetName in the screenshot below.
     
  • tableName: The tableName is the combination of sheetName and tableName. This is the unique name and will be helpful for easy identification of the respective table.
     
  • IncludeGeocode: Enter the value “No”. IncludeGeocode is not required as Category column is not dependent on Location Data (Latitude or Longitude).
     
  • enablePullData: Enter the value “Yes”. enablePullData will allow you to fetch the data specific to sheetName+tableName table selected in the template.

 

NOTE: You don’t have to enter anything in the className and category columns because the Category column is not dependent on any Class and its categories to fetch the data.

 

     

 

2.  Switch to the Schema sheet and enter the values as shown below:
 

  • sheetName: sheetName is the name of the sheet where you want the table to be created.

  • columnName: The columnName is the name of the PLEXOS Class. 

 

  • plexosEquivalent: Enter the value “Category”. plexosEquivalent is the Class or Property name in the PLEXOS input database. 

 

  • type: Enter the value “Name”. type is the type of element, for example, Class, Property, Name, Attribute, CustomColumn, etc.

 

  • keyType: Enter the value “Category”. keyType can be decided based on the value for “Type”. For example, Object, Category, Membership, Data, AttributeData, etc.

5.  Add a new row and enter the desired Category name in the Category column.

NOTE: The ClassName needs to match with the PLEXOS Classes and the drop-down helps the user in this scenario.

Tag Scenarios to Properties

The Excel Add-In now provides support for tagging Scenario Objects to any Property data. The Scenario column in the {ClassName}Property sheet allows user-friendly editing of Scenarios, by providing a drop-down input with the list of available Scenarios.

Using the drop-down, the users can browse the available Scenario names and select, avoiding the need to remember names or accidentally enter an invalid name. Users previously had to enter Scenario Object names manually, so this improvement reduces the required keystrokes for users. 

 

 

Timeslice support for Timeslice Files

The Excel Add-In now provides Timeslice support for Timeslice Files for a Class property. The two new columns get created when data is loaded. Users can attach the Timeslice tag to any property in the Timeslice Tag column to identify variations and add the Timeslice path in the Timeslice Text column.

The Timeslice reference can be defined for a property using the Timeslice Tag and Timeslice Text columns. These columns are editable. Refer to the description of the columns below:

  • Timeslice Tag: Users can Add, Edit, and Delete Object Names from the Timeslice Class.
  • Timeslice Text: Users can Add, Edit, and Delete any existing file path (For example, data, values, .csv, etc.)

View and Edit Dynamic Properties

The updated Template format in the Excel Add-In allows users to view Dynamic properties along with Static properties in any [ClassName] Property Sheet. This will eliminate the need for users to specify Dynamic properties (those Properties with a Datafile, DateFrom, DateTo, Timeslice, or Scenario), as these will be automatically determined based on the PLEXOS input database.

Static and dynamic properties will be imported into tables in separate worksheets. Like PLEXOS Desktop, users can choose to specify the "allDynamic" flag in the Template configuration table, which will import all properties into a single worksheet using the dynamic property schema.

 

To view Dynamic Properties:

 

NOTE: The user can specify the Property only in the Schema, and the property will be treated as Static or Dynamic based on:


  • Property setting in the database XML

  • The flag "All Dynamic" is set to True in the database XML, meaning that all attributes/properties are treated as Dynamic

 


  1. Open MS Excel and Log into the Excel Add-In. Select the desired Template to load.

  2. Navigate to the Template sheet and add a new column with a header name allDynamic.

  3. Enter the value Yes in the allDynamic column against the desired sheetName(s). This will fetch all the dynamic properties associated with that Class.


  4. Note that, if a user enters any value other than Yes, it will be considered as No.

    NOTE:     By default all the Dynamic Properties will be loaded in a {ClassName} Property sheet and the Static Properties in a ClassName sheet.

  5. Click on the LOAD DATA button to view the ClassName Property sheet.

  6. You can now view the properties in the Property column of the {ClassName} Property sheet for which you entered the value Yes in the Template sheet.

Configure Template and Schema using the Template Builder

The Template Builder feature enables users to interactively create or modify a Template, defining various Classes and Properties to be synchronized for editing/updating within the Excel workbook.


This feature offers a user-friendly interface for constructing or editing Templates, given the extensive range of Classes and Properties that can be edited in a PLEXOS Model. Users can browse Classes, and select their Properties, Categories, Memberships, and Attributes to construct a Template that specifies the properties synchronized via the Excel Add-In.


Utilizing the Template Editor enhances the process of filtering, finding, and selecting the correct properties to include in a Schema, eliminating the potential for manual data input errors that can occur when manually editing the Template. This saves time and minimizes errors for users creating Templates for themselves or sharing them with their team. 

 

Follow the steps below to Create or Modify a Template and Schema using the Template Builder:

 

  1. In the Excel Add-In, navigate to the INPUT DATA tab and click on the EDIT button.

  2. The SYSTEM and SIMULATION tab will get loaded showing the list of Classes, Categories, Properties, and Memberships in a similar format as available in PLEXOS Desktop.



  3. In the SYSTEM tab, select the desired Library (ClassGroup, Class, and Category) from the given list; you will see the Attributes, Properties, and Memberships data for the selected Class Name on the right-hand side of the screen.

    NOTE: A Search bar is available below the SYSTEM & SIMULATION tabs to perform a specific search for the lowest level elements such as Categories, Properties, Memberships, and Attributes.

  4. Check/uncheck to select the desired Properties, Memberships, and Attributes from the currently selected Class.



    NOTE:     You can select multiple Classes, Categories, Properties, Memberships, and Attributes at a time.

  5. Once all selections have been made, click on the APPLY button and the template will be updated in the Template and Schema worksheet tabs.

    -OR-

    Click on the APPLY & LOAD button to update the Template and Schema worksheet and also load data at the same time.

  6. The Template and Schema sheet get loaded on the left side of the screen showing the data selected in the Edit Template screen.

 

Edit Property and Membership for one-to-many Relationships

Users can perform the Input Data Editing and Synchronization for Memberships and Properties having one-to-many relationships between objects. This feature enhances Property Data and Membership editing by using the alternative data table format equivalent to PLEXOS Desktop to synchronize one-to-many relationships using Excel Add-In. Additionally, users can also specify these one-to-many relationships in the Template Builder. 

 

To support this enhancement, a new column “Parent Object” was added to the Properties sheet for dynamic Input Properties to identify the Parent of the Properties with one-to-many relationships for each Object in the Class. Previously, Property Data Synchronization was available only for Memberships having one-to-one relationships between Objects. Now, with the introduction of the "Parent Object" column in the Properties sheet, users gain the ability to edit Property Data for one-to-many relationships in the Property sheets of Excel Add-In, which displays data in a dedicated worksheet for both Static and Dynamic Properties.

Edit Simulation Horizon Start Dates (DateFrom and ChronoDateFrom)

The Excel Add-In supports editing the DateFrom and ChronoDateFrom fields for Horizon objects. These attributes define the start of the simulation horizon and can be directly configured within the spreadsheet environment.

 

These attributes define the start of the Simulation horizon:

 

  • DateFrom: Specifies the calendar-based starting point of the simulation for the Planning Horizon, which includes Preview, LT, PASA, and MT phases.

  • ChronoDateFrom: Indicates the chronological start of the simulation when using a Chronological Horizon, applicable to the ST Phase.

 

These fields are editable within the Input Data tab. When values are entered, they are recognized and automatically formatted as valid dates by the Add-In.

 

 

NOTE: When updating the DateFrom and ChronoDateFrom values, ensure that the Chronological Horizon (ST) remains a subset of the Planning Horizon. Misalignment between these may lead to simulation errors.

➤ Why This Helps?

  • This  improves usability by allowing you to control simulation start parameters without switching between tools, saving time and streamlining your workflow.  

  • Enhances Playbook development workflows, where users building Models in Excel Add-In can  easily adjust the horizon settings for each Simulation or Model directly from the spreadsheet.

Excel Add-In Validations and Restrictions

The Excel Add-In ensures accurate data linking, efficient configuration, and better error visibility when working with PLEXOS Studies.

 

  1. File Size Restrictions for Excel File Linking

    1. Displays validation messages when large Excel files are linked to a study, preventing misconfigurations and ensuring reliable uploads.

    2. File Size Limits:

      1. XLSB (binary files): Max size is 20 MB.

      2. XLSM / XLSX: Max size is 85 MB.

    3. Files exceeding the limit trigger an error message and log entry.

  2. Row Limit in Output Query Configuration Sheet

    1. Row limit is 10 to 100 in the Simulation Output Query Configuration table.


    2. Adding more than 100 rows prompts a validation message.

  3. Schema Validation with Error Reporting

    1. Automatically checks schema definitions and flags invalid/mistyped entries (e.g., wrong class, category, type, or attribute names) in the Template Schema.


    2. Invalid values are skipped, logged with timestamps, and clearly highlighted for easy correction.


Predefined Attribute Selection in Excel Add-In

The Excel Add-In allows you to select predefined simulation attribute values from a dropdown menu, removing the need to enter them manually.

 

Supported fields display a drop-down list of valid attribute values, eliminating the need to check PLEXOS Desktop or external documentation to confirm correct entries.

 


 NOTE: Drop-downs appear only for string-based attributes with fixed allowed values. Fields that take numbers or dates (like integers or timestamps) do not support drop-downs.
