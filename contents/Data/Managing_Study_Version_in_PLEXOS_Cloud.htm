Managing Study Versions in PLEXOS Cloud

Users can now save any version of a Study as a new Study. This action allows users to take a specific state, or version, of their Study and create a new one from it. This is useful if many changes were made that users aren’t happy with, and they want to revert to an earlier state. This functionality previously only existed in Cloud Bridge.

 

Note that the existing Study will not be changed, this creates a brand-new Study from an existing version. If the existing Study is no longer needed, be sure to Archive or Delete it. Follow the steps below to perform the Save as New Study action:


1.  Select a Study and navigate to the DATA > VERSIONS tab.

2.  Select the desired Version from the list and click the Actions button (Kebab menu).

 

       

3.  Click on Save as New Study.

 

4.  By default, the new Study name appears as "copy of {Studyname, current version}". You can change the name per your requirement.

 


       


NOTE: If you enter an existing or duplicate name, an error message, “Enter valid name” displays and the CONFIRM button gets disabled.

 

        


5.  Click on the CONFIRM button and a pop-up appears at the top of the screen to notify users that the Study creation process has started successfully.


       


6.  After the successful creation of a new study, another pop-up appears to notify users that the new Study is created successfully even if the user is currently in a different study or tab within the same study.


       
 

7.  Also, users will now receive email notifications indicating the Successful -OR- Failed completion status of the Study creation process. Additionally, users can view the activity change mentioned in the Activity History panel in the Overview tab. Please refer to the image below for a better understanding:

 

 

8.  To view the new Study, refresh the browser -OR- click the Refresh icon in the Study dropdown. The saved Study will appear in the Study dropdown list.

 

 

 

Rollback to a prior Study Version

The Rollback feature allows users to undo no longer required changes and avoid proceeding with them. Users can Rollback on any Study Version to revert the changes from that Version as a new Version, using the new Rollback option in the VERSIONS tab of the Data page. 


Executing a Rollback action will create a new Version that shows the desired state with detailed information about the Rollback process, including username, date, and the changes. 


For Example, if the Study has a total of 10 Versions for the Study and the user performs a Rollback on Version 4. A new Version 11 will be created that reverts all the changes that were done in Versions 5 through 10 and returns the study to the state it was in with the original Version 4. All the Versions between 4-10 will still be available in the list.   
 

 

NOTE: If a Rollback is performed on a Study Version having no changes to revert from the latest Version. The Rollback will fail, and the message, No changes found between versions will be displayed.

Key Scenarios Where Rollback Option Appears Disabled

Under specific circumstances, the Rollback option is disabled for users to maintain Data Integrity. Refer to the scenarios' details mentioned below and how they optimize your experience with version control in PLEXOS.

 

The Rollback option will be disabled in the following scenarios:
 

  • Version Rollback in Progress: When a user initiates the Rollback on any Study Version from the VERSIONS tab of the Data page, the Rollback option remains disabled with a tooltip message stating, “Version rollback is in progress.” This remains disabled until the ongoing Rollback process is complete, ensuring no conflicts and seamless collaboration. After the process is complete, the Rollback option will be enabled for the user again. 

 

  • New Version Creation: When a user makes any changes to the Study and saves it, a new version is created in the VERSIONS tab. In this case, the Rollback option remains disabled with a tooltip message stating, “A new version is being created for the study.” The Rollback option becomes available once the creation process is finished.

 

Follow the steps below to perform a Rollback your Study:
 

  1. Select the desired Study from the Study dropdown and navigate to the VERSIONS tab under the Data page.

  2. Select the desired Study Version card and click on the kebab menu (three vertical dots).

  3. Click on the Rollback option.


  4. A confirmation pop-up appears as shown below. Click on the YES, ROLLBACK button to continue.


  5. While the Rollback is in progress, you can continue with other tasks or log out of PLEXOS Cloud. An email notification is sent to you upon successful completion or failure of the Rollback process.
     
  6. After the Rollback process is successful, a toast message appears at the top center of your screen (If you are still active on the Data page.)

    NOTE: If a Rollback fails a message, “Rollback is failed” appears and users can trigger a Rollback again.

  7. Also, a new Version appears in the list that shows the details of the rollback.

 

Rollback Commit Message

Use the new Commit Message field to input a message explaining the purpose of the Rollback. A well-structured commit message increases transparency and traceability to your Study Rollback history, especially when multiple users participate in a single Study.
 

This enables all collaborators to understand the reason for the Rollback, including the identity of the person who made the changes. By default, the following Commit Message is displayed, “Changes from version {Next version in the queue on which the Rollback is performed-Latest version available for the Study before the Rollback} undone.” Users can also enter their custom message for Rollback commit. 
 

Example: If a user performs a Rollback on version 1, the default Commit Message, “Changes from version 02 - 03 undone” is displayed. So, the latest version 4 is created that reverts the changes of versions 02 and 03.
